# Огляд онбордингу

OpenClaw має два шляхи онбордингу. Обидва налаштовують автентифікацію, Gateway і
необов’язкові канали чату — вони відрізняються лише способом взаємодії з налаштуванням.

## Який шлях мені слід вибрати?

|                | Онбординг через CLI                     | Онбординг у застосунку macOS |
| -------------- | --------------------------------------- | ---------------------------- |
| **Платформи**  | macOS, Linux, Windows (нативно або WSL2) | лише macOS                   |
| **Інтерфейс**  | Майстер у терміналі                     | Покроковий UI у застосунку   |
| **Найкраще для** | Серверів, headless, повного контролю  | Настільного Mac, візуального налаштування |
| **Автоматизація** | `--non-interactive` для скриптів     | Лише вручну                  |
| **Команда**    | `openclaw onboard`                      | Запустіть застосунок         |

Більшості користувачів слід почати з **онбордингу через CLI** — він працює всюди й дає
вам найбільший контроль.

## Що налаштовує онбординг

Незалежно від того, який шлях ви оберете, онбординг налаштовує:

1. **Провайдера моделі та автентифікацію** — API-ключ, OAuth або токен налаштування для вибраного провайдера
2. **Робочий простір** — каталог для файлів агентів, bootstrap-шаблонів і пам’яті
3. **Gateway** — порт, адреса прив’язки, режим автентифікації
4. **Канали** (необов’язково) — вбудовані та комплектні канали чату, такі як
   BlueBubbles, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams,
   Telegram, WhatsApp тощо
5. **Daemon** (необов’язково) — фонову службу, щоб Gateway запускався автоматично

## Онбординг через CLI

Запустіть у будь-якому терміналі:

```bash
openclaw onboard
```

Додайте `--install-daemon`, щоб також установити фонову службу за один крок.

Повний довідник: [Онбординг (CLI)](/start/wizard)
Документація команди CLI: [`openclaw onboard`](/cli/onboard)

## Онбординг у застосунку macOS

Відкрийте застосунок OpenClaw. Майстер першого запуску проведе вас через ті самі кроки
за допомогою візуального інтерфейсу.

Повний довідник: [Онбординг (застосунок macOS)](/start/onboarding)

## Користувацькі або невказані провайдери

Якщо вашого провайдера немає в списку онбордингу, виберіть **Custom Provider** і
введіть:

- Режим сумісності API (OpenAI-compatible, Anthropic-compatible або auto-detect)
- Base URL і API-ключ
- ID моделі та необов’язковий псевдонім

Кілька користувацьких endpoint можуть співіснувати — кожен отримує власний ID endpoint.
