# Розбір розташування каналу

OpenClaw нормалізує поширені розташування з чат-каналів у:

- стислий текст координат, доданий до вхідного тіла, і
- структуровані поля в корисному навантаженні контексту автовідповіді. Надані каналом мітки, адреси та підписи/коментарі відображаються в підказці через спільний JSON-блок недовірених метаданих, а не вбудовуються в тіло користувача.

Наразі підтримуються:

- **Telegram** (позначки розташування + venue + live locations)
- **WhatsApp** (`locationMessage` + `liveLocationMessage`)
- **Matrix** (`m.location` з `geo_uri`)

## Форматування тексту

Розташування відображаються як зрозумілі рядки без дужок:

- Позначка:
  - `📍 48.858844, 2.294351 ±12m`
- Назване місце:
  - `📍 48.858844, 2.294351 ±12m`
- Поширення в реальному часі:
  - `🛰 Розташування в реальному часі: 48.858844, 2.294351 ±12m`

Якщо канал містить мітку, адресу або підпис/коментар, вони зберігаються в корисному навантаженні контексту та з’являються в підказці як огороджений недовірений JSON:

````text
Розташування (недовірені метадані):
```json
{
  "latitude": 48.858844,
  "longitude": 2.294351,
  "name": "Eiffel Tower",
  "address": "Champ de Mars, Paris",
  "caption": "Meet here"
}
```
````

## Поля контексту

Коли розташування присутнє, до `ctx` додаються такі поля:

- `LocationLat` (число)
- `LocationLon` (число)
- `LocationAccuracy` (число, метри; необов’язково)
- `LocationName` (рядок; необов’язково)
- `LocationAddress` (рядок; необов’язково)
- `LocationSource` (`pin | place | live`)
- `LocationIsLive` (булеве значення)
- `LocationCaption` (рядок; необов’язково)

Засіб рендерингу підказок обробляє `LocationName`, `LocationAddress` і `LocationCaption` як недовірені метадані та серіалізує їх через той самий обмежений шлях JSON, що й для іншого контексту каналу.

## Примітки щодо каналів

- **Telegram**: venue зіставляються з `LocationName/LocationAddress`; для live locations використовується `live_period`.
- **WhatsApp**: `locationMessage.comment` і `liveLocationMessage.caption` заповнюють `LocationCaption`.
- **Matrix**: `geo_uri` розбирається як розташування-позначка; висота ігнорується, а `LocationIsLive` завжди має значення false.
