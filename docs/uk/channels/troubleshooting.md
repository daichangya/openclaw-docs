# Усунення несправностей каналів

Використовуйте цю сторінку, коли канал підключається, але працює неправильно.

## Послідовність команд

Спочатку виконайте їх у такому порядку:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Ознаки здорового базового стану:

- `Runtime: running`
- `RPC probe: ok`
- Перевірка каналу показує, що транспорт підключено і, де це підтримується, `works` або `audit ok`

## WhatsApp

### Типові збої WhatsApp

| Симптом                         | Найшвидша перевірка                                | Виправлення                                            |
| ------------------------------- | -------------------------------------------------- | ------------------------------------------------------ |
| Підключено, але немає відповідей у приватних повідомленнях | `openclaw pairing list whatsapp`       | Підтвердьте відправника або змініть політику приватних повідомлень/allowlist. |
| Групові повідомлення ігноруються | Перевірте `requireMention` + шаблони згадок у конфігурації | Згадайте бота або послабте політику згадок для цієї групи. |
| Випадкові цикли відключення/повторного входу | `openclaw channels status --probe` + журнали | Увійдіть знову й перевірте, що каталог облікових даних у належному стані. |

Повне усунення несправностей: [/channels/whatsapp#troubleshooting](/channels/whatsapp#troubleshooting)

## Telegram

### Типові збої Telegram

| Симптом                             | Найшвидша перевірка                          | Виправлення                                                                 |
| ----------------------------------- | ------------------------------------------- | --------------------------------------------------------------------------- |
| `/start`, але немає придатного сценарію відповіді | `openclaw pairing list telegram`   | Підтвердьте прив’язку або змініть політику приватних повідомлень.           |
| Бот онлайн, але в групі мовчить     | Перевірте вимогу згадки та режим приватності бота | Вимкніть режим приватності для видимості в групі або згадайте бота.         |
| Збої надсилання з мережевими помилками | Перегляньте журнали на предмет збоїв викликів API Telegram | Виправте DNS/IPv6/маршрутизацію через проксі до `api.telegram.org`.         |
| `setMyCommands` відхиляється під час запуску | Перегляньте журнали на предмет `BOT_COMMANDS_TOO_MUCH` | Зменште кількість команд Telegram від plugins/Skills/користувацьких команд або вимкніть рідні меню. |
| Після оновлення allowlist блокує вас | `openclaw security audit` і allowlist у конфігурації | Запустіть `openclaw doctor --fix` або замініть `@username` на числові ID відправників. |

Повне усунення несправностей: [/channels/telegram#troubleshooting](/channels/telegram#troubleshooting)

## Matrix

### Типові збої Matrix

| Симптом                             | Найшвидша перевірка                  | Виправлення                                                                  |
| ----------------------------------- | ------------------------------------ | ---------------------------------------------------------------------------- |
| Вхід виконано, але повідомлення в кімнаті ігноруються | `openclaw channels status --probe` | Перевірте `groupPolicy`, allowlist кімнат і керування згадками.             |
| Приватні повідомлення не обробляються | `openclaw pairing list matrix`     | Підтвердьте відправника або змініть політику приватних повідомлень.          |
| Зашифровані кімнати не працюють     | `openclaw matrix verify status`      | Повторно підтвердьте пристрій, потім перевірте `openclaw matrix verify backup status`. |
| Відновлення резервної копії очікує/зламане | `openclaw matrix verify backup status` | Запустіть `openclaw matrix verify backup restore` або повторіть із ключем відновлення. |
| Cross-signing/bootstrap виглядає неправильно | `openclaw matrix verify bootstrap` | Виправте secret storage, cross-signing і стан резервної копії за один прохід. |

Повне налаштування й конфігурація: [Matrix](/channels/matrix)
