# Генерація та розуміння медіа

OpenClaw генерує зображення, відео й музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою перетворення тексту на мовлення. Усі медіаможливості керуються через інструменти: агент вирішує, коли їх використовувати, на основі розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного відповідного постачальника.

| -------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| Генерація зображень  | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                            | Створює або редагує зображення з текстових запитів чи референсів |
| Генерація відео      | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | Створює відео з тексту, зображень або наявних відео    |
| Генерація музики     | `music_generate` | ComfyUI, Google, MiniMax                                                                     | Створює музику або аудіодоріжки з текстових запитів    |
| Text-to-speech (TTS) | `tts`            | ElevenLabs, Microsoft, MiniMax, OpenAI, xAI                                                  | Перетворює вихідні відповіді на озвучене аудіо         |
| Розуміння медіа      | (автоматично)    | Будь-який постачальник моделей із підтримкою зору/аудіо, а також резервні варіанти CLI       | Узагальнює вхідні зображення, аудіо та відео           |

## Як працює асинхронна генерація

Генерація відео й музики виконується як фонові завдання, оскільки обробка у постачальників зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит постачальнику, одразу повертає ідентифікатор завдання та відстежує роботу в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли постачальник завершує обробку, OpenClaw пробуджує агента, щоб він міг опублікувати готове медіа назад у вихідний канал. Генерація зображень і TTS є синхронними та завершуються безпосередньо під час відповіді.

OpenAI зіставляється з поверхнями OpenClaw для зображень, відео, пакетного TTS, пакетного STT, потокового STT для Voice Call, realtime voice і memory embedding. xAI наразі зіставляється з поверхнями OpenClaw для зображень, відео, пошуку, виконання коду, пакетного TTS, пакетного STT і потокового STT для Voice Call. Голос xAI Realtime є можливістю на боці провайдера, але в OpenClaw він поки не зареєстрований, доки спільний контракт realtime voice не зможе його подати.
