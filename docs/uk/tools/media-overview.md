# Генерація та розуміння медіа

OpenClaw генерує зображення, відео та музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою перетворення тексту на мовлення. Усі медіаможливості керуються через інструменти: агент вирішує, коли їх використовувати, на основі розмови, і кожен інструмент з’являється лише тоді, коли налаштовано щонайменше одного базового постачальника.

## Як працює асинхронна генерація

Генерація відео та музики виконується як фонові завдання, оскільки обробка у постачальників зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит постачальнику, одразу повертає ідентифікатор завдання та відстежує роботу в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли постачальник завершує обробку, OpenClaw активує агента, щоб він міг опублікувати готове медіа назад у початковий канал. Генерація зображень і TTS є синхронними та завершуються в межах відповіді.

Deepgram, ElevenLabs, Mistral, OpenAI, SenseAudio та xAI можуть транскрибувати
вхідне аудіо через пакетний шлях `tools.media.audio`, якщо їх налаштовано.
Плагіни каналів, які попередньо обробляють голосове повідомлення для
перевірки згадки або розбору команд, позначають транскрибоване вкладення у
вхідному контексті, тому спільний етап розуміння медіа повторно використовує
цю транскрипцію замість виконання другого виклику STT для того самого аудіо.
Deepgram, ElevenLabs, Mistral, OpenAI та xAI також реєструють постачальників
потокового STT для Voice Call, тому живий телефонний аудіопотік можна
передавати вибраному постачальнику без очікування завершення запису.

Google відповідає поверхням OpenClaw для зображень, відео, музики, пакетного TTS, backend realtime
voice та розуміння медіа. OpenAI відповідає поверхням OpenClaw для зображень,
відео, пакетного TTS, пакетного STT, потокового STT для Voice Call, backend realtime voice
та векторних подань пам’яті. xAI наразі відповідає поверхням OpenClaw для зображень, відео,
пошуку, виконання коду, пакетного TTS, пакетного STT та потокового STT для Voice Call.
Голос xAI Realtime є можливістю на рівні зовнішнього постачальника, але він не
реєструється в OpenClaw, доки спільний контракт голосу в реальному часі не зможе
його відобразити.
