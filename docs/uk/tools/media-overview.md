# Генерація та розуміння медіа

OpenClaw генерує зображення, відео й музику, розуміє вхідні медіа (зображення, аудіо, відео) та озвучує відповіді за допомогою text-to-speech. Усі медіаможливості керуються інструментами: агент вирішує, коли їх використовувати, залежно від розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного відповідного provider.

## Можливості з першого погляду

| Можливість          | Інструмент       | Provider                                                                                     | Що робить                                              |
| ------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| Генерація зображень | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                            | Створює або редагує зображення за текстовими prompt чи посиланнями |
| Генерація відео     | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | Створює відео з тексту, зображень або наявних відео    |
| Генерація музики    | `music_generate` | ComfyUI, Google, MiniMax                                                                     | Створює музику або аудіодоріжки з текстових prompt     |
| Text-to-speech (TTS) | `tts`           | ElevenLabs, Microsoft, MiniMax, OpenAI, xAI                                                  | Перетворює вихідні відповіді на озвучене аудіо         |
| Розуміння медіа     | (автоматично)    | Будь-який provider моделі з підтримкою vision/audio, а також резервні CLI-варіанти           | Підсумовує вхідні зображення, аудіо та відео           |

## Як працює асинхронна генерація

Генерація відео й музики виконується як фонові завдання, оскільки обробка provider зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит provider, одразу повертає ідентифікатор завдання й відстежує роботу в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли provider завершує роботу, OpenClaw пробуджує агента, щоб той міг надіслати готове медіа назад у початковий канал. Генерація зображень і TTS є синхронними та завершуються в межах відповіді.

Deepgram, ElevenLabs, Mistral, OpenAI і xAI можуть транскрибувати вхідне
аудіо через пакетний шлях `tools.media.audio`, якщо це налаштовано. Deepgram,
ElevenLabs, Mistral, OpenAI і xAI також реєструють provider потокового STT для Voice Call,
тому живе телефонне аудіо можна пересилати вибраному вендору
без очікування завершення запису.

OpenAI зіставляється з поверхнями OpenClaw для зображень, відео, пакетного TTS, пакетного STT, потокового STT для Voice Call, realtime voice і memory embedding. xAI наразі
зіставляється з поверхнями OpenClaw для зображень, відео, пошуку, виконання коду, пакетного TTS, пакетного STT
і потокового STT для Voice Call. Realtime voice xAI — це
можливість upstream, але вона не зареєстрована в OpenClaw, доки спільний контракт
realtime voice не зможе її представити.
