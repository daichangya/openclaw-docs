# Вхід у браузері + публікації в X/Twitter

## Ручний вхід (рекомендовано)

Коли сайт вимагає входу, **увійдіть вручну** в профілі браузера **хоста** (браузер openclaw).

**Не** передавайте моделі свої облікові дані. Автоматизовані входи часто запускають захист від ботів і можуть заблокувати обліковий запис.

Повернутися до основної документації браузера: [Browser](/uk/tools/browser).

## Який профіль Chrome використовується?

OpenClaw керує **окремим профілем Chrome** (з назвою `openclaw`, із помаранчевим відтінком UI). Він відокремлений від вашого щоденного профілю браузера.

Для викликів browser tool агентом:

- Типовий вибір: агент має використовувати свій ізольований браузер `openclaw`.
- Використовуйте `profile="user"` лише коли важливі наявні сесії з виконаним входом і користувач перебуває за комп’ютером, щоб натиснути/підтвердити будь-який prompt на приєднання.
- Якщо у вас кілька профілів користувацького браузера, задавайте профіль явно, а не вгадуйте.

Є два прості способи отримати до нього доступ:

1. **Попросіть агента відкрити браузер**, а потім увійдіть самостійно.
2. **Відкрийте його через CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Якщо у вас кілька профілів, передайте `--browser-profile <name>` (типове значення — `openclaw`).

## X/Twitter: рекомендований процес

- **Читання/пошук/потоки:** використовуйте браузер **хоста** (ручний вхід).
- **Публікація оновлень:** використовуйте браузер **хоста** (ручний вхід).

## Sandboxing + доступ до браузера хоста

Сесії браузера в sandbox **з більшою ймовірністю** запускають виявлення ботів. Для X/Twitter (та інших суворих сайтів) надавайте перевагу браузеру **хоста**.

Якщо агент працює в sandbox, browser tool типово використовує sandbox. Щоб дозволити керування хостом:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Потім націлюйтеся на браузер хоста:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

Або вимкніть sandboxing для агента, який публікує оновлення.
