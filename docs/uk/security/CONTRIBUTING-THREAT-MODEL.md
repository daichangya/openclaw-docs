# Внесок у модель загроз OpenClaw

Дякуємо, що допомагаєте зробити OpenClaw безпечнішим. Ця модель загроз — живий документ, і ми раді внескам від будь-кого — вам не потрібно бути експертом із безпеки.

## Способи зробити внесок

### Додати загрозу

Помітили вектор атаки або ризик, який ми ще не охопили? Відкрийте issue у [openclaw/trust](https://github.com/openclaw/trust/issues) і опишіть його своїми словами. Вам не потрібно знати жодних фреймворків або заповнювати кожне поле — просто опишіть сценарій.

**Корисно вказати (але не обов’язково):**

- Сценарій атаки та те, як його можна експлуатувати
- Які частини OpenClaw зачіпає проблема (CLI, gateway, канали, ClawHub, сервери MCP тощо)
- Наскільки серйозною ви вважаєте проблему (low / medium / high / critical)
- Будь-які посилання на пов’язані дослідження, CVE або реальні приклади

Ми самі виконаємо відображення на ATLAS, призначення ідентифікаторів загроз і оцінку ризику під час рев’ю. Якщо хочете додати ці деталі — чудово, але це не очікується.

> **Це призначено для доповнення моделі загроз, а не для повідомлення про активні вразливості.** Якщо ви знайшли вразливість, яку можна експлуатувати, перегляньте нашу [сторінку Trust](https://trust.openclaw.ai) для інструкцій щодо відповідального розкриття.

### Запропонувати пом’якшення

Маєте ідею, як усунути наявну загрозу? Відкрийте issue або PR із посиланням на цю загрозу. Корисні пом’якшення є конкретними та придатними до дії — наприклад, “обмеження швидкості для кожного відправника до 10 повідомлень/хвилину на gateway” краще, ніж “реалізувати rate limiting”.

### Запропонувати ланцюг атаки

Ланцюги атак показують, як кілька загроз поєднуються в реалістичний сценарій атаки. Якщо ви бачите небезпечну комбінацію, опишіть кроки та те, як зловмисник міг би поєднати їх у ланцюг. Короткий наратив про те, як атака розгортається на практиці, цінніший за формальний шаблон.

### Виправити або покращити наявний вміст

Друкарські помилки, уточнення, застаріла інформація, кращі приклади — PR вітаються, issue не потрібен.

## Що ми використовуємо

### MITRE ATLAS

Цю модель загроз побудовано на основі [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) — фреймворку, розробленого спеціально для загроз AI/ML, таких як prompt injection, зловживання інструментами та експлуатація агентів. Вам не потрібно знати ATLAS, щоб зробити внесок — ми зіставляємо подання з цим фреймворком під час рев’ю.

### Ідентифікатори загроз

Кожна загроза отримує ідентифікатор на кшталт `T-EXEC-003`. Категорії такі:

| Code    | Категорія                                   |
| ------- | ------------------------------------------- |
| RECON   | Розвідка — збирання інформації              |
| ACCESS  | Початковий доступ — отримання входу         |
| EXEC    | Виконання — запуск зловмисних дій           |
| PERSIST | Закріплення — збереження доступу            |
| EVADE   | Ухилення від захисту — уникнення виявлення  |
| DISC    | Виявлення — вивчення середовища             |
| EXFIL   | Ексфільтрація — викрадення даних            |
| IMPACT  | Вплив — пошкодження або порушення роботи    |

Ідентифікатори призначаються супроводжувачами під час рев’ю. Вам не потрібно обирати його самостійно.

### Рівні ризику

| Рівень       | Значення                                                         |
| ------------ | ---------------------------------------------------------------- |
| **Critical** | Повна компрометація системи або висока ймовірність + критичний вплив |
| **High**     | Імовірна значна шкода або середня ймовірність + критичний вплив |
| **Medium**   | Помірний ризик або низька ймовірність + високий вплив           |
| **Low**      | Малоймовірно й обмежений вплив                                  |

Якщо ви не впевнені щодо рівня ризику, просто опишіть вплив, а ми оцінимо його.

## Процес рев’ю

1. **Тріаж** — ми переглядаємо нові подання протягом 48 годин
2. **Оцінка** — ми перевіряємо здійсненність, призначаємо відображення ATLAS і ідентифікатор загрози, підтверджуємо рівень ризику
3. **Документування** — ми гарантуємо, що все має правильний формат і є повним
4. **Злиття** — додавання до моделі загроз і візуалізації
