# Модель загроз OpenClaw v1.0

## Фреймворк MITRE ATLAS

**Версія:** 1.0-draft
**Останнє оновлення:** 2026-02-04
**Методологія:** MITRE ATLAS + діаграми потоків даних
**Фреймворк:** [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems)

### Посилання на фреймворк

Цю модель загроз побудовано на основі [MITRE ATLAS](https://atlas.mitre.org/), галузевого стандартного фреймворку для документування змагальних загроз для систем AI/ML. ATLAS підтримується [MITRE](https://www.mitre.org/) у співпраці зі спільнотою безпеки ШІ.

**Ключові ресурси ATLAS:**

- [Техніки ATLAS](https://atlas.mitre.org/techniques/)
- [Тактики ATLAS](https://atlas.mitre.org/tactics/)
- [Кейс-стаді ATLAS](https://atlas.mitre.org/studies/)
- [GitHub ATLAS](https://github.com/mitre-atlas/atlas-data)
- [Участь у розвитку ATLAS](https://atlas.mitre.org/resources/contribute)

### Участь у розвитку цієї моделі загроз

Це живий документ, який підтримує спільнота OpenClaw. Див. [CONTRIBUTING-THREAT-MODEL.md](/uk/security/CONTRIBUTING-THREAT-MODEL) для рекомендацій щодо внеску:

- повідомлення про нові загрози

- пропонування заходів пом’якшення

## 1. Вступ

### 1.1 Призначення

Ця модель загроз документує змагальні загрози для платформи ШІ-агентів OpenClaw і маркетплейсу Skills ClawHub, використовуючи фреймворк MITRE ATLAS, спеціально розроблений для систем AI/ML.

### 1.2 Межі охоплення

| Компонент              | Включено | Примітки                                          |
| ---------------------- | -------- | ------------------------------------------------- |
| Runtime агента OpenClaw | Так      | Основне виконання агента, виклики інструментів, сесії |
| Gateway                | Так      | Автентифікація, маршрутизація, інтеграція каналів |
| Інтеграції каналів     | Так      | WhatsApp, Telegram, Discord, Signal, Slack тощо   |
| Marketplace ClawHub    | Так      | Публікація, модерація, розповсюдження Skills      |
| Сервери MCP            | Так      | Зовнішні provider інструментів                    |
| Пристрої користувачів  | Частково | Мобільні застосунки, настільні клієнти            |

### 1.3 Поза межами охоплення

Для цієї моделі загроз немає нічого, що було б явно поза межами охоплення.

## 4. Аналіз ланцюга постачання ClawHub

### 4.1 Поточні механізми безпеки

| Механізм              | Реалізація                  | Ефективність                                           |
| --------------------- | --------------------------- | ------------------------------------------------------ |
| Вік облікового запису GitHub | `requireGitHubAccountAge()` | Середня — підвищує поріг для нових зловмисників |
| Санітизація шляхів    | `sanitizePath()`            | Висока — запобігає path traversal                      |
| Валідація типів файлів | `isTextFile()`             | Середня — лише текстові файли, але вони теж можуть бути шкідливими |
| Обмеження розміру     | 50 МБ на весь bundle        | Висока — запобігає вичерпанню ресурсів                 |
| Обов’язковий SKILL.md | Обов’язковий readme         | Низька цінність для безпеки — лише інформаційна        |
| Модерація за шаблонами | `FLAG_RULES` у `moderation.ts` | Низька — легко обходиться                         |
| Статус модерації      | Поле `moderationStatus`     | Середня — можлива ручна перевірка                      |

### 4.2 Шаблони прапорців модерації

Поточні шаблони в `moderation.ts`:

```javascript
// Known-bad identifiers
/(keepcold131\/ClawdAuthenticatorTool|ClawdAuthenticatorTool)/i

// Suspicious keywords
/(malware|stealer|phish|phishing|keylogger)/i
/(api[-_ ]?key|token|password|private key|secret)/i
/(wallet|seed phrase|mnemonic|crypto)/i
/(discord\.gg|webhook|hooks\.slack)/i
/(curl[^\n]+\|\s*(sh|bash))/i
/(bit\.ly|tinyurl\.com|t\.co|goo\.gl|is\.gd)/i
```

**Обмеження:**

- Перевіряються лише slug, `displayName`, `summary`, frontmatter, metadata, шляхи до файлів
- Фактичний код Skill не аналізується
- Простий regex легко обходиться через обфускацію
- Немає поведінкового аналізу

### 4.3 Заплановані покращення

| Покращення            | Статус                                | Вплив                                                               |
| --------------------- | ------------------------------------- | ------------------------------------------------------------------- |
| Інтеграція VirusTotal | У процесі                             | Високий — поведінковий аналіз Code Insight                          |
| Звітування спільноти  | Частково (`skillReports` table існує) | Середній                                                            |
| Аудит журналювання    | Частково (`auditLogs` table існує)    | Середній                                                            |
| Система значків       | Реалізовано                           | Середній — `highlighted`, `official`, `deprecated`, `redactionApproved` |

### 5.2 Критичні ланцюги атак

**Ланцюг атаки 1: Крадіжка даних через Skill**

```
T-PERSIST-001 → T-EVADE-001 → T-EXFIL-003
(Публікація шкідливого Skill) → (Обхід модерації) → (Збирання облікових даних)
```

**Ланцюг атаки 2: Prompt injection до RCE**

```
T-EXEC-001 → T-EXEC-004 → T-IMPACT-001
(Ін’єкція запиту) → (Обхід погодження exec) → (Виконання команд)
```

**Ланцюг атаки 3: Непряма ін’єкція через отриманий вміст**

```
T-EXEC-002 → T-EXFIL-001 → Зовнішня ексфільтрація
(Отруєння вмісту URL) → (Агент отримує його та виконує інструкції) → (Дані надсилаються зловмиснику)
```
