Розгорніть OpenClaw на Railway за допомогою шаблону в один клік і отримайте доступ через вебінтерфейс Control UI.
Це найпростіший шлях «без термінала на сервері»: Railway запускає Gateway за вас.

## Швидкий контрольний список (для нових користувачів)

1. Натисніть **Deploy on Railway** (нижче).
2. Додайте **Volume**, змонтований у `/data`.
3. Установіть обов’язкові **Variables** (щонайменше `OPENCLAW_GATEWAY_PORT` і `OPENCLAW_GATEWAY_TOKEN`).
4. Увімкніть **HTTP Proxy** на порту `8080`.
5. Відкрийте `https://<your-railway-domain>/openclaw` і підключіться, використовуючи налаштований спільний секрет. Цей шаблон типово використовує `OPENCLAW_GATEWAY_TOKEN`; якщо ви заміните його на автентифікацію паролем, використовуйте натомість цей пароль.

## Розгортання в один клік

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

Після розгортання знайдіть свій публічний URL у **Railway → ваш сервіс → Settings → Domains**.

Railway або:

- надасть вам згенерований домен (часто `https://<something>.up.railway.app`), або
- використає ваш власний домен, якщо ви його підключили.

Потім відкрийте:

- `https://<your-railway-domain>/openclaw` — Control UI

## Що ви отримаєте

- Розміщені OpenClaw Gateway + Control UI
- Постійне сховище через Railway Volume (`/data`), тож `openclaw.json`,
  `auth-profiles.json` для окремих агентів, стан каналів/провайдерів, сесії та
  workspace зберігаються після повторних розгортань

## Обов’язкові налаштування Railway

### Публічна мережа

Увімкніть **HTTP Proxy** для сервісу.

- Порт: `8080`

### Volume (обов’язково)

Підключіть volume, змонтований у:

- `/data`

### Variables

Установіть ці variables для сервісу:

- `OPENCLAW_GATEWAY_PORT=8080` (обов’язково — має збігатися з портом у Public Networking)
- `OPENCLAW_GATEWAY_TOKEN` (обов’язково; вважайте це адміністративним секретом)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (рекомендовано)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (рекомендовано)

## Підключення каналу

Використовуйте Control UI за шляхом `/openclaw` або запустіть `openclaw onboard` через shell Railway, щоб отримати інструкції з налаштування каналу:

- [Telegram](/channels/telegram) (найшвидше — потрібен лише токен бота)
- [Discord](/channels/discord)
- [Усі канали](/channels)

## Резервні копії та міграція

Експортуйте свій стан, конфігурацію, профілі автентифікації та workspace:

```bash
openclaw backup create
```

Це створює переносний архів резервної копії зі станом OpenClaw і будь-яким налаштованим
workspace. Докладніше див. у [Backup](/cli/backup).
