# AGENTS.md - персональний асистент OpenClaw (типово)

## Перший запуск (рекомендовано)

OpenClaw використовує окремий каталог робочого простору для агента. Типово: `~/.openclaw/workspace` (налаштовується через `agents.defaults.workspace`).

1. Створіть робочий простір (якщо він ще не існує):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте типові шаблони робочого простору до робочого простору:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов’язково: якщо вам потрібен перелік Skills для персонального асистента, замініть AGENTS.md цим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов’язково: виберіть інший робочий простір, задавши `agents.defaults.workspace` (підтримується `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Типові налаштування безпеки

- Не виводьте каталоги або секрети в чат.
- Не запускайте руйнівні команди без явного запиту.
- Не надсилайте часткові/потокові відповіді на зовнішні поверхні обміну повідомленнями (лише фінальні відповіді).

## Початок сеансу (обов’язково)

- Прочитайте `SOUL.md`, `USER.md` і записи за сьогодні+вчора в `memory/`.
- Прочитайте `MEMORY.md`, якщо він є; переходьте до нижнього регістру `memory.md` лише якщо `MEMORY.md` відсутній.
- Зробіть це до відповіді.

## Soul (обов’язково)

- `SOUL.md` визначає ідентичність, тон і межі. Підтримуйте його актуальним.
- Якщо ви змінюєте `SOUL.md`, повідомте користувача.
- У кожному сеансі ви — новий екземпляр; безперервність зберігається в цих файлах.

## Спільні простори (рекомендовано)

- Ви не є голосом користувача; будьте обережні в групових чатах або публічних каналах.
- Не діліться приватними даними, контактною інформацією або внутрішніми нотатками.

## Система пам’яті (рекомендовано)

- Щоденний журнал: `memory/YYYY-MM-DD.md` (створіть `memory/`, якщо потрібно).
- Довготривала пам’ять: `MEMORY.md` для стійких фактів, уподобань і рішень.
- Нижній регістр `memory.md` — лише застарілий резервний варіант; не зберігайте обидва кореневі файли навмисно.
- На початку сеансу прочитайте записи за сьогодні + вчора + `MEMORY.md`, якщо він є, інакше `memory.md`.
- Фіксуйте: рішення, уподобання, обмеження, відкриті цикли.
- Уникайте секретів, якщо цього явно не просять.

## Що робить OpenClaw

- Запускає шлюз WhatsApp + агент кодування Pi, щоб асистент міг читати/писати чати, отримувати контекст і запускати Skills через хостовий Mac.
- Застосунок macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через свій вбудований бінарний файл.
- Прямі чати типово зводяться до сеансу `main` агента; групи залишаються ізольованими як `agent:<agentId>:<channel>:group:<id>` (кімнати/канали: `agent:<agentId>:<channel>:channel:<id>`); heartbeat підтримують фонові завдання активними.

## Основні Skills (увімкніть у Settings → Skills)

- **mcporter** — runtime/CLI сервера інструментів для керування зовнішніми backend Skills.
- **Peekaboo** — швидкі знімки екрана macOS з необов’язковим AI-аналізом зору.
- **camsnap** — захоплення кадрів, кліпів або сповіщень про рух з камер безпеки RTSP/ONVIF.
- **oracle** — CLI агента, готовий до OpenAI, з відтворенням сеансів і керуванням браузером.
- **eightctl** — керуйте своїм сном із термінала.
- **imsg** — надсилайте, читайте, передавайте потоком iMessage і SMS.
- **wacli** — CLI WhatsApp: синхронізація, пошук, надсилання.
- **discord** — дії Discord: реакції, стікери, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (числові id без префікса неоднозначні).
- **gog** — CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** — термінальний клієнт Spotify для пошуку/черги/керування відтворенням.
- **sag** — мовлення ElevenLabs з UX у стилі mac `say`; типово передає аудіо на динаміки.
- **Sonos CLI** — керування колонками Sonos (виявлення/стан/відтворення/гучність/групування) зі скриптів.
- **blucli** — відтворення, групування й автоматизація програвачів BluOS зі скриптів.
- **OpenHue CLI** — керування освітленням Philips Hue для сцен і автоматизацій.
- **OpenAI Whisper** — локальне speech-to-text для швидкої диктовки та транскриптів голосової пошти.
- **Gemini CLI** — моделі Google Gemini з термінала для швидких запитань і відповідей.
- **agent-tools** — набір утиліт для автоматизацій і допоміжних скриптів.

## Примітки щодо використання

- Віддавайте перевагу CLI `openclaw` для скриптів; застосунок mac обробляє дозволи.
- Запускайте інсталяції з вкладки Skills; вона приховує кнопку, якщо бінарний файл уже наявний.
- Тримайте heartbeat увімкненими, щоб асистент міг планувати нагадування, відстежувати вхідні повідомлення й запускати захоплення з камер.
- Canvas UI працює на весь екран із нативними накладками. Не розміщуйте критичні елементи керування у верхньому лівому/верхньому правому/нижньому краях; додавайте явні відступи в макеті й не покладайтеся на safe-area insets.
- Для перевірки через браузер використовуйте `openclaw browser` (tabs/status/screenshot) з профілем Chrome, яким керує OpenClaw.
- Для інспекції DOM використовуйте `openclaw browser eval|query|dom|snapshot` (і `--json`/`--out`, коли вам потрібен машинний вивід).
- Для взаємодій використовуйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type вимагають посилань snapshot; використовуйте `evaluate` для CSS-селекторів).
