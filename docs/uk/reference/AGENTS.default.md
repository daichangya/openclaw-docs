# AGENTS.md - Персональний помічник OpenClaw (стандартно)

## Перший запуск (рекомендовано)

OpenClaw використовує окрему директорію робочого простору для агента. Типово: `~/.openclaw/workspace` (налаштовується через `agents.defaults.workspace`).

1. Створіть робочий простір (якщо він ще не існує):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте стандартні шаблони робочого простору до робочого простору:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов’язково: якщо вам потрібен перелік Skills для персонального помічника, замініть AGENTS.md цим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов’язково: виберіть інший робочий простір, задавши `agents.defaults.workspace` (підтримується `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

- Не виводьте списки директорій або секрети в чат.

- Щоденний журнал: `memory/YYYY-MM-DD.md` (створіть `memory/`, якщо потрібно).
- Довготривала пам’ять: `MEMORY.md` для сталих фактів, уподобань і рішень.
- Нижньорегістровий `memory.md` — лише вхід для виправлення застарілих даних; не зберігайте обидва кореневі файли навмисно.
- На початку сеансу прочитайте записи за сьогодні + вчора + `MEMORY.md`, якщо він присутній.

## Що робить OpenClaw

- Запускає WhatsApp Gateway + агент кодування Pi, щоб помічник міг читати/писати чати, отримувати контекст і запускати Skills через хостовий Mac.
- Застосунок macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через свій вбудований бінарний файл.
- Прямі чати типово згортаються в сеанс `main` агента; групи залишаються ізольованими як `agent:<agentId>:<channel>:group:<id>` (кімнати/канали: `agent:<agentId>:<channel>:channel:<id>`); Heartbeat підтримують активність фонових завдань.

## Основні Skills (увімкніть у Settings → Skills)

- **mcporter** — середовище виконання/CLI сервера інструментів для керування зовнішніми бекендами Skill.
- **Peekaboo** — швидкі знімки екрана macOS з необов’язковим AI-аналізом зображення.
- **camsnap** — захоплення кадрів, кліпів або сповіщень про рух із камер безпеки RTSP/ONVIF.
- **oracle** — сумісний з OpenAI CLI агента з відтворенням сеансів і керуванням браузером.
- **eightctl** — керуйте своїм сном із термінала.
- **imsg** — надсилання, читання, потокова передача iMessage і SMS.
- **wacli** — WhatsApp CLI: синхронізація, пошук, надсилання.
- **discord** — дії в Discord: реакції, стікери, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (числові ідентифікатори без префікса неоднозначні).
- **gog** — CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** — термінальний клієнт Spotify для пошуку/додавання в чергу/керування відтворенням.
- **sag** — мовлення ElevenLabs зі схожим на say UX для Mac; типово передає звук на динаміки.
- **Sonos CLI** — керування колонками Sonos (виявлення/стан/відтворення/гучність/групування) зі скриптів.
- **blucli** — відтворення, групування й автоматизація програвачів BluOS зі скриптів.
- **OpenHue CLI** — керування освітленням Philips Hue для сцен і автоматизацій.
- **OpenAI Whisper** — локальне перетворення мовлення на текст для швидкої диктовки й транскриптів голосової пошти.
- **Gemini CLI** — моделі Google Gemini з термінала для швидких запитань і відповідей.
- **agent-tools** — набір утиліт для автоматизацій і допоміжних скриптів.

## Примітки щодо використання

- Для скриптів надавайте перевагу CLI `openclaw`; застосунок Mac керує дозволами.
- Запускайте встановлення з вкладки Skills; вона приховує кнопку, якщо бінарний файл уже присутній.
- Тримайте Heartbeat увімкненими, щоб помічник міг планувати нагадування, відстежувати вхідні повідомлення та запускати захоплення з камер.
- Інтерфейс Canvas працює на весь екран із нативними накладками. Уникайте розміщення критично важливих елементів керування у верхньому лівому/верхньому правому/нижніх краях; додавайте явні відступи в макеті й не покладайтеся на відступи безпечної області.
- Для перевірки через браузер використовуйте `openclaw browser` (tabs/status/screenshot) з профілем Chrome, яким керує OpenClaw.
- Для інспекції DOM використовуйте `openclaw browser eval|query|dom|snapshot` (а також `--json`/`--out`, коли потрібен машинний вивід).
- Для взаємодій використовуйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type вимагають посилань snapshot; для CSS-селекторів використовуйте `evaluate`).
