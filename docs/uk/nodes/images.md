# Підтримка зображень і медіа (2025-12-05)

Канал WhatsApp працює через **Baileys Web**. У цьому документі зафіксовано поточні правила обробки медіа для надсилання, Gateway та відповідей агента.

## Цілі

- Надсилати медіа з необов’язковими підписами через `openclaw message send --media`.
- Дозволити, щоб автовідповіді з вебвхідних могли включати медіа разом із текстом.
- Зберігати обмеження для кожного типу розумними й передбачуваними.

## Поверхня CLI

- `openclaw message send --media <path-or-url> [--message <caption>]`
  - `--media` необов’язковий; підпис може бути порожнім для надсилання лише медіа.
  - `--dry-run` виводить визначений payload; `--json` повертає `{ channel, to, messageId, mediaUrl, caption }`.

## Поведінка каналу WhatsApp Web

- Вхід: локальний шлях до файлу **або** HTTP(S) URL.
- Потік: завантаження в Buffer, визначення типу медіа та побудова правильного payload:
  - **Зображення:** змінення розміру й повторне стиснення до JPEG (максимальна сторона 2048px) з орієнтацією на `channels.whatsapp.mediaMaxMb` (за замовчуванням: 50 MB).
  - **Аудіо/голосові повідомлення/відео:** передавання без змін до 16 MB; аудіо надсилається як голосове повідомлення (`ptt: true`).
  - **Документи:** усе інше, до 100 MB, зі збереженням імені файлу, якщо воно доступне.
- Відтворення у стилі GIF у WhatsApp: надсилайте MP4 з `gifPlayback: true` (у CLI: `--gif-playback`), щоб мобільні клієнти зациклювали його вбудовано.
- Визначення MIME спочатку використовує magic bytes, потім заголовки, потім розширення файлу.
- Підпис береться з `--message` або `reply.text`; порожній підпис дозволено.
- Журналювання: без verbose показує `↩️`/`✅`; verbose також включає розмір і вихідний шлях/URL.

## Конвеєр автовідповідей

- `getReplyFromConfig` повертає `{ text?, mediaUrl?, mediaUrls? }`.
- Коли є медіа, вебвідправник обробляє локальні шляхи або URL через той самий конвеєр, що й `openclaw message send`.
- Якщо надано кілька елементів медіа, вони надсилаються послідовно.

## Вхідні медіа до команд (Pi)

- Коли вхідні вебповідомлення містять медіа, OpenClaw завантажує їх у тимчасовий файл і надає змінні шаблонів:
  - `{{MediaUrl}}` псевдо-URL для вхідного медіа.
  - `{{MediaPath}}` локальний тимчасовий шлях, записаний перед запуском команди.
- Коли ввімкнено Docker sandbox для кожної сесії, вхідне медіа копіюється в робочий простір sandbox, а `MediaPath`/`MediaUrl` переписуються на відносний шлях на кшталт `media/inbound/<filename>`.
- Розуміння медіа (якщо налаштовано через `tools.media.*` або спільні `tools.media.models`) виконується до шаблонізації й може вставляти блоки `[Image]`, `[Audio]` і `[Video]` у `Body`.
  - Аудіо встановлює `{{Transcript}}` і використовує транскрипт для розбору команд, тому slash-команди продовжують працювати.
  - Описи відео й зображень зберігають будь-який текст підпису для розбору команд.
  - Якщо активна основна модель зображень уже нативно підтримує vision, OpenClaw пропускає блок підсумку `[Image]` і натомість передає моделі оригінальне зображення.
- За замовчуванням обробляється лише перше відповідне вкладення зображення/аудіо/відео; задайте `tools.media.<cap>.attachments`, щоб обробляти кілька вкладень.

## Обмеження та помилки

**Обмеження вихідного надсилання (надсилання через WhatsApp web)**

- Зображення: до `channels.whatsapp.mediaMaxMb` (за замовчуванням: 50 MB) після повторного стиснення.
- Аудіо/голосові повідомлення/відео: обмеження 16 MB; документи: 100 MB.
- Завеликі або непридатні для читання медіа → чітка помилка в журналах, і відповідь пропускається.

**Обмеження для розуміння медіа (транскрибування/опис)**

- Стандарт для зображень: 10 MB (`tools.media.image.maxBytes`).
- Стандарт для аудіо: 20 MB (`tools.media.audio.maxBytes`).
- Стандарт для відео: 50 MB (`tools.media.video.maxBytes`).
- Завеликі медіа пропускають етап розуміння, але відповіді все одно проходять з оригінальним `Body`.

## Примітки для тестів

- Покривайте потоки надсилання + відповіді для випадків із зображеннями/аудіо/документами.
- Перевіряйте повторне стиснення для зображень (обмеження розміру) і прапорець голосового повідомлення для аудіо.
- Переконайтеся, що відповіді з кількома медіа розгортаються в послідовні надсилання.
