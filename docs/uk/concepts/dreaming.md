# Dreaming (експериментально)

Dreaming — це система фонового консолідування пам’яті в `memory-core`.
Вона допомагає OpenClaw переносити сильні короткострокові сигнали в довготривалу пам’ять, водночас
зберігаючи процес зрозумілим і придатним для перевірки.

Dreaming є **опційною функцією** і за замовчуванням вимкнена.

## Що записує dreaming

Dreaming зберігає два типи виводу:

- **Машинний стан** у `memory/.dreams/` (сховище recall, сигнали фаз, контрольні точки ingestion, блокування).
- **Людинозрозумілий вивід** у `DREAMS.md` (або наявному `dreams.md`) і необов’язкових файлах звітів фаз у `memory/dreaming/<phase>/YYYY-MM-DD.md`.

Просування в довготривалу пам’ять, як і раніше, записується лише в `MEMORY.md`.

## Модель фаз

Dreaming використовує три взаємодоповнювальні фази:

| Фаза | Призначення | Постійний запис |
| ----- | ----------- | --------------- |
| Light | Сортує та готує нещодавній короткостроковий матеріал | Ні |
| Deep  | Оцінює та просуває довготривалі кандидати | Так (`MEMORY.md`) |
| REM   | Осмислює теми та повторювані ідеї | Ні |

Ці фази є внутрішніми деталями реалізації, а не окремими
користувацькими "режимами".

### Фаза light

Фаза light поглинає нещодавні щоденні сигнали пам’яті та сліди recall, усуває дублікати
й готує рядки-кандидати.

- Читає з короткострокового стану recall, нещодавніх щоденних файлів пам’яті та відредагованих транскриптів сесій, якщо вони доступні.
- Записує керований блок `## Light Sleep`, якщо сховище містить вбудований вивід.
- Фіксує сигнали підкріплення для подальшого ранжування у фазі deep.
- Ніколи не записує в `MEMORY.md`.

### Фаза deep

Фаза deep визначає, що стане довготривалою пам’яттю.

- Ранжує кандидатів за допомогою зваженого оцінювання та порогових умов.
- Вимагає проходження `minScore`, `minRecallCount` і `minUniqueQueries`.
- Перед записом повторно завантажує фрагменти з актуальних щоденних файлів, тому застарілі або видалені фрагменти пропускаються.
- Додає просунуті записи до `MEMORY.md`.
- Записує підсумок `## Deep Sleep` у `DREAMS.md` і за бажанням записує `memory/dreaming/deep/YYYY-MM-DD.md`.

### Фаза REM

Фаза REM витягує закономірності та рефлексивні сигнали.

- Формує підсумки тем і роздумів на основі нещодавніх короткострокових слідів.
- Записує керований блок `## REM Sleep`, якщо сховище містить вбудований вивід.
- Фіксує сигнали підкріплення REM, що використовуються під час ранжування у фазі deep.
- Ніколи не записує в `MEMORY.md`.

## Ingestion транскриптів сесій

Dreaming може поглинати відредаговані транскрипти сесій до корпусу dreaming. Коли
транскрипти доступні, вони подаються до фази light разом із щоденними
сигналами пам’яті та слідами recall. Особистий і чутливий вміст редагується
перед ingestion.

## Dream Diary

Dreaming також веде наративний **Dream Diary** у `DREAMS.md`.
Після того як у кожної фази накопичується достатньо матеріалу, `memory-core` виконує фоновий
хід субагента в режимі best-effort (використовуючи модель runtime за замовчуванням) і додає короткий запис щоденника.

Цей щоденник призначений для читання людьми в інтерфейсі Dreams, а не є джерелом просування.

Також є grounded-режим історичного backfill для перевірки та відновлення:

- `memory rem-harness --path ... --grounded` попередньо переглядає grounded-вивід щоденника з історичних нотаток `YYYY-MM-DD.md`.
- `memory rem-backfill --path ...` записує оборотні grounded-записи щоденника до `DREAMS.md`.
- `memory rem-backfill --path ... --stage-short-term` готує grounded-кандидатів на довготривале зберігання в тому самому короткостроковому сховищі доказів, яке вже використовує звичайна фаза deep.
- `memory rem-backfill --rollback` і `--rollback-short-term` видаляють ці підготовлені артефакти backfill, не зачіпаючи звичайні записи щоденника чи живий короткостроковий recall.

Control UI надає той самий процес backfill/reset для щоденника, щоб ви могли переглянути результати в сцені Dreams, перш ніж вирішити, чи заслуговують grounded-кандидати на просування.

## Сигнали ранжування у фазі deep

Ранжування у фазі deep використовує шість зважених базових сигналів плюс підкріплення фаз:

| Сигнал | Вага | Опис |
| ------ | ---- | ---- |
| Частота | 0.24 | Скільки короткострокових сигналів накопичив запис |
| Релевантність | 0.30 | Середня якість отримання запису |
| Різноманітність запитів | 0.15 | Окремі контексти запиту/дня, у яких він з’являвся |
| Актуальність | 0.15 | Оцінка свіжості з часовим згасанням |
| Консолідування | 0.10 | Сила повторюваності протягом кількох днів |
| Концептуальна насиченість | 0.06 | Щільність концептуальних тегів із фрагмента/шляху |

Спрацьовування фаз light і REM додають невелике згасаюче з часом підсилення з
`memory/.dreams/phase-signals.json`.

## Планування

Коли функцію ввімкнено, `memory-core` автоматично керує одним cron-завданням для повного циклу dreaming.
Кожен цикл виконує фази в такому порядку: light -> REM -> deep.

Стандартна поведінка частоти:

| Налаштування | Значення за замовчуванням |
| ------------ | ------------------------- |
| `dreaming.frequency` | `0 3 * * *` |

## Швидкий старт

Увімкніть dreaming:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

Увімкніть dreaming із власною частотою циклів:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

```
/dreaming status

/dreaming off
/dreaming help
```

## Робочий процес CLI

Використовуйте просування через CLI для попереднього перегляду або ручного застосування:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

Ручний `memory promote` за замовчуванням використовує пороги фази deep, якщо їх не перевизначено
прапорцями CLI.

Пояснити, чому конкретний кандидат буде або не буде просунутий:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

Попередньо переглянути REM-рефлексії, кандидатні істини та вивід просування у фазі deep без
будь-якого запису:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## Ключові значення за замовчуванням

Усі налаштування знаходяться в `plugins.entries.memory-core.config.dreaming`.

| Ключ | Значення за замовчуванням |
| ---- | ------------------------- |
| `enabled` | `false` |
| `frequency` | `0 3 * * *` |

Політика фаз, пороги та поведінка сховища є внутрішніми деталями реалізації
(не користувацькою конфігурацією).

Повний список ключів див. у [довіднику з конфігурації Memory](/uk/reference/memory-config#dreaming-experimental).

## Інтерфейс Dreams

Коли функцію ввімкнено, вкладка Gateway **Dreams** показує:

- поточний стан увімкнення dreaming
- статус на рівні фаз і наявність керованого циклу
- кількість короткострокових, довготривалих і просунутих сьогодні записів
- час наступного запланованого запуску
- розгортаний читач Dream Diary на основі `doctor.memory.dreamDiary`
