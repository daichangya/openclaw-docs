# Політика повторних спроб

## Цілі

- Повторювати спробу для кожного HTTP-запиту, а не для багатокрокового потоку.
- Зберігати порядок, повторюючи спробу лише для поточного кроку.
- Уникати дублювання неідемпотентних операцій.

## Стандартні параметри

- Кількість спроб: 3
- Максимальна межа затримки: 30000 мс
- Джитер: 0.1 (10 відсотків)
- Стандартні параметри провайдерів:
  - Мінімальна затримка Telegram: 400 мс
  - Мінімальна затримка Discord: 500 мс

## Поведінка

### Провайдери моделей

- OpenClaw дозволяє SDK провайдерів обробляти звичайні короткі повторні спроби.
- Для SDK на базі Stainless, таких як Anthropic і OpenAI, відповіді, для яких дозволені повторні спроби (`408`, `409`, `429` і `5xx`), можуть містити `retry-after-ms` або `retry-after`. Якщо це очікування довше за 60 секунд, OpenClaw додає `x-should-retry: false`, щоб SDK негайно повернув помилку, а механізм перемикання моделей міг перейти на інший профіль автентифікації або резервну модель.
- Перевизначте межу за допомогою `OPENCLAW_SDK_RETRY_MAX_WAIT_SECONDS=<seconds>`.
  Установіть значення `0`, `false`, `off`, `none` або `disabled`, щоб дозволити SDK внутрішньо дотримуватися тривалих затримок `Retry-After`.

### Discord

- Повторює спроби лише для помилок обмеження частоти запитів (HTTP 429).
- Використовує `retry_after` Discord, якщо доступно, інакше — експоненційний backoff.

### Telegram

- Повторює спроби для тимчасових помилок (429, timeout, connect/reset/closed, temporarily unavailable).
- Використовує `retry_after`, якщо доступно, інакше — експоненційний backoff.
- Помилки розбору Markdown не повторюються; натомість використовується звичайний текст.

## Конфігурація

Установіть політику повторних спроб для кожного провайдера в `~/.openclaw/openclaw.json`:

```json5
{
  channels: {
    telegram: {
      retry: {
        attempts: 3,
        minDelayMs: 400,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
    discord: {
      retry: {
        attempts: 3,
        minDelayMs: 500,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
  },
}
```

## Примітки

- Повторні спроби застосовуються до кожного запиту окремо (надсилання повідомлення, завантаження медіа, реакція, опитування, стікер).
- Для складених потоків завершені кроки не повторюються.
