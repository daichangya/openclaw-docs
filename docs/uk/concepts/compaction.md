Кожна модель має вікно контексту — максимальну кількість токенів, яку вона може обробити.
Коли розмова наближається до цього ліміту, OpenClaw **виконує Compaction** старіших повідомлень
у підсумок, щоб чат міг продовжуватися.

## Як це працює

1. Старіші ходи розмови підсумовуються в компактний запис.
2. Підсумок зберігається в транскрипті сесії.
3. Недавні повідомлення зберігаються без змін.

Коли OpenClaw розбиває історію на фрагменти Compaction, він зберігає виклики
інструментів асистента в парі з відповідними записами `toolResult`. Якщо точка
розбиття потрапляє всередину блоку інструмента, OpenClaw зсуває межу, щоб пара
залишалася разом і поточний непідсумований хвіст було збережено.

Повна історія розмови залишається на диску. Compaction змінює лише те, що
модель бачить на наступному ході.

## Auto-compaction

Auto-compaction увімкнено за замовчуванням. Воно запускається, коли сесія
наближається до ліміту контексту, або коли модель повертає помилку переповнення
контексту (у такому разі OpenClaw виконує Compaction і повторює спробу). Типові
ознаки переповнення включають
`request_too_large`, `context length exceeded`, `input exceeds the maximum
number of tokens`, `input token count exceeds the maximum number of input
tokens`, `input is too long for the model` і `ollama error: context length
exceeded`.

<Info>
Перед виконанням Compaction OpenClaw автоматично нагадує агенту зберегти важливі
нотатки у файли [memory](/uk/concepts/memory). Це запобігає втраті контексту.
</Info>

Використовуйте налаштування `agents.defaults.compaction` у вашому `openclaw.json`, щоб налаштувати поведінку Compaction (режим, цільову кількість токенів тощо).
Підсумовування Compaction за замовчуванням зберігає непрозорі ідентифікатори (`identifierPolicy: "strict"`). Ви можете перевизначити це через `identifierPolicy: "off"` або надати власний текст за допомогою `identifierPolicy: "custom"` і `identifierInstructions`.

За бажанням ви можете вказати іншу модель для підсумовування Compaction через `agents.defaults.compaction.model`. Це корисно, коли ваша основна модель є локальною або невеликою моделлю, а ви хочете, щоб підсумки Compaction створювалися потужнішою моделлю. Перевизначення приймає будь-який рядок `provider/model-id`:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "openrouter/anthropic/claude-sonnet-4-6"
      }
    }
  }
}
```

Це також працює з локальними моделями, наприклад із другою моделлю Ollama, виділеною для підсумовування, або тонко налаштованим спеціалістом із Compaction:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "ollama/llama3.1:8b"
      }
    }
  }
}
```

Якщо значення не задано, Compaction використовує основну модель агента.

## Підключувані провайдери Compaction

Plugins можуть реєструвати власний провайдер Compaction через `registerCompactionProvider()` в API плагіна. Коли провайдер зареєстрований і налаштований, OpenClaw делегує йому підсумовування замість вбудованого LLM-конвеєра.

Щоб використати зареєстрований провайдер, установіть ідентифікатор провайдера у вашій конфігурації:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "provider": "my-provider"
      }
    }
  }
}
```

Установлення `provider` автоматично примусово задає `mode: "safeguard"`. Провайдери отримують ті самі інструкції Compaction і політику збереження ідентифікаторів, що й вбудований шлях, а OpenClaw усе одно зберігає контекст суфікса недавніх ходів і розділених ходів після виводу провайдера. Якщо провайдер зазнає невдачі або повертає порожній результат, OpenClaw повертається до вбудованого підсумовування LLM.

## Auto-compaction (увімкнено за замовчуванням)

Коли сесія наближається до вікна контексту моделі або перевищує його, OpenClaw запускає auto-compaction і може повторити початковий запит, використовуючи стиснений контекст.

Ви побачите:

- `🧹 Auto-compaction complete` у докладному режимі
- `/status`, що показує `🧹 Compactions: <count>`

Перед Compaction OpenClaw може виконати **тихий скид memory**, щоб зберегти
довготривалі нотатки на диск. Докладніше про це та налаштування див. у [Memory](/uk/concepts/memory).

## Ручний Compaction

Введіть `/compact` у будь-якому чаті, щоб примусово виконати Compaction. Додайте інструкції, щоб спрямувати
підсумок:

```
/compact Focus on the API design decisions
```

Коли встановлено `agents.defaults.compaction.keepRecentTokens`, ручний Compaction
враховує цю точку відсікання Pi і зберігає недавній хвіст у відновленому контексті. Без
явно заданого бюджету збереження ручний Compaction поводиться як жорстка контрольна точка і
продовжується лише з нового підсумку.

## Використання іншої моделі

За замовчуванням Compaction використовує основну модель вашого агента. Ви можете використати потужнішу
модель для кращих підсумків:

```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6",
      },
    },
  },
}
```

## Сповіщення Compaction

За замовчуванням Compaction виконується безшумно. Щоб показувати короткі сповіщення, коли Compaction
починається і коли завершується, увімкніть `notifyUser`:

```json5
{
  agents: {
    defaults: {
      compaction: {
        notifyUser: true,
      },
    },
  },
}
```

Коли це увімкнено, користувач бачить короткі повідомлення про стан під час кожного запуску Compaction
(наприклад, "Compacting context..." і "Compaction complete").

## Compaction і обрізання

|                  | Compaction                    | Обрізання                        |
| ---------------- | ----------------------------- | -------------------------------- |
| **Що робить**    | Підсумовує старішу розмову    | Обрізає старі результати інструментів |
| **Зберігається?**| Так (у транскрипті сесії)     | Ні (лише в пам’яті, для кожного запиту) |
| **Обсяг**        | Уся розмова                   | Лише результати інструментів     |

[Обрізання сесії](/uk/concepts/session-pruning) — це легше доповнення, яке
обрізає вивід інструментів без підсумовування.

## Усунення проблем

**Compaction виконується надто часто?** Вікно контексту моделі може бути малим, або
виводи інструментів можуть бути великими. Спробуйте ввімкнути
[обрізання сесії](/uk/concepts/session-pruning).

**Після Compaction контекст здається застарілим?** Використайте `/compact Focus on <topic>`, щоб
спрямувати підсумок, або ввімкніть [скид memory](/uk/concepts/memory), щоб нотатки
зберігалися.

**Потрібно почати з чистого аркуша?** `/new` запускає нову сесію без виконання Compaction.

Для розширеного налаштування (резерв токенів, збереження ідентифікаторів, власні
рушії контексту, серверний Compaction OpenAI) див.
[Поглиблений огляд керування сесіями](/uk/reference/session-management-compaction).
