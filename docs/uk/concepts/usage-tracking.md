## Що це таке

- Отримує використання/квоту провайдера безпосередньо з їхніх endpoint використання.
- Жодних оцінених витрат; лише вікна, повідомлені самим провайдером.
- Зрозумілий для людини вивід стану нормалізується до `X% left`, навіть якщо
  API вище за потоком повідомляє про використану квоту, залишкову квоту або лише сирі лічильники.
- `/status` на рівні сесії та `session_status` можуть використовувати
  останній запис використання з транскрипту як запасний варіант, коли live snapshot сесії є неповним. Такий
  запасний варіант заповнює відсутні лічильники токенів/кешу, може відновити мітку
  активної моделі runtime і надає перевагу більшому загальному значенню, орієнтованому на prompt, коли метадані сесії
  відсутні або менші. Наявні ненульові live-значення все одно мають пріоритет.

## Де це відображається

- `/status` у чатах: насичена emoji картка стану з токенами сесії та оціненою вартістю (лише для API key). Використання провайдера показується для **поточного провайдера моделі**, коли доступне, як нормалізоване вікно `X% left`.
- `/usage off|tokens|full` у чатах: нижній колонтитул використання для кожної відповіді (OAuth показує лише токени).
- `/usage cost` у чатах: локальний підсумок вартості, агрегований із журналів сесій OpenClaw.
- CLI: `openclaw status --usage` виводить повну розбивку за провайдерами.
- CLI: `openclaw channels list` виводить той самий snapshot використання поруч із config провайдера (використайте `--no-usage`, щоб пропустити).
- Рядок меню macOS: розділ “Usage” у Context (лише якщо доступно).

## Провайдери й облікові дані

- **Anthropic (Claude)**: OAuth tokens в auth profiles.
- **GitHub Copilot**: OAuth tokens в auth profiles.
- **Gemini CLI**: OAuth tokens в auth profiles.
  - JSON використання як запасний варіант використовує `stats`; `stats.cached` нормалізується до
    `cacheRead`.
- **OpenAI Codex**: OAuth tokens в auth profiles (`accountId` використовується, якщо присутній).
- **MiniMax**: API key або auth profile MiniMax OAuth. OpenClaw розглядає
  `minimax`, `minimax-cn` і `minimax-portal` як одну й ту саму поверхню квот MiniMax,
  надає перевагу збереженому MiniMax OAuth, якщо він є, а інакше використовує
  `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY` або `MINIMAX_API_KEY`.
  Сирі поля MiniMax `usage_percent` / `usagePercent` означають **залишкову**
  квоту, тому OpenClaw інвертує їх перед показом; поля на основі лічильників мають пріоритет, якщо присутні.
  - Мітки вікон coding-plan беруться з полів годин/хвилин провайдера, якщо вони є,
    а потім як запасний варіант — з діапазону `start_time` / `end_time`.
  - Якщо endpoint coding-plan повертає `model_remains`, OpenClaw надає перевагу
    запису chat-model, виводить мітку вікна з часових міток, коли явні
    поля `window_hours` / `window_minutes` відсутні, і включає назву моделі
    до мітки плану.
- **Xiaomi MiMo**: API key через env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: API key через env/config/auth store.

Використання приховується, коли не вдається визначити придатну автентифікацію використання провайдера. Провайдери
можуть надавати логіку автентифікації використання, специфічну для Plugin; інакше OpenClaw використовує
як запасний варіант відповідні облікові дані OAuth/API-key з auth profiles, змінних середовища
або config.
