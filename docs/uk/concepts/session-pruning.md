Очищення сесії обрізає **старі результати інструментів** із контексту перед кожним
викликом LLM. Це зменшує розростання контексту через накопичені виводи інструментів (результати exec, читання
файлів, результати пошуку), не переписуючи звичайний текст розмови.

<Info>
Очищення виконується лише в пам’яті — воно не змінює транскрипт сесії на диску.
Уся ваша історія завжди зберігається повністю.
</Info>

## Чому це важливо

У довгих сесіях накопичується вивід інструментів, який роздуває контекстне вікно. Це
збільшує вартість і може змусити [Compaction](/uk/concepts/compaction) спрацювати раніше, ніж
потрібно.

Очищення особливо цінне для **кешування prompt-ів Anthropic**. Після завершення
TTL кешу наступний запит повторно кешує весь prompt. Очищення зменшує розмір
запису в кеш, що безпосередньо знижує вартість.

## Як це працює

1. Дочекатися завершення TTL кешу (типово 5 хвилин).
2. Знайти старі результати інструментів для звичайного очищення (текст розмови не змінюється).
3. **М’яко обрізати** надто великі результати — зберегти початок і кінець, вставити `...`.
4. **Жорстко очистити** решту — замінити placeholder-ом.
5. Скинути TTL, щоб наступні запити повторно використовували свіжий кеш.

## Очищення legacy-зображень

OpenClaw також виконує окреме ідемпотентне очищення для старих legacy-сесій, які
зберігали сирі блоки зображень в історії.

- Воно зберігає **3 найновіші завершені цикли** байт у байт, щоб префікси prompt cache
  для недавніх наступних запитів залишалися стабільними.
- Старі вже оброблені блоки зображень в історії `user` або `toolResult` можуть бути
  замінені на `[image data removed - already processed by model]`.
- Це окремо від звичайного очищення за TTL кешу. Воно існує, щоб повторні
  payload-и зображень не ламали prompt cache у пізніших циклах.

## Розумні типові значення

OpenClaw автоматично вмикає очищення для профілів Anthropic:

| Тип профілю                                            | Очищення ввімкнено | Heartbeat |
| ------------------------------------------------------ | ------------------ | --------- |
| OAuth/автентифікація токеном Anthropic (включно з повторним використанням Claude CLI) | Так                | 1 година  |
| API-ключ                                               | Так                | 30 хв     |

Якщо ви задаєте явні значення, OpenClaw не перевизначає їх.

## Увімкнення або вимкнення

Для провайдерів не Anthropic очищення типово вимкнено. Щоб увімкнути:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

Щоб вимкнути: задайте `mode: "off"`.

## Очищення vs Compaction

|            | Очищення            | Compaction              |
| ---------- | ------------------- | ----------------------- |
| **Що**     | Обрізає результати інструментів | Підсумовує розмову      |
| **Зберігається?** | Ні (для кожного запиту) | Так (у транскрипті)     |
| **Обсяг**  | Лише результати інструментів | Уся розмова             |

Вони доповнюють одне одного — очищення підтримує компактність виводу інструментів між
циклами Compaction.
