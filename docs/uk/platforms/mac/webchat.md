Рядковий застосунок macOS вбудовує інтерфейс WebChat як нативне подання SwiftUI. Він
підключається до Gateway і типово використовує **основну сесію** для вибраного
агента (із перемикачем сесій для інших сесій).

- **Локальний режим**: підключається безпосередньо до локального WebSocket Gateway.
- **Віддалений режим**: переадресовує контрольний порт Gateway через SSH і використовує цей
  тунель як площину даних.

## Запуск і налагодження

- Вручну: меню Lobster → “Open Chat”.
- Автовідкриття для тестування:

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --webchat
  ```

- Логи: `./scripts/clawlog.sh` (підсистема `ai.openclaw`, категорія `WebChatSwiftUI`).

## Як це підключено

- Площина даних: методи WS Gateway `chat.history`, `chat.send`, `chat.abort`,
  `chat.inject` і події `chat`, `agent`, `presence`, `tick`, `health`.
- `chat.history` повертає рядки транскрипту, нормалізовані для відображення: вбудовані
  теги директив прибираються з видимого тексту, XML-корисні навантаження викликів інструментів у звичайному тексті
  (зокрема `<tool_call>...</tool_call>`,
  `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>`,
  `<function_calls>...</function_calls>` і усічені блоки викликів інструментів), а також
  витеклі ASCII/повноширинні токени керування моделлю прибираються, рядки помічника, що складаються лише
  з тихих токенів, як-от точне `NO_REPLY` / `no_reply`, пропускаються,
  а надто великі рядки можуть бути замінені заповнювачами.
- Сесія: типово використовується основна сесія (`main` або `global`, коли область дії
  глобальна). UI може перемикатися між сесіями.
- Онбординг використовує окрему сесію, щоб відокремити початкове налаштування.

## Поверхня безпеки

- Віддалений режим переадресовує через SSH лише контрольний порт WebSocket Gateway.

## Відомі обмеження

- UI оптимізовано для чат-сесій (а не для повноцінного sandbox браузера).
