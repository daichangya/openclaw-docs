Надання дозволів у macOS є крихким. TCC пов’язує наданий дозвіл із
підписом коду застосунку, ідентифікатором bundle і шляхом на диску. Якщо будь-що з цього змінюється,
macOS розглядає застосунок як новий і може скинути або приховати запити на дозволи.

## Вимоги для стабільного збереження дозволів

- Той самий шлях: запускайте застосунок із фіксованого розташування (для OpenClaw це `dist/OpenClaw.app`).
- Той самий ідентифікатор bundle: зміна bundle ID створює нову ідентичність дозволів.
- Підписаний застосунок: непідписані або ad-hoc signed збірки не зберігають дозволи.
- Узгоджений підпис: використовуйте справжній сертифікат Apple Development або Developer ID,
  щоб підпис залишався стабільним між перебудовами.

Ad-hoc підписи генерують нову ідентичність на кожне збирання. macOS забуватиме попередньо
надані дозволи, а запити можуть узагалі зникати, доки не буде очищено застарілі записи.

## Контрольний список відновлення, коли запити зникають

1. Завершіть застосунок.
2. Видаліть запис застосунку в System Settings -> Privacy & Security.
3. Знову запустіть застосунок із того самого шляху й повторно надайте дозволи.
4. Якщо запит усе ще не з’являється, скиньте записи TCC через `tccutil` і спробуйте знову.
5. Деякі дозволи повторно з’являються лише після повного перезапуску macOS.

Приклади скидання (за потреби замініть bundle ID):

```bash
sudo tccutil reset Accessibility ai.openclaw.mac
sudo tccutil reset ScreenCapture ai.openclaw.mac
sudo tccutil reset AppleEvents
```

## Дозволи на файли та папки (Desktop/Documents/Downloads)

macOS також може обмежувати доступ до Desktop, Documents і Downloads для процесів terminal/background. Якщо читання файлів або перелік каталогів зависає, надайте доступ тому самому контексту процесу, який виконує файлові операції (наприклад, Terminal/iTerm, застосунок, запущений через LaunchAgent, або процес SSH).

Обхідний шлях: перемістіть файли до робочого простору OpenClaw (`~/.openclaw/workspace`), якщо ви хочете уникнути надання дозволів для кожної папки окремо.

Якщо ви тестуєте дозволи, завжди підписуйте застосунок справжнім сертифікатом. Ad-hoc
збірки прийнятні лише для швидких локальних запусків, де дозволи не мають значення.
