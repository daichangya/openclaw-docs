Implante o OpenClaw na Northflank com um template de um clique e acesse-o pela UI web de controle.
Este é o caminho mais fácil de “sem terminal no servidor”: a Northflank executa o Gateway para você.

## Como começar

1. Clique em [Deploy OpenClaw](https://northflank.com/stacks/deploy-openclaw) para abrir o template.
2. Crie uma [conta na Northflank](https://app.northflank.com/signup) se ainda não tiver uma.
3. Clique em **Deploy OpenClaw now**.
4. Defina a variável de ambiente obrigatória: `OPENCLAW_GATEWAY_TOKEN` (use um valor aleatório forte).
5. Clique em **Deploy stack** para fazer build e executar o template do OpenClaw.
6. Aguarde a implantação ser concluída e clique em **View resources**.
7. Abra o serviço OpenClaw.
8. Abra a URL pública do OpenClaw em `/openclaw` e conecte-se usando o segredo compartilhado configurado. Este template usa `OPENCLAW_GATEWAY_TOKEN` por padrão; se você substituí-lo por autenticação por senha, use essa senha em vez disso.

## O que você recebe

- Gateway OpenClaw hospedado + UI de controle
- Armazenamento persistente por meio do Northflank Volume (`/data`), para que `openclaw.json`,
  `auth-profiles.json` por agente, estado de canal/provedor, sessões e
  workspace sobrevivam a reimplantações

## Conectar um canal

Use a UI de controle em `/openclaw` ou execute `openclaw onboard` via SSH para obter instruções de configuração de canal:
