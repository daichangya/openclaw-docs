# بيان Plugin ‏(`openclaw.plugin.json`)

هذه الصفحة مخصّصة لـ **بيان Plugin الأصلي في OpenClaw** فقط.

لتنسيقات الحِزم المتوافقة، راجع [حِزم Plugin](/ar/plugins/bundles).

تستخدم تنسيقات الحِزم المتوافقة ملفات بيان مختلفة:

- حزمة Codex: `.codex-plugin/plugin.json`
- حزمة Claude: `.claude-plugin/plugin.json` أو تخطيط مكوّن Claude الافتراضي
  بدون بيان
- حزمة Cursor: `.cursor-plugin/plugin.json`

يكتشف OpenClaw تلك التخطيطات المتوافقة للحِزم تلقائيًا أيضًا، لكنها لا تُتحقق
مقابل مخطط `openclaw.plugin.json` الموصوف هنا.

بالنسبة إلى الحِزم المتوافقة، يقرأ OpenClaw حاليًا بيانات الحزمة الوصفية إضافةً إلى
جذور المهارات المعلنة، وجذور أوامر Claude، والقيم الافتراضية لـ `settings.json` في حزمة Claude،
وقيم LSP الافتراضية في حزمة Claude، وحِزم الخطافات المدعومة عندما يطابق التخطيط
توقعات وقت تشغيل OpenClaw.

يجب على كل Plugin أصلي في OpenClaw **أن يضم** ملف `openclaw.plugin.json` في
**جذر Plugin**. يستخدم OpenClaw هذا البيان للتحقق من صحة الإعدادات
**من دون تنفيذ كود Plugin**. وتُعامل البيانات المفقودة أو غير الصالحة على أنها
أخطاء Plugin وتمنع التحقق من صحة الإعدادات.

راجع دليل نظام Plugin الكامل: [Plugins](/ar/tools/plugin).
ولنموذج القدرات الأصلي وإرشادات التوافق الخارجي الحالية:
[نموذج القدرات](/ar/plugins/architecture#public-capability-model).

## ما الذي يفعله هذا الملف

`openclaw.plugin.json` هو البيانات الوصفية التي يقرأها OpenClaw قبل أن يحمّل
كود Plugin الخاص بك.

استخدمه من أجل:

- هوية Plugin
- التحقق من صحة الإعدادات
- بيانات المصادقة والإعداد الأوّلي التي ينبغي أن تكون متاحة دون تشغيل وقت تنفيذ Plugin
- تلميحات تنشيط منخفضة الكلفة يمكن لواجهات مستوى التحكّم فحصها قبل تحميل وقت التنفيذ
- واصفات إعداد منخفضة الكلفة يمكن لواجهات الإعداد/التهيئة الأوّلية فحصها قبل
  تحميل وقت التنفيذ
- بيانات الأسماء المستعارة والتفعيل التلقائي التي ينبغي حلّها قبل تحميل وقت تنفيذ Plugin
- بيانات مختصرة عن ملكية عائلات النماذج ينبغي أن تفعّل Plugin تلقائيًا قبل تحميل وقت التنفيذ
- لقطات ثابتة لملكية القدرات تُستخدم في توصيل التوافق المضمّن وتغطية العقود
- بيانات وصفية منخفضة الكلفة لمشغّل QA يمكن للمضيف المشترك `openclaw qa` فحصها
  قبل تحميل وقت تنفيذ Plugin
- بيانات إعدادات خاصة بالقنوات ينبغي دمجها في أسطح الفهرسة والتحقق من الصحة
  من دون تحميل وقت التنفيذ
- تلميحات واجهة الإعدادات

لا تستخدمه من أجل:

- تسجيل سلوك وقت التنفيذ
- التصريح بنقاط دخول الكود
- بيانات تثبيت npm الوصفية

فهذه تنتمي إلى كود Plugin الخاص بك و`package.json`.

## مثال أدنى

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

## مرجع الحقول ذات المستوى الأعلى

| الحقل                               | مطلوب | النوع                            | ما الذي يعنيه                                                                                                                                                                                                |
| ----------------------------------- | ------ | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `id`                                | نعم    | `string`                         | المعرّف القياسي لـ Plugin. هذا هو المعرّف المستخدم في `plugins.entries.<id>`.                                                                                                                               |
| `configSchema`                      | نعم    | `object`                         | مخطط JSON مضمّن لإعدادات Plugin هذا.                                                                                                                                                                        |
| `enabledByDefault`                  | لا     | `true`                           | يحدّد أن Plugin المضمّن مفعّل افتراضيًا. احذفه، أو عيّن أي قيمة غير `true`، لترك Plugin معطّلًا افتراضيًا.                                                                                                 |
| `legacyPluginIds`                   | لا     | `string[]`                       | معرّفات قديمة تُسوّى إلى معرّف Plugin القياسي هذا.                                                                                                                                                          |
| `autoEnableWhenConfiguredProviders` | لا     | `string[]`                       | معرّفات الموفّرين التي ينبغي أن تفعّل هذا Plugin تلقائيًا عندما تشير إليها المصادقة أو الإعدادات أو مراجع النماذج.                                                                                          |
| `kind`                              | لا     | `"memory"` \| `"context-engine"` | يصرّح بنوع Plugin حصري يُستخدم بواسطة `plugins.slots.*`.                                                                                                                                                   |
| `channels`                          | لا     | `string[]`                       | معرّفات القنوات التي يملكها هذا Plugin. تُستخدم للاكتشاف والتحقق من صحة الإعدادات.                                                                                                                         |
| `providers`                         | لا     | `string[]`                       | معرّفات الموفّرين التي يملكها هذا Plugin.                                                                                                                                                                   |
| `modelSupport`                      | لا     | `object`                         | بيانات وصفية مختصرة لعائلة النماذج مملوكة للبيان وتُستخدم لتحميل Plugin تلقائيًا قبل وقت التنفيذ.                                                                                                          |
| `providerEndpoints`                 | لا     | `object[]`                       | بيانات وصفية للمضيف/‏`baseUrl` لنقاط نهاية الموفّر مملوكة للبيان للمسارات التي يجب على النواة تصنيفها قبل تحميل وقت تنفيذ الموفّر.                                                                       |
| `cliBackends`                       | لا     | `string[]`                       | معرّفات الخلفيات الاستدلالية لـ CLI التي يملكها هذا Plugin. تُستخدم للتفعيل التلقائي عند بدء التشغيل من مراجع الإعدادات الصريحة.                                                                          |
| `syntheticAuthRefs`                 | لا     | `string[]`                       | مراجع الموفّرين أو خلفيات CLI التي ينبغي فحص خطاف المصادقة الاصطناعية المملوك لـ Plugin لها أثناء الاكتشاف البارد للنماذج قبل تحميل وقت التنفيذ.                                                          |
| `nonSecretAuthMarkers`              | لا     | `string[]`                       | قيم مفاتيح API النائبة المملوكة لـ Plugin المضمّن التي تمثل حالة بيانات اعتماد محلية أو OAuth أو محيطة غير سرية.                                                                                          |
| `commandAliases`                    | لا     | `object[]`                       | أسماء الأوامر التي يملكها هذا Plugin والتي ينبغي أن تنتج إعدادات وتشخيصات CLI واعية بـ Plugin قبل تحميل وقت التنفيذ.                                                                                        |
| `providerAuthEnvVars`               | لا     | `Record<string, string[]>`       | بيانات وصفية منخفضة الكلفة لمتغيرات بيئة مصادقة الموفّر يمكن لـ OpenClaw فحصها دون تحميل كود Plugin.                                                                                                      |
| `providerAuthAliases`               | لا     | `Record<string, string>`         | معرّفات الموفّرين التي ينبغي أن تعيد استخدام معرّف موفّر آخر لبحث المصادقة، مثل موفّر برمجي يشارك مفتاح API الأساسي وملفات تعريف المصادقة مع الموفّر الأساسي.                                           |
| `channelEnvVars`                    | لا     | `Record<string, string[]>`       | بيانات وصفية منخفضة الكلفة لمتغيرات بيئة القنوات يمكن لـ OpenClaw فحصها دون تحميل كود Plugin. استخدم هذا لأسطح إعداد القنوات أو المصادقة المعتمدة على البيئة التي ينبغي أن تراها مساعدات البدء/الإعداد العامة. |
| `providerAuthChoices`               | لا     | `object[]`                       | بيانات وصفية منخفضة الكلفة لخيارات المصادقة لمنتقيات الإعداد الأوّلي، وحلّ الموفّر المفضّل، وربط أعلام CLI البسيط.                                                                                         |
| `activation`                        | لا     | `object`                         | تلميحات تفعيل منخفضة الكلفة للتحميل المُحفَّز بالموفّر أو الأمر أو القناة أو المسار أو القدرة. بيانات وصفية فقط؛ إذ يظل وقت تنفيذ Plugin مالكًا للسلوك الفعلي.                                           |
| `setup`                             | لا     | `object`                         | واصفات إعداد/تهيئة أولية منخفضة الكلفة يمكن لواجهات الاكتشاف والإعداد فحصها دون تحميل وقت تنفيذ Plugin.                                                                                                   |
| `qaRunners`                         | لا     | `object[]`                       | واصفات منخفضة الكلفة لمشغّلات QA يستخدمها المضيف المشترك `openclaw qa` قبل تحميل وقت تنفيذ Plugin.                                                                                                        |
| `contracts`                         | لا     | `object`                         | لقطة ثابتة للقدرات المضمّنة لملكية الكلام، والنسخ الفوري، والصوت الفوري، وفهم الوسائط، وتوليد الصور، وتوليد الموسيقى، وتوليد الفيديو، وجلب الويب، والبحث في الويب، وملكية الأدوات.                     |
| `channelConfigs`                    | لا     | `Record<string, object>`         | بيانات وصفية لإعدادات القنوات مملوكة للبيان وتُدمج في أسطح الاكتشاف والتحقق قبل تحميل وقت التنفيذ.                                                                                                        |
| `skills`                            | لا     | `string[]`                       | أدلة Skills المطلوب تحميلها، نسبةً إلى جذر Plugin.                                                                                                                                                          |
| `name`                              | لا     | `string`                         | اسم Plugin مقروء بشريًا.                                                                                                                                                                                     |
| `description`                       | لا     | `string`                         | ملخّص قصير يُعرض في أسطح Plugin.                                                                                                                                                                             |
| `version`                           | لا     | `string`                         | إصدار Plugin لأغراض معلوماتية.                                                                                                                                                                               |
| `uiHints`                           | لا     | `Record<string, object>`         | تسميات واجهة المستخدم، والعناصر النائبة، وتلميحات الحساسية لحقول الإعدادات.                                                                                                                                 |

## مرجع `providerAuthChoices`

يصف كل إدخال في `providerAuthChoices` خيارًا واحدًا للإعداد الأوّلي أو المصادقة.
يقرأ OpenClaw هذا قبل تحميل وقت تنفيذ الموفّر.

| الحقل                 | مطلوب | النوع                                           | ما الذي يعنيه                                                                                           |
| --------------------- | ------ | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `provider`            | نعم    | `string`                                        | معرّف الموفّر الذي ينتمي إليه هذا الخيار.                                                                |
| `method`              | نعم    | `string`                                        | معرّف طريقة المصادقة المطلوب التوجيه إليها.                                                              |
| `choiceId`            | نعم    | `string`                                        | معرّف ثابت لخيار المصادقة تستخدمه تدفقات الإعداد الأوّلي وCLI.                                           |
| `choiceLabel`         | لا     | `string`                                        | تسمية موجهة للمستخدم. إذا حُذفت، يعود OpenClaw إلى `choiceId`.                                          |
| `choiceHint`          | لا     | `string`                                        | نص مساعد قصير للمنتقي.                                                                                  |
| `assistantPriority`   | لا     | `number`                                        | تُرتَّب القيم الأقل أولًا في المنتقيات التفاعلية التي يقودها المساعد.                                   |
| `assistantVisibility` | لا     | `"visible"` \| `"manual-only"`                  | يُخفي الخيار من منتقيات المساعد مع السماح مع ذلك بالاختيار اليدوي عبر CLI.                              |
| `deprecatedChoiceIds` | لا     | `string[]`                                      | معرّفات قديمة لخيارات ينبغي أن تعيد توجيه المستخدمين إلى هذا الخيار البديل.                             |
| `groupId`             | لا     | `string`                                        | معرّف مجموعة اختياري لتجميع الخيارات المرتبطة.                                                           |
| `groupLabel`          | لا     | `string`                                        | تسمية موجهة للمستخدم لتلك المجموعة.                                                                      |
| `groupHint`           | لا     | `string`                                        | نص مساعد قصير للمجموعة.                                                                                  |
| `optionKey`           | لا     | `string`                                        | مفتاح خيار داخلي لتدفقات المصادقة البسيطة ذات العلم الواحد.                                              |
| `cliFlag`             | لا     | `string`                                        | اسم علم CLI، مثل `--openrouter-api-key`.                                                                 |
| `cliOption`           | لا     | `string`                                        | شكل خيار CLI الكامل، مثل `--openrouter-api-key <key>`.                                                   |
| `cliDescription`      | لا     | `string`                                        | الوصف المستخدم في مساعدة CLI.                                                                            |
| `onboardingScopes`    | لا     | `Array<"text-inference" \| "image-generation">` | أسطح الإعداد الأوّلي التي ينبغي أن يظهر فيها هذا الخيار. إذا حُذفت، تكون القيمة الافتراضية `["text-inference"]`. |

## مرجع `commandAliases`

استخدم `commandAliases` عندما يملك Plugin اسم أمر في وقت التنفيذ قد يضعه المستخدمون
بالخطأ في `plugins.allow` أو يحاولون تشغيله كأمر CLI جذري. يستخدم OpenClaw
هذه البيانات الوصفية للتشخيصات من دون استيراد كود وقت تنفيذ Plugin.

```json
{
  "commandAliases": [
    {
      "name": "dreaming",
      "kind": "runtime-slash",
      "cliCommand": "memory"
    }
  ]
}
```

| الحقل        | مطلوب | النوع             | ما الذي يعنيه                                                              |
| ------------ | ------ | ----------------- | -------------------------------------------------------------------------- |
| `name`       | نعم    | `string`          | اسم الأمر الذي ينتمي إلى هذا Plugin.                                       |
| `kind`       | لا     | `"runtime-slash"` | يحدّد أن الاسم المستعار أمر شرطة مائلة في الدردشة بدلًا من أمر CLI جذري.   |
| `cliCommand` | لا     | `string`          | أمر CLI جذري ذو صلة يُقترح لعمليات CLI، إذا كان موجودًا.                   |

## مرجع `activation`

استخدم `activation` عندما يستطيع Plugin أن يصرّح بكلفة منخفضة عن أحداث مستوى التحكّم
التي ينبغي أن تفعّله لاحقًا.

## مرجع `qaRunners`

استخدم `qaRunners` عندما يضيف Plugin مشغّل نقل واحدًا أو أكثر تحت الجذر المشترك
`openclaw qa`. أبقِ هذه البيانات الوصفية منخفضة الكلفة وثابتة؛ إذ يظل وقت تنفيذ Plugin
مالكًا لتسجيل CLI الفعلي عبر سطح `runtime-api.ts`
خفيف الوزن الذي يصدّر `qaRunnerCliRegistrations`.

```json
{
  "qaRunners": [
    {
      "commandName": "matrix",
      "description": "Run the Docker-backed Matrix live QA lane against a disposable homeserver"
    }
  ]
}
```

| الحقل         | مطلوب | النوع    | ما الذي يعنيه                                                      |
| ------------- | ------ | -------- | ------------------------------------------------------------------ |
| `commandName` | نعم    | `string` | الأمر الفرعي المُركّب تحت `openclaw qa`، مثل `matrix`.             |
| `description` | لا     | `string` | نص مساعدة احتياطي يُستخدم عندما يحتاج المضيف المشترك إلى أمر هيكلي. |

هذه الكتلة بيانات وصفية فقط. فهي لا تسجل سلوك وقت التنفيذ، ولا
تستبدل `register(...)` أو `setupEntry` أو نقاط دخول وقت التنفيذ/Plugin الأخرى.
يستخدمها المستهلكون الحاليون كتلميح تضييق قبل تحميل Plugin على نطاق أوسع، لذا فإن
غياب بيانات `activation` الوصفية لا يكلّف عادةً إلا الأداء؛ ويجب ألّا
يغيّر الصحة ما دامت بدائل ملكية البيان القديمة ما تزال موجودة.

```json
{
  "activation": {
    "onProviders": ["openai"],
    "onCommands": ["models"],
    "onChannels": ["web"],
    "onRoutes": ["gateway-webhook"],
    "onCapabilities": ["provider", "tool"]
  }
}
```

| الحقل            | مطلوب | النوع                                                | ما الذي يعنيه                                                   |
| ---------------- | ------ | ---------------------------------------------------- | --------------------------------------------------------------- |
| `onProviders`    | لا     | `string[]`                                           | معرّفات الموفّرين التي ينبغي أن تفعّل هذا Plugin عند طلبها.      |
| `onCommands`     | لا     | `string[]`                                           | معرّفات الأوامر التي ينبغي أن تفعّل هذا Plugin.                  |
| `onChannels`     | لا     | `string[]`                                           | معرّفات القنوات التي ينبغي أن تفعّل هذا Plugin.                  |
| `onRoutes`       | لا     | `string[]`                                           | أنواع المسارات التي ينبغي أن تفعّل هذا Plugin.                  |
| `onCapabilities` | لا     | `Array<"provider" \| "channel" \| "tool" \| "hook">` | تلميحات واسعة للقدرات تُستخدم في تخطيط التفعيل على مستوى التحكّم. |

المستهلكون الأحياء الحاليون:

- يعود تخطيط CLI المُحفَّز بالأوامر إلى `commandAliases[].cliCommand`
  أو `commandAliases[].name` القديمين
- يعود تخطيط الإعداد/القناة المُحفَّز بالقنوات إلى ملكية `channels[]`
  القديمة عندما تكون بيانات تفعيل القنوات الصريحة مفقودة
- يعود تخطيط الإعداد/وقت التنفيذ المُحفَّز بالموفّرين إلى ملكية
  `providers[]` القديمة وملكية `cliBackends[]` ذات المستوى الأعلى عندما تكون
  بيانات تفعيل الموفّر الصريحة مفقودة

## مرجع `setup`

استخدم `setup` عندما تحتاج أسطح الإعداد والتهيئة الأوّلية إلى بيانات وصفية منخفضة الكلفة
مملوكة لـ Plugin قبل تحميل وقت التنفيذ.

```json
{
  "setup": {
    "providers": [
      {
        "id": "openai",
        "authMethods": ["api-key"],
        "envVars": ["OPENAI_API_KEY"]
      }
    ],
    "cliBackends": ["openai-cli"],
    "configMigrations": ["legacy-openai-auth"],
    "requiresRuntime": false
  }
}
```

يبقى `cliBackends` ذو المستوى الأعلى صالحًا ويواصل وصف الخلفيات الاستدلالية لـ CLI.
أما `setup.cliBackends` فهو سطح الواصفات الخاص بالإعداد لتدفقات مستوى التحكّم/الإعداد
التي ينبغي أن تبقى بيانات وصفية فقط.

عند وجودهما، يكون `setup.providers` و`setup.cliBackends` هما
سطح البحث المفضّل القائم على الواصفات أولًا لاكتشاف الإعداد. وإذا كان الواصف
يضيّق Plugin المرشّح فقط وكان الإعداد ما يزال يحتاج إلى خطافات وقت إعداد أكثر غنىً في وقت التنفيذ،
فعَيِّن `requiresRuntime: true` وأبقِ `setup-api` في مكانه بوصفه
مسار التنفيذ الاحتياطي.

ولأن بحث الإعداد قد ينفّذ كود `setup-api` المملوك لـ Plugin، يجب أن تبقى
القيم المطبّعة في `setup.providers[].id` و`setup.cliBackends[]` فريدة عبر
Plugins المكتشفة. وتفشل الملكية الملتبسة بشكل مغلق بدلًا من اختيار
فائز وفق ترتيب الاكتشاف.

### مرجع `setup.providers`

| الحقل         | مطلوب | النوع      | ما الذي يعنيه                                                                      |
| ------------- | ------ | ---------- | ---------------------------------------------------------------------------------- |
| `id`          | نعم    | `string`   | معرّف الموفّر الذي يُعرَض أثناء الإعداد أو التهيئة الأوّلية. أبقِ المعرّفات المطبّعة فريدة عالميًا. |
| `authMethods` | لا     | `string[]` | معرّفات طرق الإعداد/المصادقة التي يدعمها هذا الموفّر من دون تحميل وقت التنفيذ الكامل. |
| `envVars`     | لا     | `string[]` | متغيرات البيئة التي يمكن لأسطح الإعداد/الحالة العامة التحقق منها قبل تحميل وقت تنفيذ Plugin. |

### حقول `setup`

| الحقل              | مطلوب | النوع      | ما الذي يعنيه                                                                                     |
| ------------------ | ------ | ---------- | ------------------------------------------------------------------------------------------------- |
| `providers`        | لا     | `object[]` | واصفات إعداد الموفّر التي تُعرَض أثناء الإعداد والتهيئة الأوّلية.                                  |
| `cliBackends`      | لا     | `string[]` | معرّفات الخلفيات في وقت الإعداد المستخدمة في بحث الإعداد القائم على الواصفات أولًا. أبقِ المعرّفات المطبّعة فريدة عالميًا. |
| `configMigrations` | لا     | `string[]` | معرّفات ترحيل الإعدادات المملوكة لسطح إعداد هذا Plugin.                                           |
| `requiresRuntime`  | لا     | `boolean`  | ما إذا كان الإعداد ما يزال يحتاج إلى تنفيذ `setup-api` بعد البحث عبر الواصفات.                    |

## مرجع `uiHints`

`uiHints` هو خريطة من أسماء حقول الإعدادات إلى تلميحات عرض صغيرة.

```json
{
  "uiHints": {
    "apiKey": {
      "label": "API key",
      "help": "Used for OpenRouter requests",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  }
}
```

يمكن أن يتضمن تلميح كل حقل:

| الحقل         | النوع      | ما الذي يعنيه                        |
| ------------- | ---------- | ------------------------------------ |
| `label`       | `string`   | تسمية الحقل الموجّهة للمستخدم.       |
| `help`        | `string`   | نص مساعد قصير.                       |
| `tags`        | `string[]` | وسوم UI اختيارية.                    |
| `advanced`    | `boolean`  | يحدّد الحقل على أنه متقدّم.          |
| `sensitive`   | `boolean`  | يحدّد الحقل على أنه سري أو حساس.     |
| `placeholder` | `string`   | نص العنصر النائب لمدخلات النماذج.    |

## مرجع `contracts`

استخدم `contracts` فقط لبيانات وصفية ثابتة لملكية القدرات يمكن لـ OpenClaw
قراءتها من دون استيراد وقت تنفيذ Plugin.

```json
{
  "contracts": {
    "speechProviders": ["openai"],
    "realtimeTranscriptionProviders": ["openai"],
    "realtimeVoiceProviders": ["openai"],
    "mediaUnderstandingProviders": ["openai", "openai-codex"],
    "imageGenerationProviders": ["openai"],
    "videoGenerationProviders": ["qwen"],
    "webFetchProviders": ["firecrawl"],
    "webSearchProviders": ["gemini"],
    "tools": ["firecrawl_search", "firecrawl_scrape"]
  }
}
```

كل قائمة اختيارية:

| الحقل                            | النوع      | ما الذي يعنيه                                                |
| -------------------------------- | ---------- | ------------------------------------------------------------ |
| `speechProviders`                | `string[]` | معرّفات موفّري الكلام التي يملكها هذا Plugin.                |
| `realtimeTranscriptionProviders` | `string[]` | معرّفات موفّري النسخ الفوري التي يملكها هذا Plugin.          |
| `realtimeVoiceProviders`         | `string[]` | معرّفات موفّري الصوت الفوري التي يملكها هذا Plugin.          |
| `mediaUnderstandingProviders`    | `string[]` | معرّفات موفّري فهم الوسائط التي يملكها هذا Plugin.           |
| `imageGenerationProviders`       | `string[]` | معرّفات موفّري توليد الصور التي يملكها هذا Plugin.           |
| `videoGenerationProviders`       | `string[]` | معرّفات موفّري توليد الفيديو التي يملكها هذا Plugin.         |
| `webFetchProviders`              | `string[]` | معرّفات موفّري جلب الويب التي يملكها هذا Plugin.            |
| `webSearchProviders`             | `string[]` | معرّفات موفّري البحث في الويب التي يملكها هذا Plugin.       |
| `tools`                          | `string[]` | أسماء أدوات الوكيل التي يملكها هذا Plugin لفحوصات العقود المضمّنة. |

## مرجع `channelConfigs`

استخدم `channelConfigs` عندما يحتاج Plugin القناة إلى بيانات وصفية منخفضة الكلفة للإعدادات
قبل تحميل وقت التنفيذ.

```json
{
  "channelConfigs": {
    "matrix": {
      "schema": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "homeserverUrl": { "type": "string" }
        }
      },
      "uiHints": {
        "homeserverUrl": {
          "label": "Homeserver URL",
          "placeholder": "https://matrix.example.com"
        }
      },
      "label": "Matrix",
      "description": "Matrix homeserver connection",
      "preferOver": ["matrix-legacy"]
    }
  }
}
```

يمكن أن يتضمن كل إدخال قناة:

| الحقل         | النوع                    | ما الذي يعنيه                                                                             |
| ------------- | ------------------------ | ----------------------------------------------------------------------------------------- |
| `schema`      | `object`                 | مخطط JSON لـ `channels.<id>`. وهو مطلوب لكل إدخال إعدادات قناة مصرَّح به.                |
| `uiHints`     | `Record<string, object>` | تسميات UI أو العناصر النائبة أو تلميحات الحساسية الاختيارية لذلك القسم من إعدادات القناة. |
| `label`       | `string`                 | تسمية القناة التي تُدمج في أسطح المنتقي والفحص عندما لا تكون بيانات وقت التنفيذ الوصفية جاهزة. |
| `description` | `string`                 | وصف قصير للقناة لأسطح الفحص والفهرسة.                                                     |
| `preferOver`  | `string[]`               | معرّفات Plugins قديمة أو أقل أولوية ينبغي أن تتفوّق عليها هذه القناة في أسطح الاختيار.     |

## مرجع `modelSupport`

استخدم `modelSupport` عندما ينبغي على OpenClaw أن يستنتج Plugin الموفّر الخاص بك من
معرّفات نماذج مختصرة مثل `gpt-5.4` أو `claude-sonnet-4.6` قبل تحميل وقت تنفيذ Plugin.

```json
{
  "modelSupport": {
    "modelPrefixes": ["gpt-", "o1", "o3", "o4"],
    "modelPatterns": ["^computer-use-preview"]
  }
}
```

يطبّق OpenClaw أسبقية الترتيب التالية:

- تستخدم مراجع `provider/model` الصريحة بيانات البيان الوصفية `providers` المالكة
- تتفوّق `modelPatterns` على `modelPrefixes`
- إذا طابق Plugin غير مضمّن وPlugin مضمّن معًا، يفوز Plugin
  غير المضمّن
- يُتجاهل الغموض المتبقي إلى أن يحدّد المستخدم أو الإعدادات موفّرًا

الحقول:

| الحقل           | النوع      | ما الذي يعنيه                                                                  |
| --------------- | ---------- | ------------------------------------------------------------------------------ |
| `modelPrefixes` | `string[]` | بادئات تُطابق باستخدام `startsWith` مقابل معرّفات النماذج المختصرة.            |
| `modelPatterns` | `string[]` | مصادر تعبيرات منتظمة تُطابق مع معرّفات النماذج المختصرة بعد إزالة لاحقة الملف التعريفي. |

مفاتيح القدرات القديمة ذات المستوى الأعلى مهجورة. استخدم `openclaw doctor --fix` من أجل
نقل `speechProviders` و`realtimeTranscriptionProviders` و
`realtimeVoiceProviders` و`mediaUnderstandingProviders` و
`imageGenerationProviders` و`videoGenerationProviders` و
`webFetchProviders` و`webSearchProviders` تحت `contracts`؛ إذ لم يعد
تحميل البيان العادي يعامل تلك الحقول ذات المستوى الأعلى على أنها ملكية
قدرات.

## البيان مقابل package.json

يخدم الملفان مهمتين مختلفتين:

| الملف                  | استخدمه من أجل                                                                                                                   |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.plugin.json` | الاكتشاف، والتحقق من صحة الإعدادات، وبيانات خيارات المصادقة الوصفية، وتلميحات UI التي يجب أن توجد قبل تشغيل كود Plugin        |
| `package.json`         | بيانات npm الوصفية، وتثبيت الاعتماديات، وكتلة `openclaw` المستخدمة لنقاط الدخول، وضبط التثبيت، والإعداد، أو بيانات الفهرسة الوصفية |

إذا لم تكن متأكدًا من الموضع الذي ينبغي أن تنتمي إليه قطعة من البيانات الوصفية، فاستخدم هذه القاعدة:

- إذا كان يجب على OpenClaw معرفتها قبل تحميل كود Plugin، فضعها في `openclaw.plugin.json`
- إذا كانت تتعلق بالتغليف، أو ملفات الدخول، أو سلوك تثبيت npm، فضعها في `package.json`

### حقول `package.json` التي تؤثر في الاكتشاف

توجد بعض بيانات Plugin الوصفية قبل وقت التنفيذ عمدًا في `package.json` تحت
كتلة `openclaw` بدلًا من `openclaw.plugin.json`.

أمثلة مهمة:

| الحقل                                                             | ما الذي يعنيه                                                                                                                               |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.extensions`                                             | يصرّح بنقاط دخول Plugin الأصلية.                                                                                                           |
| `openclaw.setupEntry`                                             | نقطة دخول خفيفة مخصّصة للإعداد فقط وتُستخدم أثناء التهيئة الأوّلية، وبدء تشغيل القنوات المؤجل، واكتشاف حالة القناة/SecretRef للقراءة فقط. |
| `openclaw.channel`                                                | بيانات وصفية منخفضة الكلفة لفهرس القنوات مثل التسميات، ومسارات الوثائق، والأسماء المستعارة، ونصوص الاختيار.                               |
| `openclaw.channel.configuredState`                                | بيانات وصفية منخفضة الكلفة لفاحص الحالة المُعدّة يمكنها الإجابة عن "هل يوجد إعداد قائم يعتمد على البيئة فقط؟" دون تحميل وقت تنفيذ القناة الكامل. |
| `openclaw.channel.persistedAuthState`                             | بيانات وصفية منخفضة الكلفة لفاحص المصادقة المحفوظة يمكنها الإجابة عن "هل يوجد أي تسجيل دخول بالفعل؟" دون تحميل وقت تنفيذ القناة الكامل. |
| `openclaw.install.npmSpec` / `openclaw.install.localPath`         | تلميحات التثبيت/التحديث لـ Plugins المضمّنة والمنشورة خارجيًا.                                                                             |
| `openclaw.install.defaultChoice`                                  | مسار التثبيت المفضّل عندما تتوفر مصادر تثبيت متعددة.                                                                                       |
| `openclaw.install.minHostVersion`                                 | الحد الأدنى المدعوم من إصدار مضيف OpenClaw، باستخدام حد أدنى semver مثل `>=2026.3.22`.                                                   |
| `openclaw.install.allowInvalidConfigRecovery`                     | يسمح بمسار استرداد ضيق لإعادة تثبيت Plugin المضمّن عندما تكون الإعدادات غير صالحة.                                                         |
| `openclaw.startup.deferConfiguredChannelFullLoadUntilAfterListen` | يسمح لأسطح القنوات الخاصة بالإعداد فقط أن تُحمَّل قبل Plugin القناة الكامل أثناء بدء التشغيل.                                              |

يُفرض `openclaw.install.minHostVersion` أثناء التثبيت وتحميل
سجل البيانات الوصفية. تُرفض القيم غير الصالحة؛ أما القيم الأحدث لكنها صالحة فتتخطى
Plugin على المضيفين الأقدم.

ينبغي أن توفر Plugins القنوات `openclaw.setupEntry` عندما تحتاج الحالة أو قائمة القنوات
أو فحوصات SecretRef إلى تحديد الحسابات المُعدّة دون تحميل وقت التنفيذ الكامل.
ينبغي أن تكشف نقطة دخول الإعداد بيانات القناة الوصفية إضافة إلى موائمات الإعدادات،
والحالة، والأسرار الآمنة للإعداد؛ وأبقِ عملاء الشبكة، ومستمعي Gateway، وأوقات تنفيذ
النقل في نقطة دخول الامتداد الرئيسية.

إن `openclaw.install.allowInvalidConfigRecovery` ضيق عمدًا.
فهو لا يجعل الإعدادات المعطوبة التعسفية قابلة للتثبيت. فهو اليوم يسمح فقط
لتدفقات التثبيت بالتعافي من إخفاقات محددة قديمة لترقية Plugin مضمّن،
مثل فقدان مسار Plugin مضمّن أو إدخال `channels.<id>` قديم لذلك Plugin
المضمّن نفسه. وما تزال أخطاء الإعدادات غير المرتبطة تمنع التثبيت وتوجّه
المشغّلين إلى `openclaw doctor --fix`.

`openclaw.channel.persistedAuthState` هو بيانات حزمة وصفية لوحدة فاحص صغيرة:

```json
{
  "openclaw": {
    "channel": {
      "id": "whatsapp",
      "persistedAuthState": {
        "specifier": "./auth-presence",
        "exportName": "hasAnyWhatsAppAuth"
      }
    }
  }
}
```

استخدمه عندما تحتاج تدفقات الإعداد أو Doctor أو الحالة المُعدّة إلى
فحص مصادقة منخفض الكلفة بنعم/لا قبل تحميل Plugin القناة الكامل. وينبغي أن يكون
التصدير الهدف دالة صغيرة تقرأ الحالة المحفوظة فقط؛ ولا تمرره عبر شريط
وقت تنفيذ القناة الكامل.

يتبع `openclaw.channel.configuredState` الشكل نفسه لفحوصات الحالة المُعدّة
الرخيصة المعتمدة على البيئة فقط:

```json
{
  "openclaw": {
    "channel": {
      "id": "telegram",
      "configuredState": {
        "specifier": "./configured-state",
        "exportName": "hasTelegramConfiguredState"
      }
    }
  }
}
```

استخدمه عندما تستطيع القناة الإجابة عن الحالة المُعدّة من البيئة أو من مدخلات صغيرة
أخرى غير وقت التنفيذ. وإذا كان الفحص يحتاج إلى حل كامل للإعدادات أو إلى وقت تنفيذ
القناة الحقيقي، فأبقِ ذلك المنطق في خطاف Plugin ‏`config.hasConfiguredState`
بدلًا من ذلك.

## متطلبات مخطط JSON

- **يجب على كل Plugin أن يضم مخطط JSON**، حتى إذا كان لا يقبل أي إعدادات.
- يكون المخطط الفارغ مقبولًا (على سبيل المثال، `{ "type": "object", "additionalProperties": false }`).
- يجري التحقق من المخططات في وقت قراءة/كتابة الإعدادات، وليس في وقت التنفيذ.

## سلوك التحقق

- مفاتيح `channels.*` غير المعروفة تُعد **أخطاء**، ما لم يكن معرّف القناة مصرَّحًا به
  بواسطة بيان Plugin.
- يجب أن تشير `plugins.entries.<id>` و`plugins.allow` و`plugins.deny` و`plugins.slots.*`
  إلى معرّفات Plugins **قابلة للاكتشاف**. والمعرّفات غير المعروفة تُعد **أخطاء**.
- إذا كان Plugin مثبّتًا لكن بيانه أو مخططه معطوبًا أو مفقودًا،
  يفشل التحقق ويبلّغ Doctor عن خطأ Plugin.
- إذا كانت إعدادات Plugin موجودة لكن Plugin **معطّل**، فستُحتفَظ بالإعدادات
  ويظهر **تحذير** في Doctor + السجلات.

راجع [مرجع الإعدادات](/ar/gateway/configuration) للاطلاع على مخطط `plugins.*` الكامل.

## ملاحظات

- البيان **مطلوب لـ Plugins الأصلية في OpenClaw**، بما في ذلك التحميلات المحلية من نظام الملفات.
- يظل وقت التنفيذ يحمّل وحدة Plugin بشكل منفصل؛ والبيان مخصّص فقط
  للاكتشاف + التحقق.
- تُحلَّل البيانات الأصلية باستخدام JSON5، لذا تُقبل التعليقات، والفواصل اللاحقة،
  والمفاتيح غير المقتبسة ما دامت القيمة النهائية ما تزال كائنًا.
- لا يقرأ محمّل البيان إلا حقول البيان الموثقة. تجنّب إضافة
  مفاتيح مخصّصة ذات مستوى أعلى هنا.
- `providerAuthEnvVars` هو مسار البيانات الوصفية منخفض الكلفة لفحوصات المصادقة، والتحقق
  من علامات البيئة، وأسطر مصادقة الموفّر المشابهة التي ينبغي ألّا تشغّل وقت تنفيذ Plugin
  فقط لفحص أسماء البيئة.
- يتيح `providerAuthAliases` لمتغيرات الموفّر إعادة استخدام متغيرات بيئة مصادقة
  موفّر آخر، وملفات تعريف المصادقة، والمصادقة المعتمدة على الإعدادات، وخيار التهيئة
  الأوّلية لمفتاح API من دون ترميز صلب لتلك العلاقة في النواة.
- يتيح `providerEndpoints` لـ Plugins الموفّرين امتلاك بيانات وصفية بسيطة لمطابقة
  مضيف/‏`baseUrl` لنقاط النهاية. استخدمه فقط لفئات نقاط النهاية التي تدعمها النواة
  بالفعل؛ وما يزال Plugin يملك سلوك وقت التنفيذ.
- `syntheticAuthRefs` هو مسار البيانات الوصفية منخفض الكلفة لخطافات المصادقة الاصطناعية
  المملوكة للموفّر التي يجب أن تكون مرئية لاكتشاف النماذج البارد قبل وجود
  سجل وقت التنفيذ. أدرج فقط المراجع التي ينفّذ فيها موفّر وقت التنفيذ أو خلفية CLI
  فعلًا `resolveSyntheticAuth`.
- `nonSecretAuthMarkers` هو مسار البيانات الوصفية منخفض الكلفة لمفاتيح API النائبة
  المملوكة لـ Plugin المضمّن مثل العلامات المحلية أو OAuth أو بيانات الاعتماد المحيطة.
  تتعامل النواة معها على أنها غير سرية لعرض المصادقة وعمليات تدقيق الأسرار من دون
  ترميز صلب للموفّر المالك.
- `channelEnvVars` هو مسار البيانات الوصفية منخفض الكلفة للرجوع إلى بيئة الصدفة،
  ومطالبات الإعداد، وأسطر القنوات المشابهة التي ينبغي ألّا تشغّل وقت تنفيذ Plugin
  فقط لفحص أسماء البيئة.
- `providerAuthChoices` هو مسار البيانات الوصفية منخفض الكلفة لمنتقيات خيارات المصادقة،
  وحل `--auth-choice`، وربط الموفّر المفضّل، وتسجيل أعلام CLI البسيطة الخاصة
  بالتهيئة الأوّلية قبل تحميل وقت تنفيذ الموفّر. ولبيانات معالج وقت التنفيذ الوصفية
  التي تتطلب كود الموفّر، راجع
  [خطافات وقت تنفيذ الموفّر](/ar/plugins/architecture#provider-runtime-hooks).
- تُختار أنواع Plugin الحصرية عبر `plugins.slots.*`.
  - يُختار `kind: "memory"` بواسطة `plugins.slots.memory`.
  - يُختار `kind: "context-engine"` بواسطة `plugins.slots.contextEngine`
    (الافتراضي: `legacy` المضمّن).
- يمكن حذف `channels` و`providers` و`cliBackends` و`skills` عندما
  لا يحتاجها Plugin.
- إذا كان Plugin الخاص بك يعتمد على وحدات أصلية، فوثّق خطوات البناء وأي
  متطلبات قائمة سماح لمدير الحزم (على سبيل المثال، pnpm ‏`allow-build-scripts`
  - `pnpm rebuild <package>`).
