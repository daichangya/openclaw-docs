الحالة: تجريبي. الرسائل الخاصة مدعومة. يعكس قسم [الإمكانات](#capabilities) أدناه السلوك الحالي لروبوتات Marketplace.

## Plugin مضمّن

يأتي Zalo كـ Plugin مضمّن في إصدارات OpenClaw الحالية، لذا لا تحتاج
الإصدارات المجمعة العادية إلى تثبيت منفصل.

إذا كنت تستخدم إصدارًا أقدم أو تثبيتًا مخصصًا لا يتضمن Zalo، فقم بتثبيته
يدويًا:

- ثبّت عبر CLI: `openclaw plugins install @openclaw/zalo`
- أو من نسخة مصدر: `openclaw plugins install ./path/to/local/zalo-plugin`
- التفاصيل: [Plugins](/ar/tools/plugin)

## إعداد سريع (للمبتدئين)

1. تأكد من أن Plugin الخاص بـ Zalo متاح.
   - إصدارات OpenClaw المجمعة الحالية تتضمنه بالفعل.
   - يمكن للإصدارات الأقدم/المخصصة إضافته يدويًا باستخدام الأوامر أعلاه.
2. اضبط الرمز المميز:
   - متغير بيئة: `ZALO_BOT_TOKEN=...`
   - أو في الإعدادات: `channels.zalo.accounts.default.botToken: "..."`.
3. أعد تشغيل Gateway (أو أكمل الإعداد).
4. يكون الوصول إلى الرسائل الخاصة عبر الاقتران افتراضيًا؛ وافق على رمز الاقتران عند أول تواصل.

الحد الأدنى من الإعدادات:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

## ما هو

Zalo هو تطبيق مراسلة يركز على فيتنام؛ وتسمح واجهة Bot API الخاصة به لـ Gateway بتشغيل روبوت لمحادثات 1:1.
وهو مناسب للدعم أو الإشعارات عندما تريد توجيهًا حتميًا للعودة إلى Zalo.

تعكس هذه الصفحة السلوك الحالي لـ OpenClaw لروبوتات **Zalo Bot Creator / Marketplace**.
أما روبوتات **Zalo Official Account (OA)** فهي واجهة منتج مختلفة في Zalo وقد تتصرف بشكل مختلف.

- قناة Zalo Bot API مملوكة بواسطة Gateway.
- توجيه حتمي: تعود الردود إلى Zalo؛ ولا يختار النموذج القنوات أبدًا.
- تشارك الرسائل الخاصة الجلسة الرئيسية للوكيل.
- يعرض قسم [الإمكانات](#capabilities) أدناه الدعم الحالي لروبوتات Marketplace.

## الإعداد (المسار السريع)

### 1) إنشاء رمز مميز للروبوت (Zalo Bot Platform)

1. انتقل إلى [https://bot.zaloplatforms.com](https://bot.zaloplatforms.com) وسجّل الدخول.
2. أنشئ روبوتًا جديدًا واضبط إعداداته.
3. انسخ الرمز المميز الكامل للروبوت (عادة `numeric_id:secret`). بالنسبة لروبوتات Marketplace، قد يظهر رمز التشغيل القابل للاستخدام في رسالة الترحيب الخاصة بالروبوت بعد إنشائه.

### 2) تكوين الرمز المميز (متغير بيئة أو إعدادات)

مثال:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

إذا انتقلت لاحقًا إلى واجهة روبوت Zalo تتوفر فيها المجموعات، يمكنك إضافة إعدادات خاصة بالمجموعات مثل `groupPolicy` و`groupAllowFrom` بشكل صريح. بالنسبة لسلوك روبوتات Marketplace الحالي، راجع [الإمكانات](#capabilities).

خيار متغير البيئة: `ZALO_BOT_TOKEN=...` (يعمل مع الحساب الافتراضي فقط).

دعم الحسابات المتعددة: استخدم `channels.zalo.accounts` مع رموز مميزة لكل حساب و`name` اختياري.

3. أعد تشغيل Gateway. يبدأ Zalo عند حل الرمز المميز (من متغير البيئة أو الإعدادات).
4. يكون الوصول إلى الرسائل الخاصة عبر الاقتران افتراضيًا. وافق على الرمز عند أول تواصل مع الروبوت.

## كيف يعمل (السلوك)

- تتم تسوية الرسائل الواردة ضمن غلاف القناة المشترك مع عناصر نائبة للوسائط.
- يتم توجيه الردود دائمًا إلى دردشة Zalo نفسها.
- يستخدم الاستطلاع الطويل افتراضيًا؛ ويتوفر وضع Webhook عبر `channels.zalo.webhookUrl`.

## القيود

- يتم تقسيم النص الصادر إلى أجزاء من 2000 حرف (حد Zalo API).
- يحد `channels.zalo.mediaMaxMb` من تنزيل/رفع الوسائط (الافتراضي 5).
- يتم حظر البث افتراضيًا لأن حد 2000 حرف يجعل البث أقل فائدة.

## التحكم في الوصول (الرسائل الخاصة)

### الوصول إلى الرسائل الخاصة

- الافتراضي: `channels.zalo.dmPolicy = "pairing"`. يتلقى المرسلون غير المعروفين رمز اقتران؛ ويتم تجاهل الرسائل حتى الموافقة (تنتهي صلاحية الرموز بعد ساعة واحدة).
- وافق عبر:
  - `openclaw pairing list zalo`
  - `openclaw pairing approve zalo <CODE>`
- الاقتران هو تبادل الرموز المميز الافتراضي. التفاصيل: [الاقتران](/ar/channels/pairing)
- يقبل `channels.zalo.allowFrom` معرفات المستخدمين الرقمية (لا يتوفر البحث باسم المستخدم).

## التحكم في الوصول (المجموعات)

بالنسبة إلى روبوتات **Zalo Bot Creator / Marketplace**، لم يكن دعم المجموعات متاحًا عمليًا لأنه لم يكن بالإمكان إضافة الروبوت إلى مجموعة أصلًا.

وهذا يعني أن مفاتيح الإعدادات المتعلقة بالمجموعة أدناه موجودة في المخطط، لكنها لم تكن قابلة للاستخدام مع روبوتات Marketplace:

- يتحكم `channels.zalo.groupPolicy` في معالجة الرسائل الواردة من المجموعات: `open | allowlist | disabled`.
- يقيّد `channels.zalo.groupAllowFrom` معرفات المرسلين الذين يمكنهم تفعيل الروبوت في المجموعات.
- إذا لم يتم ضبط `groupAllowFrom`، يستخدم Zalo القيمة `allowFrom` كخيار احتياطي لفحص المرسلين.
- ملاحظة وقت التشغيل: إذا كان `channels.zalo` مفقودًا بالكامل، فسيعود وقت التشغيل إلى `groupPolicy="allowlist"` لأسباب تتعلق بالأمان.

قيم سياسة المجموعات (عندما يكون الوصول إلى المجموعات متاحًا على واجهة الروبوت لديك) هي:

- `groupPolicy: "disabled"` — يحظر جميع رسائل المجموعات.
- `groupPolicy: "open"` — يسمح لأي عضو في المجموعة (مع اشتراط الإشارة).
- `groupPolicy: "allowlist"` — الافتراضي المغلق عند الفشل؛ لا يُقبل إلا المرسلون المسموح لهم.

إذا كنت تستخدم واجهة منتج مختلفة لروبوت Zalo وتحققت من أن سلوك المجموعات يعمل فعليًا، فوثّق ذلك بشكل منفصل بدلًا من افتراض مطابقته لتدفق روبوتات Marketplace.

## الاستطلاع الطويل مقابل Webhook

- الافتراضي: الاستطلاع الطويل (لا يلزم URL عام).
- وضع Webhook: اضبط `channels.zalo.webhookUrl` و`channels.zalo.webhookSecret`.
  - يجب أن يتكون سر Webhook من 8 إلى 256 حرفًا.
  - يجب أن يستخدم URL الخاص بـ Webhook بروتوكول HTTPS.
  - يرسل Zalo الأحداث مع ترويسة `X-Bot-Api-Secret-Token` للتحقق.
  - يتعامل HTTP الخاص بـ Gateway مع طلبات Webhook على `channels.zalo.webhookPath` (الافتراضي هو مسار URL الخاص بـ Webhook).
  - يجب أن تستخدم الطلبات `Content-Type: application/json` (أو أنواع وسائط `+json`).
  - يتم تجاهل الأحداث المكررة (`event_name + message_id`) ضمن نافذة إعادة تشغيل قصيرة.
  - يتم تقييد الزيادات المفاجئة في الحركة حسب المسار/المصدر وقد تعيد HTTP 429.

**ملاحظة:** إن `getUpdates` (الاستطلاع) وWebhook متنافيان لكل وثائق Zalo API.

## أنواع الرسائل المدعومة

للحصول على لمحة سريعة عن الدعم، راجع [الإمكانات](#capabilities). تضيف الملاحظات أدناه تفاصيل حين يحتاج السلوك إلى سياق إضافي.

- **الرسائل النصية**: دعم كامل مع تقسيم إلى 2000 حرف.
- **عناوين URL العادية داخل النص**: تتصرف مثل إدخال نصي عادي.
- **معاينات الروابط / بطاقات الروابط الغنية**: راجع حالة روبوتات Marketplace في [الإمكانات](#capabilities)؛ إذ لم تكن تؤدي إلى رد بشكل موثوق.
- **رسائل الصور**: راجع حالة روبوتات Marketplace في [الإمكانات](#capabilities)؛ كانت معالجة الصور الواردة غير موثوقة (مؤشر كتابة دون رد نهائي).
- **الملصقات**: راجع حالة روبوتات Marketplace في [الإمكانات](#capabilities).
- **الملاحظات الصوتية / الملفات الصوتية / الفيديو / مرفقات الملفات العامة**: راجع حالة روبوتات Marketplace في [الإمكانات](#capabilities).
- **الأنواع غير المدعومة**: يتم تسجيلها في السجل (مثل الرسائل من المستخدمين المحميين).

## أهداف التسليم (CLI/cron)

- استخدم معرّف الدردشة كهدف.
- مثال: `openclaw message send --channel zalo --target 123456789 --message "hi"`.

## استكشاف الأخطاء وإصلاحها

**الروبوت لا يستجيب:**

- تحقق من صلاحية الرمز المميز: `openclaw channels status --probe`
- تحقق من أن المرسِل تمت الموافقة عليه (عبر الاقتران أو allowFrom)
- تحقق من سجلات Gateway: `openclaw logs --follow`

**Webhook لا يستقبل الأحداث:**

- تأكد من أن URL الخاص بـ Webhook يستخدم HTTPS
- تحقق من أن الرمز السري يتكون من 8 إلى 256 حرفًا
- تأكد من أن نقطة نهاية HTTP الخاصة بـ Gateway يمكن الوصول إليها على المسار المضبوط
- تحقق من أن استطلاع `getUpdates` لا يعمل (فهما متنافيان)

## مرجع الإعدادات (Zalo)

الإعدادات الكاملة: [الإعدادات](/ar/gateway/configuration)

تُعد المفاتيح المسطحة ذات المستوى الأعلى (`channels.zalo.botToken` و`channels.zalo.dmPolicy` وما شابه) صيغة قديمة مختصرة للحساب الواحد. فضّل `channels.zalo.accounts.<id>.*` في الإعدادات الجديدة. لا يزال كلا الشكلين موثقًا هنا لأنهما موجودان في المخطط.

خيارات المزوّد:

- `channels.zalo.enabled`: تمكين/تعطيل بدء تشغيل القناة.
- `channels.zalo.botToken`: الرمز المميز للروبوت من Zalo Bot Platform.
- `channels.zalo.tokenFile`: قراءة الرمز المميز من مسار ملف عادي. يتم رفض الروابط الرمزية.
- `channels.zalo.dmPolicy`: ‏`pairing | allowlist | open | disabled` (الافتراضي: pairing).
- `channels.zalo.allowFrom`: قائمة سماح الرسائل الخاصة (معرفات المستخدمين). يتطلب `open` وجود `"*"`. سيطلب المعالج المعرفات الرقمية.
- `channels.zalo.groupPolicy`: ‏`open | allowlist | disabled` (الافتراضي: allowlist). موجود في الإعدادات؛ راجع [الإمكانات](#capabilities) و[التحكم في الوصول (المجموعات)](#access-control-groups) لمعرفة سلوك روبوتات Marketplace الحالي.
- `channels.zalo.groupAllowFrom`: قائمة سماح مرسلي المجموعات (معرفات المستخدمين). ترجع إلى `allowFrom` عند عدم ضبطها.
- `channels.zalo.mediaMaxMb`: حد الوسائط الواردة/الصادرة (ميغابايت، الافتراضي 5).
- `channels.zalo.webhookUrl`: تمكين وضع Webhook (يتطلب HTTPS).
- `channels.zalo.webhookSecret`: سر Webhook (من 8 إلى 256 حرفًا).
- `channels.zalo.webhookPath`: مسار Webhook على خادم HTTP الخاص بـ Gateway.
- `channels.zalo.proxy`: URL للوكيل الخاص بطلبات API.

خيارات الحسابات المتعددة:

- `channels.zalo.accounts.<id>.botToken`: رمز مميز لكل حساب.
- `channels.zalo.accounts.<id>.tokenFile`: ملف رمز مميز عادي لكل حساب. يتم رفض الروابط الرمزية.
- `channels.zalo.accounts.<id>.name`: اسم العرض.
- `channels.zalo.accounts.<id>.enabled`: تمكين/تعطيل الحساب.
- `channels.zalo.accounts.<id>.dmPolicy`: سياسة الرسائل الخاصة لكل حساب.
- `channels.zalo.accounts.<id>.allowFrom`: قائمة سماح لكل حساب.
- `channels.zalo.accounts.<id>.groupPolicy`: سياسة المجموعات لكل حساب. موجودة في الإعدادات؛ راجع [الإمكانات](#capabilities) و[التحكم في الوصول (المجموعات)](#access-control-groups) لمعرفة سلوك روبوتات Marketplace الحالي.
- `channels.zalo.accounts.<id>.groupAllowFrom`: قائمة سماح مرسلي المجموعات لكل حساب.
- `channels.zalo.accounts.<id>.webhookUrl`: URL لـ Webhook لكل حساب.
- `channels.zalo.accounts.<id>.webhookSecret`: سر Webhook لكل حساب.
- `channels.zalo.accounts.<id>.webhookPath`: مسار Webhook لكل حساب.
- `channels.zalo.accounts.<id>.proxy`: URL وكيل لكل حساب.
