# استكشاف أخطاء القنوات وإصلاحها

استخدم هذه الصفحة عندما تتصل قناة ما ولكن يكون السلوك غير صحيح.

## تسلسل الأوامر

شغّل هذه الأوامر بهذا الترتيب أولًا:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

خط الأساس السليم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only` أو `write-capable` أو `admin-capable`
- يُظهر فحص القناة أن النقل متصل، وحيثما كان ذلك مدعومًا، `works` أو `audit ok`

## WhatsApp

### بصمات فشل WhatsApp

| العَرَض                         | أسرع فحص                                           | الإصلاح                                                  |
| ------------------------------- | -------------------------------------------------- | -------------------------------------------------------- |
| متصل ولكن لا توجد ردود على الرسائل المباشرة | `openclaw pairing list whatsapp`                   | وافق على المرسل أو بدّل سياسة الرسائل المباشرة/قائمة السماح. |
| يتم تجاهل رسائل المجموعات       | تحقّق من `requireMention` + أنماط الإشارة في التكوين | اذكر bot أو خفف سياسة الإشارة لتلك المجموعة.            |
| عمليات قطع اتصال/إعادة تسجيل دخول عشوائية | `openclaw channels status --probe` + السجلات        | أعد تسجيل الدخول وتحقق من سلامة دليل بيانات الاعتماد.    |

الاستكشاف الكامل للأخطاء وإصلاحها: [استكشاف أخطاء WhatsApp وإصلاحها](/ar/channels/whatsapp#troubleshooting)

## Telegram

### بصمات فشل Telegram

| العَرَض                                 | أسرع فحص                                        | الإصلاح                                                                                                                      |
| --------------------------------------- | ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `/start` ولكن لا يوجد تدفق رد قابل للاستخدام | `openclaw pairing list telegram`                | وافق على الاقتران أو غيّر سياسة الرسائل المباشرة.                                                                            |
| bot متصل ولكن المجموعة تظل صامتة         | تحقّق من شرط الإشارة ووضع خصوصية bot           | عطّل وضع الخصوصية لإتاحة رؤية المجموعة أو اذكر bot.                                                                         |
| فشل الإرسال مع أخطاء شبكة               | افحص السجلات بحثًا عن فشل استدعاءات Telegram API | أصلح توجيه DNS/IPv6/proxy إلى `api.telegram.org`.                                                                            |
| يتوقف polling أو يعاود الاتصال ببطء     | `openclaw logs --follow` لتشخيصات polling       | قم بالترقية؛ وإذا كانت عمليات إعادة التشغيل إيجابيات كاذبة، فاضبط `pollingStallThresholdMs`. وما زالت حالات التوقف المستمرة تشير إلى proxy/DNS/IPv6. |
| تم رفض `setMyCommands` عند بدء التشغيل   | افحص السجلات بحثًا عن `BOT_COMMANDS_TOO_MUCH`   | قلّل أوامر Telegram المخصصة لـ plugin/Skills/المستخدمة أو عطّل القوائم الأصلية.                                             |
| قمت بالترقية وقائمة السماح تمنعك         | `openclaw security audit` وقوائم السماح في التكوين | شغّل `openclaw doctor --fix` أو استبدل `@username` بمعرّفات مرسلين رقمية.                                                   |
