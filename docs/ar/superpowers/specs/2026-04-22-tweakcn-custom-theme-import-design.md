# تصميم استيراد السمة المخصصة من Tweakcn

الحالة: تمت الموافقة عليها في الطرفية بتاريخ 2026-04-22

## الملخص

أضف خانة واحدة فقط لسمة مخصصة محلية للمتصفح في Control UI يمكن استيرادها من رابط مشاركة tweakcn. تظل عائلات السمات المضمّنة الحالية هي `claw` و`knot` و`dash`. تتصرف العائلة الجديدة `custom` مثل عائلة سمات OpenClaw عادية، وتدعم وضعي `light` و`dark` ووضع `system` عندما تتضمن حمولة tweakcn المستوردة مجموعتي tokens لكل من الوضع الفاتح والداكن.

تُخزَّن السمة المستوردة فقط في ملف تعريف المتصفح الحالي مع بقية إعدادات Control UI. ولا تُكتب إلى إعدادات Gateway ولا تتم مزامنتها عبر الأجهزة أو المتصفحات.

## المشكلة

نظام السمات في Control UI مغلق حاليًا على ثلاث عائلات سمات مضمّنة ومشفّرة بشكل ثابت:

- `ui/src/ui/theme.ts`
- `ui/src/ui/views/config.ts`
- `ui/src/styles/base.css`

يمكن للمستخدمين التبديل بين العائلات المضمّنة ومتغيرات الأوضاع، لكن لا يمكنهم جلب سمة من tweakcn دون تعديل CSS في المستودع. النتيجة المطلوبة أصغر من نظام سمات عام: الإبقاء على السمات الثلاث المضمّنة وإضافة خانة واحدة يتحكم فيها المستخدم لسمة مستوردة يمكن استبدالها من رابط tweakcn.

## الأهداف

- الإبقاء على عائلات السمات المضمّنة الحالية دون تغيير.
- إضافة خانة واحدة فقط للاستيراد، وليس مكتبة سمات.
- قبول رابط مشاركة tweakcn أو عنوان URL مباشر من الشكل `https://tweakcn.com/r/themes/{id}`.
- حفظ السمة المستوردة في التخزين المحلي للمتصفح فقط.
- جعل الخانة المستوردة تعمل مع عناصر التحكم الحالية `light` و`dark` و`system`.
- الحفاظ على سلوك آمن عند الفشل: يجب ألا يؤدي الاستيراد السيئ إلى كسر السمة النشطة في واجهة المستخدم.

## ما ليس من الأهداف

- لا توجد مكتبة سمات متعددة أو قائمة محلية في المتصفح للاستيرادات.
- لا يوجد حفظ على جانب Gateway أو مزامنة عبر الأجهزة.
- لا يوجد محرر CSS عشوائي أو محرر JSON خام للسمات.
- لا يوجد تحميل تلقائي لأصول خطوط بعيدة من tweakcn.
- لا توجد محاولة لدعم حمولات tweakcn التي تعرض وضعًا واحدًا فقط.
- لا توجد إعادة هيكلة عامة للسمات على مستوى المستودع تتجاوز الوصلات المطلوبة لـ Control UI.

## قرارات المستخدم المتخذة مسبقًا

- الإبقاء على السمات الثلاث المضمّنة.
- إضافة خانة استيراد واحدة مدعومة بواسطة tweakcn.
- تخزين السمة المستوردة في المتصفح، وليس في إعدادات Gateway.
- دعم `light` و`dark` و`system` للسمة المستوردة.
- الكتابة فوق الخانة المخصصة بالاستيراد التالي هو السلوك المقصود.

## النهج الموصى به

أضف معرّف عائلة سمة رابعًا، `custom`، إلى نموذج سمات Control UI. تصبح عائلة `custom` قابلة للاختيار فقط عند وجود استيراد tweakcn صالح. وتُطبّع الحمولة المستوردة إلى سجل سمة مخصصة خاص بـ OpenClaw وتُخزَّن في التخزين المحلي للمتصفح مع بقية إعدادات واجهة المستخدم.

أثناء وقت التشغيل، يعرض OpenClaw وسم `<style>` مُدارًا يعرّف كتل متغيرات CSS المخصصة المحلولة:

```css
:root[data-theme="custom"] { ... }
:root[data-theme="custom-light"] { ... }
```

يبقي هذا متغيرات السمة المخصصة ضمن نطاق العائلة `custom` ويمنع تسرّب متغيرات CSS المضمّنة مباشرة إلى العائلات المضمّنة.

## البنية

### نموذج السمة

حدّث `ui/src/ui/theme.ts`:

- وسّع `ThemeName` ليشمل `custom`.
- وسّع `ResolvedTheme` ليشمل `custom` و`custom-light`.
- وسّع `VALID_THEME_NAMES`.
- حدّث `resolveTheme()` بحيث تعكس `custom` سلوك العائلة الحالي:
  - `custom + dark` -> `custom`
  - `custom + light` -> `custom-light`
  - `custom + system` -> `custom` أو `custom-light` وفقًا لتفضيل نظام التشغيل

لا تُضاف أسماء بديلة قديمة لـ `custom`.

### نموذج الحفظ

وسّع حفظ `UiSettings` في `ui/src/ui/storage.ts` بحمولة اختيارية واحدة للسمة المخصصة:

- `customTheme?: ImportedCustomTheme`

الشكل الموصى به للتخزين:

```ts
type ImportedCustomTheme = {
  sourceUrl: string;
  themeId: string;
  label: string;
  importedAt: string;
  light: Record<string, string>;
  dark: Record<string, string>;
};
```

ملاحظات:

- يخزّن `sourceUrl` إدخال المستخدم الأصلي بعد التطبيع.
- `themeId` هو معرّف سمة tweakcn المستخرج من الرابط.
- `label` هو الحقل `name` من tweakcn عند وجوده، وإلا فالقيمة `Custom`.
- القيمتان `light` و`dark` هما خرائط tokens مطبّعة مسبقًا لـ OpenClaw، وليستا حمولة tweakcn الخام.
- تعيش الحمولة المستوردة بجانب بقية الإعدادات المحلية للمتصفح وتُسلسَل في مستند التخزين المحلي نفسه.
- إذا كانت بيانات السمة المخصصة المخزنة مفقودة أو غير صالحة عند التحميل، فتجاهل الحمولة وارجع إلى `theme: "claw"` عندما تكون العائلة المحفوظة هي `custom`.

### التطبيق أثناء وقت التشغيل

أضف مدير stylesheet ضيقًا للسمة المخصصة في وقت تشغيل Control UI، ويكون مملوكًا بالقرب من `ui/src/ui/app-settings.ts` و`ui/src/ui/theme.ts`.

المسؤوليات:

- إنشاء أو تحديث وسم `<style id="openclaw-custom-theme">` ثابت واحد في `document.head`.
- توليد CSS فقط عند وجود حمولة سمة مخصصة صالحة.
- إزالة محتوى وسم النمط عند مسح الحمولة.
- إبقاء CSS الخاص بالعائلات المضمّنة في `ui/src/styles/base.css`؛ ولا تقم بدمج tokens المستوردة داخل stylesheet المُثبت في المستودع.

يعمل هذا المدير كلما جرى تحميل الإعدادات أو حفظها أو استيرادها أو مسحها.

### محددات الوضع الفاتح

يجب أن يفضّل التنفيذ `data-theme-mode="light"` للتنسيق الفاتح عبر العائلات بدلًا من المعالجة الخاصة لـ `custom-light`. وإذا كان هناك محدد حاليًا مثبت على `data-theme="light"` ويجب أن ينطبق على كل عائلة فاتحة، فوسّعه ضمن هذا العمل.

## تجربة الاستيراد

حدّث `ui/src/ui/views/config.ts` في قسم `Appearance`:

- أضف بطاقة سمة `Custom` إلى جانب `Claw` و`Knot` و`Dash`.
- أظهر البطاقة على أنها معطلة عند عدم وجود سمة مخصصة مستوردة.
- أضف لوحة استيراد تحت شبكة السمات تتضمن:
  - حقل نص واحد لرابط مشاركة tweakcn أو عنوان URL من الشكل `/r/themes/{id}`
  - زر `Import` واحد
  - مسار `Replace` واحد عندما تكون هناك حمولة مخصصة موجودة بالفعل
  - إجراء `Clear` واحد عندما تكون هناك حمولة مخصصة موجودة بالفعل
- اعرض تسمية السمة المستوردة واسم المضيف للمصدر عند وجود حمولة.
- إذا كانت السمة النشطة هي `custom`، فإن استيراد بديل يطبَّق فورًا.
- إذا لم تكن السمة النشطة هي `custom`، فإن الاستيراد يخزن الحمولة الجديدة فقط حتى يختار المستخدم بطاقة `Custom`.

كما يجب أن يُظهر منتقي السمة في الإعدادات السريعة في `ui/src/ui/views/config-quick.ts` الخيار `Custom` فقط عند وجود حمولة.

## تحليل URL والجلب البعيد

يقبل مسار الاستيراد في المتصفح:

- `https://tweakcn.com/themes/{id}`
- `https://tweakcn.com/r/themes/{id}`

يجب أن يطبّع التنفيذ كلا الشكلين إلى:

- `https://tweakcn.com/r/themes/{id}`

ثم يجلب المتصفح نقطة النهاية المطَبَّعة `/r/themes/{id}` مباشرة.

استخدم مدقق schema ضيقًا للحمولة الخارجية. يُفضَّل استخدام zod لأن هذه حدود خارجية غير موثوقة.

الحقول البعيدة المطلوبة:

- `name` على المستوى الأعلى كسلسلة اختيارية
- `cssVars.theme` ككائن اختياري
- `cssVars.light` ككائن
- `cssVars.dark` ككائن

إذا كان أي من `cssVars.light` أو `cssVars.dark` مفقودًا، فارفض الاستيراد. وهذا مقصود: السلوك المعتمد للمنتج هو دعم كامل للأوضاع، وليس توليدًا تقريبيًا عند غياب أحد الجانبين.

### الخطوط

تُستورد سلاسل حزم الخطوط عند وجودها، لكن OpenClaw لا يحمّل أصول خطوط بعيدة في الإصدار الأول. وإذا كانت الحزمة المستوردة تشير إلى خطوط غير متاحة في المتصفح، فسيُطبق سلوك التراجع العادي.

## سلوك الفشل

يجب أن تفشل الاستيرادات السيئة بشكل مغلق.

- تنسيق URL غير صالح: اعرض خطأ تحقق مضمنًا، ولا تنفذ الجلب.
- مضيف أو شكل مسار غير مدعوم: اعرض خطأ تحقق مضمنًا، ولا تنفذ الجلب.
- فشل الشبكة، أو استجابة غير سليمة، أو JSON غير صالح: اعرض خطأ مضمنًا، وأبقِ الحمولة المخزنة الحالية دون تعديل.
- فشل schema أو غياب كتلتي light/dark: اعرض خطأ مضمنًا، وأبقِ الحمولة المخزنة الحالية دون تعديل.
- إجراء المسح:
  - يزيل الحمولة المخصصة المخزنة
  - يزيل محتوى وسم النمط المخصص المُدار
  - إذا كانت `custom` نشطة، فبدّل عائلة السمة مرة أخرى إلى `claw`
- حمولة سمة مخصصة مخزنة غير صالحة عند أول تحميل:
  - تجاهل الحمولة المخزنة
  - لا تولّد CSS مخصصًا
  - إذا كانت عائلة السمة المحفوظة هي `custom`، فارجع إلى `claw`

في أي نقطة يجب ألا يترك الاستيراد الفاشل المستند النشط مع تطبيق جزئي لمتغيرات CSS المخصصة.
