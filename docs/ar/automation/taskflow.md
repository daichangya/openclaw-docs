# تدفق المهام

تدفق المهام هو البنية الأساسية لتنسيق التدفقات التي تقع فوق [المهام في الخلفية](/ar/automation/tasks). وهو يدير التدفقات المستمرة متعددة الخطوات مع حالتها الخاصة، وتتبع المراجعات، ودلالات المزامنة، بينما تظل المهام الفردية هي وحدة العمل المنفصل.

## متى تستخدم تدفق المهام

استخدم تدفق المهام عندما يمتد العمل عبر خطوات متسلسلة متعددة أو متفرعة وتحتاج إلى تتبع تقدّم مستمر عبر عمليات إعادة تشغيل Gateway. بالنسبة لعمليات الخلفية المنفردة، تكون [المهمة](/ar/automation/tasks) العادية كافية.

| السيناريو                              | الاستخدام                 |
| ------------------------------------- | ------------------------- |
| مهمة واحدة في الخلفية                 | مهمة عادية                |
| مسار متعدد الخطوات (A ثم B ثم C)      | تدفق المهام (مُدار)       |
| مراقبة المهام المُنشأة خارجيًا         | تدفق المهام (معكوس)       |
| تذكير لمرة واحدة                      | مهمة Cron                 |

## أوضاع المزامنة

### الوضع المُدار

يمتلك تدفق المهام دورة الحياة بالكامل من البداية إلى النهاية. فهو ينشئ المهام كخطوات للتدفق، ويدفعها حتى الاكتمال، ويُقدّم حالة التدفق تلقائيًا.

مثال: تدفق تقرير أسبوعي يقوم بـ (1) جمع البيانات، و(2) إنشاء التقرير، و(3) تسليمه. ينشئ تدفق المهام كل خطوة كمهمة في الخلفية، وينتظر اكتمالها، ثم ينتقل إلى الخطوة التالية.

```bash
Flow: weekly-report
  Step 1: gather-data     → task created → succeeded
  Step 2: generate-report → task created → succeeded
  Step 3: deliver         → task created → running
```

### الوضع المعكوس

يراقب تدفق المهام المهام المُنشأة خارجيًا ويحافظ على مزامنة حالة التدفق من دون تولّي إنشاء المهام. ويكون هذا مفيدًا عندما تنشأ المهام من وظائف Cron أو أوامر CLI أو مصادر أخرى وتريد عرضًا موحدًا لتقدّمها كتدفق.

مثال: ثلاث وظائف Cron مستقلة تشكّل معًا روتين "عمليات الصباح". يتتبّع التدفق المعكوس تقدّمها الجماعي من دون التحكم في وقت تشغيلها أو كيفية تشغيلها.

## الحالة المستمرة وتتبع المراجعات

يحتفظ كل تدفق بحالته الخاصة ويتتبّع المراجعات بحيث يستمر التقدّم عبر عمليات إعادة تشغيل Gateway. ويتيح تتبّع المراجعات اكتشاف التعارضات عندما تحاول مصادر متعددة تقديم نفس التدفق في الوقت نفسه.

## سلوك الإلغاء

يضبط `openclaw tasks flow cancel` نية إلغاء ثابتة على التدفق. تُلغى المهام النشطة داخل التدفق، ولا تبدأ أي خطوات جديدة. وتستمر نية الإلغاء عبر عمليات إعادة التشغيل، لذلك يظل التدفق الملغى ملغى حتى إذا أُعيد تشغيل Gateway قبل انتهاء جميع المهام التابعة.

## أوامر CLI

```bash
# List active and recent flows
openclaw tasks flow list

# Show details for a specific flow
openclaw tasks flow show <lookup>

# Cancel a running flow and its active tasks
openclaw tasks flow cancel <lookup>
```

| الأمر                           | الوصف                                          |
| --------------------------------- | --------------------------------------------- |
| `openclaw tasks flow list`        | يعرض التدفقات المتتبعة مع الحالة ووضع المزامنة |
| `openclaw tasks flow show <id>`   | يفحص تدفقًا واحدًا حسب معرّف التدفق أو مفتاح البحث |
| `openclaw tasks flow cancel <id>` | يلغي تدفقًا قيد التشغيل ومهامه النشطة         |

## كيف ترتبط التدفقات بالمهام

تنسّق التدفقات المهام، ولا تستبدلها. قد يدير تدفق واحد عدة مهام في الخلفية على مدار عمره. استخدم `openclaw tasks` لفحص سجلات المهام الفردية، واستخدم `openclaw tasks flow` لفحص التدفق المُنسِّق.
