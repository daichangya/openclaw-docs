يمكن أن تحمل مخرجات المساعد مجموعة صغيرة من توجيهات التسليم/العرض:

- `MEDIA:` لتسليم المرفقات
- `[[audio_as_voice]]` لتلميحات عرض الصوت
- `[[reply_to_current]]` / `[[reply_to:<id>]]` لبيانات الرد الوصفية
- `[embed ...]` للعرض الغني في واجهة مستخدم Control

هذه التوجيهات منفصلة. تظل `MEDIA:` ووسوم الرد/الصوت بيانات وصفية للتسليم؛ أما `[embed ...]` فهو مسار العرض الغني المخصص للويب فقط.

عند تفعيل البث الكتلي، تظل `MEDIA:` بيانات وصفية لتسليم واحد فقط لكل
دورة. وإذا أُرسل عنوان URL للوسائط نفسه في كتلة متدفقة ثم تكرر في حمولة
المساعد النهائية، يسلّم OpenClaw المرفق مرة واحدة ويزيل النسخة المكررة
من الحمولة النهائية.

## `[embed ...]`

يُعد `[embed ...]` صيغة العرض الغني الوحيدة الموجهة للعامل في واجهة مستخدم Control.

مثال ذاتي الإغلاق:

```text
[embed ref="cv_123" title="Status" /]
```

القواعد:

- لم يعد `[view ...]` صالحًا للمخرجات الجديدة.
- تُعرض الشيفرات القصيرة للتضمين داخل سطح رسالة المساعد فقط.
- لا تُعرض إلا التضمينات المدعومة بعنوان URL. استخدم `ref="..."` أو `url="..."`.
- لا تُعرض الشيفرات القصيرة للتضمين بصيغة HTML المضمنة على هيئة كتلة.
- تزيل واجهة الويب الشيفرة القصيرة من النص المرئي وتعرض التضمين داخل السطر.
- لا تُعد `MEDIA:` اسمًا مستعارًا للتضمين ويجب عدم استخدامها لعرض التضمين الغني.

## شكل العرض المخزن

كتلة محتوى المساعد الموحّدة/المخزنة هي عنصر `canvas` منظم:

```json
{
  "type": "canvas",
  "preview": {
    "kind": "canvas",
    "surface": "assistant_message",
    "render": "url",
    "viewId": "cv_123",
    "url": "/__openclaw__/canvas/documents/cv_123/index.html",
    "title": "Status",
    "preferredHeight": 320
  }
}
```

تستخدم الكتل الغنية المخزنة/المعروضة شكل `canvas` هذا مباشرة. ولا يتم التعرف على `present_view`.
