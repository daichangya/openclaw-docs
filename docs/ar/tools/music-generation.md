تتيح الأداة `music_generate` للعامل إنشاء الموسيقى أو الصوت عبر
قدرة توليد الموسيقى المشتركة باستخدام مزوّدين مضبوطين مثل Google
وMiniMax وComfyUI المضبوط عبر سير العمل.

بالنسبة إلى جلسات العامل المعتمدة على مزوّدين مشتركين، يبدأ OpenClaw توليد الموسيقى
كمهمة في الخلفية، ويتتبعها في سجل المهام، ثم يوقظ العامل مرة أخرى عندما
يصبح المقطع جاهزًا حتى يتمكن العامل من نشر الصوت النهائي مرة أخرى في
القناة الأصلية.

<Note>
لا تظهر الأداة المشتركة المضمّنة إلا عند توفر مزوّد واحد على الأقل لتوليد الموسيقى. إذا لم ترَ `music_generate` ضمن أدوات العامل، فاضبط `agents.defaults.musicGenerationModel` أو أعد إعداد مفتاح API للمزوّد.
</Note>

## البدء السريع

### التوليد المعتمد على مزوّدين مشتركين

1. اضبط مفتاح API لمزوّد واحد على الأقل، مثل `GEMINI_API_KEY` أو
   `MINIMAX_API_KEY`.
2. اضبط نموذجك المفضل اختياريًا:

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "google/lyria-3-clip-preview",
      },
    },
  },
}
```

3. اطلب من العامل: _"أنشئ مقطع synthpop مفعمًا بالحيوية عن قيادة ليلية
   عبر مدينة نيون."_

سيستدعي العامل `music_generate` تلقائيًا. ولا حاجة إلى قائمة سماح للأداة.

أما في السياقات المتزامنة المباشرة من دون تشغيل عامل مدعوم بجلسة، فإن
الأداة المضمّنة ما تزال تعود إلى التوليد المضمن وتعيد مسار الوسائط النهائي في
نتيجة الأداة.

أمثلة على المطالبات:

```text
أنشئ مقطع بيانو سينمائيًا مع أوتار ناعمة ومن دون غناء.
```

```text
أنشئ حلقة chiptune حماسية عن إطلاق صاروخ عند شروق الشمس.
```

### التوليد عبر Comfy المدفوع بسير العمل

يتكامل Plugin ‏`comfy` المضمّن مع الأداة المشتركة `music_generate` عبر
سجل مزوّدي توليد الموسيقى.

1. اضبط `plugins.entries.comfy.config.music` باستخدام JSON خاص بسير العمل
   وعُقد prompt/output.
2. إذا كنت تستخدم Comfy Cloud، فاضبط `COMFY_API_KEY` أو `COMFY_CLOUD_API_KEY`.
3. اطلب من العامل إنشاء موسيقى أو استدعِ الأداة مباشرة.

مثال:

```text
/tool music_generate prompt="حلقة synth ambient دافئة مع ملمس شريط ناعم"
```

## دعم المزوّدين المشتركين المضمّنين

| المزوّد | النموذج الافتراضي      | مدخلات مرجعية    | عناصر التحكم المدعومة                                  | مفتاح API                              |
| ------- | ---------------------- | ---------------- | ------------------------------------------------------ | -------------------------------------- |
| ComfyUI | `workflow`             | حتى صورة واحدة   | موسيقى أو صوت يحدده سير العمل                          | `COMFY_API_KEY`, `COMFY_CLOUD_API_KEY` |
| Google  | `lyria-3-clip-preview` | حتى 10 صور       | `lyrics`, `instrumental`, `format`                     | `GEMINI_API_KEY`, `GOOGLE_API_KEY`     |
| MiniMax | `music-2.6`            | لا شيء           | `lyrics`, `instrumental`, `durationSeconds`, `format=mp3` | `MINIMAX_API_KEY`                   |

### مصفوفة القدرات المعلنة

هذا هو عقد الأوضاع الصريح الذي تستخدمه `music_generate` واختبارات العقود
والتشغيل الحي المشترك.

| المزوّد | `generate` | `edit` | حد التحرير | مسارات الاختبار الحي المشتركة                                                 |
| ------- | ---------- | ------ | ---------- | ----------------------------------------------------------------------------- |
| ComfyUI | نعم        | نعم    | صورة واحدة | ليس ضمن التشغيل المشترك؛ ومغطى بواسطة `extensions/comfy/comfy.live.test.ts` |
| Google  | نعم        | نعم    | 10 صور     | `generate`, `edit`                                                            |
| MiniMax | نعم        | لا     | لا يوجد    | `generate`                                                                    |

استخدم `action: "list"` لفحص المزوّدين والنماذج المشتركة المتاحة
أثناء وقت التشغيل:

```text
/tool music_generate action=list
```

استخدم `action: "status"` لفحص مهمة الموسيقى النشطة المدعومة بالجلسة:

```text
/tool music_generate action=status
```

مثال على التوليد المباشر:

```text
/tool music_generate prompt="هيب هوب lo-fi حالم مع ملمس فينيل ومطر خفيف" instrumental=true
```

## معلمات الأداة المضمّنة

| المعلمة           | النوع     | الوصف                                                                                  |
| ----------------- | -------- | -------------------------------------------------------------------------------------- |
| `prompt`          | string   | مطالبة توليد الموسيقى (مطلوبة عند `action: "generate"`)                               |
| `action`          | string   | `"generate"` (الافتراضي)، أو `"status"` لمهمة الجلسة الحالية، أو `"list"` لفحص المزوّدين |
| `model`           | string   | تجاوز للمزوّد/النموذج، مثل `google/lyria-3-pro-preview` أو `comfy/workflow`            |
| `lyrics`          | string   | كلمات أغنية اختيارية عندما يدعم المزوّد إدخال كلمات صريحًا                            |
| `instrumental`    | boolean  | طلب إخراج آلي فقط عندما يدعمه المزوّد                                                  |
| `image`           | string   | مسار أو URL لصورة مرجعية واحدة                                                         |
| `images`          | string[] | صور مرجعية متعددة (حتى 10)                                                             |
| `durationSeconds` | number   | المدة المستهدفة بالثواني عندما يدعم المزوّد تلميحات المدة                              |
| `timeoutMs`       | number   | مهلة اختيارية لطلب المزوّد بالمللي ثانية                                               |
| `format`          | string   | تلميح تنسيق الإخراج (`mp3` أو `wav`) عندما يدعمه المزوّد                               |
| `filename`        | string   | تلميح اسم ملف الإخراج                                                                  |

لا تدعم جميع المزوّدات جميع المعلمات. وما يزال OpenClaw يتحقق من الحدود الصارمة
مثل أعداد المدخلات قبل الإرسال. وعندما يدعم المزوّد المدة لكنه
يستخدم حدًا أقصى أقصر من القيمة المطلوبة، يقوم OpenClaw تلقائيًا بتقييدها
إلى أقرب مدة مدعومة. أما التلميحات الاختيارية غير المدعومة فعلًا فتُتجاهل
مع تحذير عندما يتعذر على المزوّد أو النموذج المحدد الالتزام بها.

تبلغ نتائج الأداة عن الإعدادات المطبقة. وعندما يقيّد OpenClaw المدة أثناء
التبديل الاحتياطي بين المزوّدين، تعكس القيمة المعادة `durationSeconds` القيمة المُرسلة، وتعرض `details.normalization.durationSeconds` تعيين القيمة المطلوبة إلى القيمة المطبقة.

## السلوك غير المتزامن للمسار المشترك المعتمد على المزوّدين

- تشغيلات العامل المدعومة بجلسات: تنشئ `music_generate` مهمة في الخلفية، وتعيد استجابة بدء/مهمة فورًا، ثم تنشر المقطع النهائي لاحقًا في رسالة متابعة من العامل.
- منع التكرار: طالما أن تلك المهمة الخلفية ما تزال `queued` أو `running`، فإن استدعاءات `music_generate` اللاحقة في الجلسة نفسها تعيد حالة المهمة بدلًا من بدء توليد آخر.
- البحث عن الحالة: استخدم `action: "status"` لفحص مهمة الموسيقى النشطة المدعومة بالجلسة من دون بدء مهمة جديدة.
- تتبع المهام: استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` لفحص الحالة المصطفة والجارية والنهائية لعملية التوليد.
- تنبيه الإكمال: يحقن OpenClaw حدث إكمال داخليًا مرة أخرى في الجلسة نفسها حتى يتمكن النموذج من كتابة رسالة المتابعة المواجهة للمستخدم بنفسه.
- تلميح المطالبة: تتلقى الأدوار اللاحقة من المستخدم/الأدوار اليدوية في الجلسة نفسها تلميحًا صغيرًا أثناء وقت التشغيل عندما تكون مهمة موسيقى قيد التنفيذ بالفعل حتى لا يستدعي النموذج `music_generate` مرة أخرى بشكل أعمى.
- بديل عدم وجود جلسة: ما تزال السياقات المباشرة/المحلية من دون جلسة عامل حقيقية تعمل بشكل مضمن وتعيد نتيجة الصوت النهائية في الدور نفسه.

### دورة حياة المهمة

ينتقل كل طلب `music_generate` عبر أربع حالات:

1. **queued** -- أُنشئت المهمة وتنتظر أن يقبلها المزوّد.
2. **running** -- يعالجها المزوّد (عادة من 30 ثانية إلى 3 دقائق بحسب المزوّد والمدة).
3. **succeeded** -- أصبح المقطع جاهزًا؛ فيستيقظ العامل وينشره في المحادثة.
4. **failed** -- خطأ من المزوّد أو انتهاء مهلة؛ فيستيقظ العامل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك مهمة موسيقى `queued` أو `running` بالفعل للجلسة الحالية، فإن `music_generate` يعيد حالة المهمة الحالية بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً من دون إطلاق توليد جديد.

## الإعدادات

### اختيار النموذج

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "google/lyria-3-clip-preview",
        fallbacks: ["minimax/music-2.6"],
      },
    },
  },
}
```

### ترتيب اختيار المزوّد

عند توليد الموسيقى، يحاول OpenClaw المزوّدين بهذا الترتيب:

1. معلمة `model` من استدعاء الأداة، إذا حدد العامل واحدة
2. `musicGenerationModel.primary` من الإعدادات
3. `musicGenerationModel.fallbacks` بالترتيب
4. الاكتشاف التلقائي باستخدام القيم الافتراضية للمزوّدين المدعومة بالمصادقة فقط:
   - المزوّد الافتراضي الحالي أولًا
   - ثم بقية مزوّدي توليد الموسيقى المسجلين بترتيب معرّف المزوّد

إذا فشل مزوّد ما، فستُجرَّب المرشحة التالية تلقائيًا. وإذا فشل الجميع، فإن
الخطأ يتضمن تفاصيل من كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد
أن يستخدم توليد الموسيقى فقط الإدخالات الصريحة `model` و`primary` و`fallbacks`.

## ملاحظات حول المزوّدين

- تستخدم Google توليد Lyria 3 الدفعي. ويدعم التدفق المضمّن الحالي
  المطالبة، ونص كلمات اختياري، وصورًا مرجعية اختيارية.
- تستخدم MiniMax نقطة النهاية الدفعية `music_generation`. ويدعم التدفق المضمّن الحالي
  المطالبة، والكلمات الاختيارية، ووضع الآلي، وتوجيه المدة،
  وإخراج mp3.
- دعم ComfyUI مدفوع بسير العمل ويعتمد على الرسم البياني المضبوط إضافة إلى
  تعيين عُقد حقول prompt/output.

## أوضاع قدرات المزوّد

يدعم عقد توليد الموسيقى المشترك الآن تعريفات أوضاع صريحة:

- `generate` للتوليد بالاعتماد على المطالبة فقط
- `edit` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر

يجب أن تفضّل تطبيقات المزوّدين الجديدة كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxTracks: 1,
    supportsLyrics: true,
    supportsFormat: true,
  },
  edit: {
    enabled: true,
    maxTracks: 1,
    maxInputImages: 1,
    supportsFormat: true,
  },
}
```

لا تكفي الحقول القديمة المسطحة مثل `maxInputImages` و`supportsLyrics` و
`supportsFormat` للإعلان عن دعم التحرير. يجب على المزوّدين
التصريح بـ `generate` و`edit` صراحةً حتى تتمكن الاختبارات الحية واختبارات العقود
والأداة المشتركة `music_generate` من التحقق من دعم الأوضاع بشكل حتمي.

## اختيار المسار المناسب

- استخدم المسار المشترك المعتمد على المزوّدين عندما تريد اختيار النموذج، والتبديل الاحتياطي بين المزوّدين، وتدفق المهام/الحالة غير المتزامن المضمّن.
- استخدم مسار Plugin مثل ComfyUI عندما تحتاج إلى رسم بياني مخصص لسير العمل أو إلى مزوّد ليس جزءًا من قدرة الموسيقى المشتركة المضمّنة.
- إذا كنت تصحح سلوكًا خاصًا بـ ComfyUI، فراجع [ComfyUI](/ar/providers/comfy). وإذا كنت تصحح سلوك المزوّدين المشتركين، فابدأ من [Google (Gemini)](/ar/providers/google) أو [MiniMax](/ar/providers/minimax).

## الاختبارات الحية

تغطية حية اختيارية للمزوّدين المشتركين المضمّنين:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/music-generation-providers.live.test.ts
```

غلاف المستودع:

```bash
pnpm test:live:media music
```

يقوم هذا الملف الحي بتحميل متغيرات بيئة المزوّدين المفقودة من `~/.profile`، ويفضّل
مفاتيح API الحية/البيئية على ملفات تعريف المصادقة المخزنة افتراضيًا، ويشغّل
كلًا من تغطية `generate` و`edit` المعلنة عندما يفعّل المزوّد وضع التحرير.

وهذا يعني اليوم:

- `google`: ‏`generate` بالإضافة إلى `edit`
- `minimax`: ‏`generate` فقط
- `comfy`: تغطية حية منفصلة لـ Comfy، وليست ضمن التشغيل المشترك للمزوّدين

تغطية حية اختيارية لمسار الموسيقى المضمّن في ComfyUI:

```bash
OPENCLAW_LIVE_TEST=1 COMFY_LIVE_TEST=1 pnpm test:live -- extensions/comfy/comfy.live.test.ts
```

كما يغطي ملف Comfy الحي أيضًا مهام سير عمل الصور والفيديو الخاصة بـ comfy عندما تكون
تلك الأقسام مضبوطة.
