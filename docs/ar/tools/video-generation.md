يمكن لوكلاء OpenClaw توليد مقاطع فيديو من موجّهات نصية أو صور مرجعية أو مقاطع فيديو موجودة. هناك أربع عشرة واجهة خلفية من المزودين مدعومة، ولكل منها خيارات نماذج وأنماط إدخال ومجموعات ميزات مختلفة. يختار الوكيل المزود المناسب تلقائيًا بناءً على إعداداتك ومفاتيح API المتاحة.

<Note>
لا تظهر أداة `video_generate` إلا عندما يكون هناك مزود واحد على الأقل لتوليد الفيديو متاحًا. إذا لم ترها ضمن أدوات الوكيل، فاضبط مفتاح API لمزوّد أو أعد إعداد `agents.defaults.videoGenerationModel`.
</Note>

يتعامل OpenClaw مع توليد الفيديو على أنه ثلاثة أوضاع وقت تشغيل:

- `generate` لطلبات النص إلى فيديو من دون وسائط مرجعية
- `imageToVideo` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر
- `videoToVideo` عندما يتضمن الطلب مقطع فيديو مرجعيًا واحدًا أو أكثر

يمكن للمزودين دعم أي مجموعة فرعية من هذه الأوضاع. تتحقق الأداة من صحة
الوضع النشط قبل الإرسال وتعرض الأوضاع المدعومة في `action=list`.

## البدء السريع

1. اضبط مفتاح API لأي مزود مدعوم:

```bash
export GEMINI_API_KEY="your-key"
```

2. اختياريًا، ثبّت نموذجًا افتراضيًا:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. اطلب من الوكيل:

> أنشئ فيديو سينمائيًا مدته 5 ثوانٍ لجراد بحر ودود يتزلج على الأمواج عند الغروب.

سيستدعي الوكيل `video_generate` تلقائيًا. لا حاجة إلى قائمة سماح للأدوات.

## ما الذي يحدث عند توليد فيديو

توليد الفيديو غير متزامن. عندما يستدعي الوكيل `video_generate` في جلسة:

1. يرسل OpenClaw الطلب إلى المزوّد ويعيد فورًا معرّف مهمة.
2. يعالج المزوّد المهمة في الخلفية (عادةً من 30 ثانية إلى 5 دقائق حسب المزوّد والدقة).
3. عندما يصبح الفيديو جاهزًا، يوقظ OpenClaw الجلسة نفسها بحدث إكمال داخلي.
4. ينشر الوكيل الفيديو النهائي مرة أخرى في المحادثة الأصلية.

أثناء تنفيذ المهمة، فإن استدعاءات `video_generate` المكررة في الجلسة نفسها تعيد حالة المهمة الحالية بدلًا من بدء توليد آخر. استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` للتحقق من التقدم من CLI.

خارج عمليات الوكيل المعتمدة على الجلسة (على سبيل المثال، استدعاءات الأدوات المباشرة)، تعود الأداة إلى التوليد المضمن وتعيد مسار الوسائط النهائي في الدورة نفسها.

تُحفَظ ملفات الفيديو المولّدة ضمن مساحة تخزين الوسائط المُدارة بواسطة OpenClaw عندما
يعيد المزوّد البيانات الثنائية. ويتبع الحد الافتراضي لحفظ الفيديو المولّد حد وسائط الفيديو،
ويرفع `agents.defaults.mediaMaxMb` هذا الحد لعمليات التصيير الأكبر.
وعندما يعيد المزوّد أيضًا عنوان URL مستضافًا للإخراج، يمكن لـ OpenClaw تسليم ذلك العنوان
بدلًا من فشل المهمة إذا رفض الحفظ المحلي ملفًا كبير الحجم.

### دورة حياة المهمة

تنتقل كل طلبات `video_generate` عبر أربع حالات:

1. **queued** -- تم إنشاء المهمة، وهي تنتظر قبولها من قبل المزوّد.
2. **running** -- يعالج المزوّد المهمة (عادةً من 30 ثانية إلى 5 دقائق حسب المزوّد والدقة).
3. **succeeded** -- أصبح الفيديو جاهزًا؛ يستيقظ الوكيل وينشره في المحادثة.
4. **failed** -- خطأ من المزوّد أو انتهاء مهلة؛ يستيقظ الوكيل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك مهمة فيديو بحالة `queued` أو `running` بالفعل للجلسة الحالية، فإن `video_generate` يعيد حالة المهمة الحالية بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً من دون تشغيل توليد جديد.

### مطلوبة

### مدخلات المحتوى

| المعلمة     | النوع    | الوصف                                                                                                                                |
| ----------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `image`      | string   | صورة مرجعية واحدة (مسار أو URL)                                                                                                      |
| `images`     | string[] | عدة صور مرجعية (حتى 9)                                                                                                               |
| `imageRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الصور المدمجة. القيم القياسية: `first_frame` و`last_frame` و`reference_image`   |
| `video`      | string   | مقطع فيديو مرجعي واحد (مسار أو URL)                                                                                                  |
| `videos`     | string[] | عدة مقاطع فيديو مرجعية (حتى 4)                                                                                                       |
| `videoRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الفيديو المدمجة. القيمة القياسية: `reference_video`                              |
| `audioRef`   | string   | مرجع صوتي واحد (مسار أو URL). يُستخدم مثلًا للموسيقى الخلفية أو المرجع الصوتي عندما يدعم المزوّد مدخلات الصوت                      |
| `audioRefs`  | string[] | عدة مراجع صوتية (حتى 3)                                                                                                              |
| `audioRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الصوت المدمجة. القيمة القياسية: `reference_audio`                                |

يتم تمرير تلميحات الأدوار إلى المزوّد كما هي. تأتي القيم القياسية من
اتحاد `VideoGenerationAssetRole` لكن قد يقبل المزوّدون سلاسل أدوار إضافية.
يجب ألا تحتوي مصفوفات `*Roles` على عدد عناصر أكبر من
قائمة المراجع المقابلة؛ وتفشل أخطاء الزيادة أو النقصان بعنصر واحد برسالة خطأ واضحة.
استخدم سلسلة فارغة لترك خانة غير مضبوطة.

### عناصر التحكم في النمط

| المعلمة          | النوع   | الوصف                                                                                  |
| ---------------- | ------- | -------------------------------------------------------------------------------------- |
| `aspectRatio`     | string  | `1:1` أو `2:3` أو `3:2` أو `3:4` أو `4:3` أو `4:5` أو `5:4` أو `9:16` أو `16:9` أو `21:9` أو `adaptive` |
| `resolution`      | string  | `480P` أو `720P` أو `768P` أو `1080P`                                                  |
| `durationSeconds` | number  | المدة المستهدفة بالثواني (تُقرّب إلى أقرب قيمة يدعمها المزوّد)                         |
| `size`            | string  | تلميح للحجم عندما يدعمه المزوّد                                                          |
| `audio`           | boolean | تفعيل الصوت المولّد في الإخراج عندما يكون مدعومًا. وهو مختلف عن `audioRef*` (المدخلات) |
| `watermark`       | boolean | تبديل العلامة المائية الخاصة بالمزوّد عندما تكون مدعومة                                 |

`adaptive` هي قيمة حارسة خاصة بالمزوّد: يتم تمريرها كما هي إلى
المزوّدين الذين يعلنون `adaptive` ضمن قدراتهم (مثلًا تستخدمها BytePlus
Seedance لاكتشاف النسبة تلقائيًا من أبعاد
الصورة المدخلة). أما المزوّدون الذين لا يعلنون عنها فيعرضون القيمة عبر
`details.ignoredOverrides` في نتيجة الأداة بحيث يكون تجاهلها مرئيًا.

### متقدمة

| المعلمة          | النوع  | الوصف                                                                                                                                                                                                                                                                                                                                 |
| ---------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `action`          | string | `"generate"` (الافتراضي) أو `"status"` أو `"list"`                                                                                                                                                                                                                                                                                   |
| `model`           | string | تجاوز المزوّد/النموذج (مثل `runway/gen4.5`)                                                                                                                                                                                                                                                                                          |
| `filename`        | string | تلميح لاسم ملف الإخراج                                                                                                                                                                                                                                                                                                                |
| `timeoutMs`       | number | مهلة اختيارية لطلب المزوّد بالمللي ثانية                                                                                                                                                                                                                                                                                             |
| `providerOptions` | object | خيارات خاصة بالمزوّد على شكل كائن JSON (مثل `{"seed": 42, "draft": true}`). تتحقق المزوّدات التي تعلن عن مخطط typed من صحة المفاتيح والأنواع؛ بينما تؤدي المفاتيح غير المعروفة أو عدم التطابق إلى تخطي المرشح أثناء التبديل الاحتياطي. أما المزوّدون الذين لا يعلنون عن مخطط فيتلقون الخيارات كما هي. شغّل `video_generate action=list` لمعرفة ما يقبله كل مزوّد |

لا تدعم جميع المزوّدين جميع المعلمات. يقوم OpenClaw بالفعل بتوحيد المدة إلى أقرب قيمة يدعمها المزوّد، كما يعيد أيضًا تعيين تلميحات الهندسة المترجمة مثل تحويل الحجم إلى نسبة أبعاد عندما يعرّض مزوّد احتياطي سطح تحكم مختلفًا. ويتم تجاهل التجاوزات غير المدعومة حقًا على أساس أفضل جهد مع الإبلاغ عنها كتحذيرات في نتيجة الأداة. أما حدود القدرات الصارمة (مثل وجود عدد كبير جدًا من المدخلات المرجعية) فتفشل قبل الإرسال.

تعرض نتائج الأداة الإعدادات المطبقة. وعندما يعيد OpenClaw تعيين المدة أو الهندسة أثناء التبديل الاحتياطي بين المزوّدين، فإن القيم المعادة `durationSeconds` و`size` و`aspectRatio` و`resolution` تعكس ما تم إرساله، بينما تلتقط `details.normalization` التحويل من المطلوب إلى المطبق.

تحدد المدخلات المرجعية أيضًا وضع وقت التشغيل:

- بدون وسائط مرجعية: `generate`
- أي صورة مرجعية: `imageToVideo`
- أي فيديو مرجعي: `videoToVideo`
- لا تغيّر المدخلات الصوتية المرجعية الوضع المحسوم؛ بل تُطبَّق فوق أي وضع تحدده المراجع الصورية/الفيديوية، ولا تعمل إلا مع المزوّدين الذين يعلنون `maxInputAudios`

لا يُعد المزج بين الصور المرجعية ومقاطع الفيديو المرجعية سطح قدرات مشتركة مستقرًا.
يُفضّل استخدام نوع مرجعي واحد في كل طلب.

#### التبديل الاحتياطي والخيارات typed

تُطبّق بعض فحوصات القدرات في طبقة التبديل الاحتياطي بدلًا من
حدود الأداة حتى يتمكن الطلب الذي يتجاوز حدود المزوّد الأساسي
من العمل على مزوّد احتياطي قادر:

- إذا كان المرشح النشط لا يعلن عن `maxInputAudios` (أو يعلن عنه على أنه
  `0`)، فسيتم تخطيه عندما يحتوي الطلب على مراجع صوتية، ثم
  تتم تجربة المرشح التالي.
- إذا كانت قيمة `maxDurationSeconds` للمرشح النشط أقل من
  قيمة `durationSeconds` المطلوبة ولم يعلن المرشح عن قائمة
  `supportedDurationSeconds`، فسيتم تخطيه.
- إذا احتوى الطلب على `providerOptions` وكان المرشح النشط
  يعلن صراحةً عن مخطط typed لـ `providerOptions`، فسيتم
  تخطي المرشح عندما لا تكون المفاتيح الموردة ضمن المخطط أو عندما لا
  تتطابق أنواع القيم. أما المزوّدون الذين لم يعلنوا بعد عن مخطط فيتلقون
  الخيارات كما هي (تمرير متوافق مع الإصدارات السابقة). ويمكن للمزوّد
  إلغاء جميع خيارات المزوّد صراحةً عبر إعلان مخطط فارغ
  (`capabilities.providerOptions: {}`)، مما يؤدي إلى نفس التخطي الذي يحدث عند
  عدم تطابق النوع.

يتم تسجيل أول سبب للتخطي في الطلب عند المستوى `warn` حتى يرى المشغّلون
متى تم تجاوز مزوّدهم الأساسي؛ أما عمليات التخطي اللاحقة فتُسجّل عند
المستوى `debug` للحفاظ على هدوء سلاسل التبديل الاحتياطي الطويلة. وإذا تم تخطي كل المرشحين،
فإن الخطأ المجمّع يتضمن سبب التخطي لكل واحد منهم.

- **generate** (الافتراضي) -- أنشئ فيديو من الموجّه المعطى والمدخلات المرجعية الاختيارية.
- **status** -- تحقّق من حالة مهمة الفيديو الجارية للجلسة الحالية من دون بدء توليد آخر.
- **list** -- اعرض المزوّدين والنماذج المتاحة وقدراتها.

## اختيار النموذج

عند توليد فيديو، يحل OpenClaw النموذج بهذا الترتيب:

1. **معلمة الأداة `model`** -- إذا حددها الوكيل في الاستدعاء.
2. **`videoGenerationModel.primary`** -- من الإعدادات.
3. **`videoGenerationModel.fallbacks`** -- تتم تجربتها بالترتيب.
4. **الاكتشاف التلقائي** -- يستخدم المزوّدين الذين لديهم مصادقة صالحة، بدءًا من المزوّد الافتراضي الحالي، ثم بقية المزوّدين بالترتيب الأبجدي.

إذا فشل مزوّد ما، تتم تجربة المرشح التالي تلقائيًا. وإذا فشل جميع المرشحين، فسيتضمن الخطأ تفاصيل من كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد
أن يستخدم توليد الفيديو فقط الإدخالات الصريحة `model` و`primary` و`fallbacks`.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

## أوضاع قدرات المزوّد

يتيح عقد توليد الفيديو المشترك الآن للمزوّدين إعلان قدرات خاصة بكل وضع
بدلًا من الاكتفاء بحدود إجمالية مسطحة فقط. يجب على
تنفيذات المزوّدين الجديدة تفضيل كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

لا تكفي الحقول الإجمالية المسطحة مثل `maxInputImages` و`maxInputVideos`
للإعلان عن دعم أوضاع التحويل. يجب على المزوّدين إعلان
`generate` و`imageToVideo` و`videoToVideo` صراحةً حتى تتمكن الاختبارات المباشرة،
واختبارات العقد، وأداة `video_generate` المشتركة من التحقق من دعم الأوضاع
بشكل حتمي.

## الاختبارات المباشرة

تغطية مباشرة اختيارية للمزوّدين المضمّنين المشتركين:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

غلاف المستودع:

```bash
pnpm test:live:media video
```

يقوم هذا الملف المباشر بتحميل متغيرات env المفقودة للمزوّد من `~/.profile`،
ويفضّل مفاتيح API المباشرة/الموجودة في env على ملفات تعريف المصادقة المخزنة افتراضيًا،
ويشغّل اختبار smoke آمنًا للإصدارات افتراضيًا:

- `generate` لكل مزوّد غير FAL ضمن الفحص
- موجّه جراد بحر مدته ثانية واحدة
- حد عمليات لكل مزوّد من `OPENCLAW_LIVE_VIDEO_GENERATION_TIMEOUT_MS`
  (الافتراضي `180000`)

يكون FAL اختياريًا لأن زمن تأخر قائمة الانتظار على جانب المزوّد قد يهيمن على وقت الإصدار:

```bash
pnpm test:live:media video --video-providers fal
```

اضبط `OPENCLAW_LIVE_VIDEO_GENERATION_FULL_MODES=1` لتشغيل أوضاع التحويل المعلنة أيضًا
التي يمكن للفحص المشترك ممارستها بأمان باستخدام وسائط محلية:

- `imageToVideo` عندما تكون `capabilities.imageToVideo.enabled`
- `videoToVideo` عندما تكون `capabilities.videoToVideo.enabled` ويقبل المزوّد/النموذج
  إدخال الفيديو المحلي المدعوم بالمخزن المؤقت في الفحص المشترك

يغطي المسار المباشر المشترك `videoToVideo` اليوم:

- `runway` فقط عندما تختار `runway/gen4_aleph`

## الإعدادات

اضبط نموذج توليد الفيديو الافتراضي في إعدادات OpenClaw الخاصة بك:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

أو عبر CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
