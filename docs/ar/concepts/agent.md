يشغّل OpenClaw **وقت تشغيل وكيلًا مضمّنًا واحدًا** — عملية وكيل واحدة لكل
Gateway، مع مساحة العمل الخاصة بها، وملفات التهيئة الابتدائية، ومخزن الجلسات الخاص بها. وتغطي هذه الصفحة
عقد وقت التشغيل هذا: ما الذي يجب أن تحتويه مساحة العمل، وما الملفات التي يتم حقنها، وكيف تُهيَّأ الجلسات انطلاقًا منها.

## مساحة العمل (مطلوبة)

يستخدم OpenClaw دليل مساحة عمل وكيل واحدًا (`agents.defaults.workspace`) باعتباره **الدليل العامل الوحيد** (`cwd`) للأدوات والسياق.

الموصى به: استخدم `openclaw setup` لإنشاء `~/.openclaw/openclaw.json` إذا كان مفقودًا وتهيئة ملفات مساحة العمل.

دليل تخطيط مساحة العمل الكامل + النسخ الاحتياطي: [مساحة عمل الوكيل](/ar/concepts/agent-workspace)

إذا كان `agents.defaults.sandbox` مفعّلًا، فيمكن للجلسات غير الرئيسية تجاوز ذلك باستخدام
مساحات عمل لكل جلسة ضمن `agents.defaults.sandbox.workspaceRoot` (راجع
[إعدادات Gateway](/ar/gateway/configuration)).

## ملفات التهيئة الابتدائية (المحقونة)

داخل `agents.defaults.workspace`، يتوقع OpenClaw هذه الملفات القابلة للتحرير من قبل المستخدم:

- `AGENTS.md` — تعليمات التشغيل + “الذاكرة”
- `SOUL.md` — الشخصية، والحدود، والأسلوب
- `TOOLS.md` — ملاحظات الأدوات التي يحتفظ بها المستخدم (مثل `imsg` و`sag` والاصطلاحات)
- `BOOTSTRAP.md` — طقس تشغيل أولي لمرة واحدة (يُحذف بعد الاكتمال)
- `IDENTITY.md` — اسم الوكيل/طابعه/emoji
- `USER.md` — ملف تعريف المستخدم + أسلوب المخاطبة المفضل

في الدورة الأولى من جلسة جديدة، يحقن OpenClaw محتويات هذه الملفات مباشرة في سياق الوكيل.

يتم تخطي الملفات الفارغة. وتُقص الملفات الكبيرة وتُختصر مع علامة توضيحية حتى تبقى المطالبات خفيفة (اقرأ الملف للحصول على المحتوى الكامل).

إذا كان أحد الملفات مفقودًا، يحقن OpenClaw سطر علامة واحدًا يفيد بأن الملف مفقود (وسيُنشئ `openclaw setup` قالبًا افتراضيًا آمنًا).

لا يُنشأ `BOOTSTRAP.md` إلا من أجل **مساحة عمل جديدة تمامًا** (عند عدم وجود أي ملفات تهيئة ابتدائية أخرى). وإذا حذفته بعد إكمال الطقس، فلا ينبغي إعادة إنشائه في عمليات إعادة التشغيل اللاحقة.

لتعطيل إنشاء ملفات التهيئة الابتدائية بالكامل (لمساحات العمل المجهزة مسبقًا)، اضبط:

```json5
{ agents: { defaults: { skipBootstrap: true } } }
```

## الأدوات المضمنة

تكون الأدوات الأساسية (read/exec/edit/write وأدوات النظام ذات الصلة) متاحة دائمًا،
مع مراعاة سياسة الأدوات. ويكون `apply_patch` اختياريًا وتتحكم به
`tools.exec.applyPatch`. ولا يتحكم `TOOLS.md` في الأدوات الموجودة؛ بل هو
إرشاد لكيفية _رغبتك_ في استخدامها.

## Skills

يحمّل OpenClaw Skills من هذه المواقع (الأولوية الأعلى أولًا):

- مساحة العمل: `<workspace>/skills`
- Skills وكيل المشروع: `<workspace>/.agents/skills`
- Skills الوكيل الشخصية: `~/.agents/skills`
- المُدارة/المحلية: `~/.openclaw/skills`
- المضمنة (المشحونة مع التثبيت)
- مجلدات Skills إضافية: `skills.load.extraDirs`

يمكن تقييد Skills عبر الإعدادات/البيئة (راجع `skills` في [إعدادات Gateway](/ar/gateway/configuration)).

## حدود وقت التشغيل

بُني وقت تشغيل الوكيل المضمّن على نواة وكيل Pi (النماذج، والأدوات، ومسار المطالبات).
إدارة الجلسات، والاكتشاف، وربط الأدوات، وتسليم القنوات هي طبقات يملكها OpenClaw فوق هذه النواة.

## الجلسات

تُخزَّن transcripts الجلسات بصيغة JSONL في:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

يكون معرّف الجلسة ثابتًا ويختاره OpenClaw.
ولا تتم قراءة مجلدات الجلسات القديمة من الأدوات الأخرى.

## التوجيه أثناء البث

عندما يكون وضع الطابور هو `steer`، تُحقن الرسائل الواردة في التشغيل الحالي.
ويُسلَّم التوجيه الموضوع في الطابور **بعد انتهاء دورة المساعد الحالية من
تنفيذ استدعاءات أدواتها**، وقبل استدعاء LLM التالي. ولم يعد التوجيه يتخطى
استدعاءات الأدوات المتبقية من رسالة المساعد الحالية؛ بل يحقن الرسالة الموضوعة في الطابور عند حد النموذج التالي بدلًا من ذلك.

عندما يكون وضع الطابور `followup` أو `collect`، تُحتجز الرسائل الواردة حتى
تنتهي الدورة الحالية، ثم تبدأ دورة وكيل جديدة بالحمولات الموضوعة في الطابور. راجع
[الطابور](/ar/concepts/queue) لمعرفة سلوك الوضع + إزالة الارتداد/الحد الأقصى.

يرسل بث الكتل كتل المساعد المكتملة بمجرد انتهائها؛ وهو
**معطّل افتراضيًا** (`agents.defaults.blockStreamingDefault: "off"`).
اضبط الحد عبر `agents.defaults.blockStreamingBreak` (`text_end` مقابل `message_end`؛ والافتراضي هو text_end).
وتحكم في تجزئة الكتل المرنة عبر `agents.defaults.blockStreamingChunk` (الافتراضي
800–1200 حرفًا؛ مع تفضيل فواصل الفقرات، ثم الأسطر الجديدة؛ والجمل أخيرًا).
وقم بدمج المقاطع المتدفقة عبر `agents.defaults.blockStreamingCoalesce` لتقليل
الرسائل المزعجة أحادية السطر (دمج قائم على الخمول قبل الإرسال). وتتطلب القنوات غير
Telegram تفعيل `*.blockStreaming: true` صراحةً لتفعيل ردود الكتل.
وتُرسل ملخصات الأدوات المطوّلة عند بدء الأداة (من دون إزالة ارتداد)؛ وتبث Control UI
مخرجات الأدوات عبر أحداث الوكيل عندما تكون متاحة.
لمزيد من التفاصيل: [البث + التجزئة](/ar/concepts/streaming).

## مراجع النماذج

تُحلَّل مراجع النماذج في الإعدادات (مثل `agents.defaults.model` و`agents.defaults.models`) بالتقسيم عند **أول** `/`.

- استخدم `provider/model` عند ضبط النماذج.
- إذا كان معرّف النموذج نفسه يحتوي على `/` (بنمط OpenRouter)، فضمّن بادئة المزوّد (مثال: `openrouter/moonshotai/kimi-k2`).
- إذا حذفت المزوّد، فسيحاول OpenClaw أولًا اسمًا مستعارًا، ثم تطابقًا فريدًا
  لمزوّد مُهيأ لذلك معرّف النموذج الدقيق، وبعدها فقط سيعود
  إلى المزوّد الافتراضي المُهيأ. وإذا لم يعد ذلك المزوّد يعرّض
  النموذج الافتراضي المُهيأ، فسيعود OpenClaw إلى أول
  مزوّد/نموذج مُهيأ بدلًا من إظهار قيمة افتراضية قديمة لمزوّد تمت إزالته.
