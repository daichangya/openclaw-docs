# أدوات الجلسة

يمنح OpenClaw الوكلاء أدوات للعمل عبر الجلسات، وفحص الحالة،
وتنسيق الوكلاء الفرعيين.

## الأدوات المتاحة

| الأداة             | ما الذي تفعله                                                              |
| ------------------ | -------------------------------------------------------------------------- |
| `sessions_list`    | عرض الجلسات مع عوامل تصفية اختيارية (النوع، التسمية، الوكيل، الحداثة، المعاينة) |
| `sessions_history` | قراءة النص الكامل لجلسة محددة                                              |
| `sessions_send`    | إرسال رسالة إلى جلسة أخرى والانتظار اختياريًا                              |
| `sessions_spawn`   | إنشاء جلسة وكيل فرعي معزولة للعمل في الخلفية                              |
| `sessions_yield`   | إنهاء الدورة الحالية وانتظار نتائج متابعة من الوكلاء الفرعيين              |
| `subagents`        | عرض الوكلاء الفرعيين الذين أُنشئوا لهذه الجلسة أو توجيههم أو إنهاؤهم        |
| `session_status`   | عرض بطاقة على نمط `/status` وضبط تجاوز نموذج لكل جلسة اختياريًا            |

## عرض الجلسات وقراءتها

تعيد `sessions_list` الجلسات مع مفتاحها، و`agentId`، والنوع، والقناة، والنموذج،
وأعداد الرموز، والطوابع الزمنية. يمكنك التصفية حسب النوع (`main` أو `group` أو `cron` أو `hook` أو `node`)،
أو `label` المطابقة تمامًا، أو `agentId` المطابقة تمامًا، أو نص البحث، أو الحداثة
(`activeMinutes`). وعندما تحتاج إلى فرز على نمط صندوق البريد، يمكنها أيضًا طلب
عنوان مشتق ضمن نطاق الرؤية، أو مقتطف معاينة للرسالة الأخيرة، أو رسائل حديثة
محدودة في كل صف. ولا تُنتج العناوين المشتقة والمعاينات إلا للجلسات التي يستطيع
المستدعي رؤيتها أصلًا ضمن سياسة الرؤية المضبوطة لأدوات الجلسة،
بحيث تبقى الجلسات غير المرتبطة مخفية.

تجلب `sessions_history` نص المحادثة الكامل لجلسة محددة.
افتراضيًا، تُستبعد نتائج الأدوات -- مرّر `includeTools: true` لرؤيتها.
ويكون العرض المعاد محدودًا ومصفّى أمنيًا عن قصد:

- يُطبَّع نص المساعد قبل الاستدعاء:
  - تُزال وسوم التفكير
  - تُزال كتل البنية `\<relevant-memories>` / `\<relevant_memories>`
  - تُزال كتل XML النصية الصريحة الخاصة بحمولات استدعاء الأدوات مثل `<tool_call>...</tool_call>`،
    و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و
    `<function_calls>...</function_calls>`، بما في ذلك الحمولات المبتورة
    التي لا تُغلَق بشكل نظيف
  - تُزال البنية المخفّضة الخاصة باستدعاء/نتيجة الأداة مثل `[Tool Call: ...]`،
    و`[Tool Result ...]`، و`[Historical context ...]`
  - تُزال رموز التحكم الخاصة بالنموذج المسربة مثل `<|assistant|>`، ورموز
    ASCII الأخرى على النمط `<|...|>`، والمتغيرات كاملة العرض `<｜...｜>`
  - تُزال XML استدعاء الأدوات غير الصحيحة الخاصة بـ MiniMax مثل `<invoke ...>` /
    `</minimax:tool_call>`
- يُحجب النص المشابه لبيانات الاعتماد/الرموز قبل إرجاعه
- تُقتطع الكتل النصية الطويلة
- قد تُسقط السجلات الكبيرة جدًا الصفوف الأقدم أو تستبدل صفًا كبيرًا جدًا بـ
  `[sessions_history omitted: message too large]`
- تُبلغ الأداة عن أعلام ملخّصة مثل `truncated` و`droppedMessages` و
  `contentTruncated` و`contentRedacted` و`bytes`

تقبل كلتا الأداتين إما **مفتاح جلسة** (مثل `"main"`) أو **معرّف جلسة**
من استدعاء عرض سابق.

إذا كنت تحتاج إلى النص الكامل الدقيق بايتًا ببايت، فافحص ملف النص الكامل على
القرص بدلًا من التعامل مع `sessions_history` على أنه تفريغ خام.

## إرسال رسائل عبر الجلسات

تسلّم `sessions_send` رسالة إلى جلسة أخرى ويمكنها الانتظار اختياريًا
للرد:

- **إرسال من دون انتظار:** اضبط `timeoutSeconds: 0` لإدراج الرسالة في الطابور والعودة
  فورًا.
- **انتظار الرد:** اضبط مهلة واحصل على الرد مضمنًا.

بعد أن ترد الجلسة الهدف، يمكن لـ OpenClaw تشغيل **حلقة رد متبادل**
يتناوب فيها الوكلاء تبادل الرسائل (حتى 5 دورات). ويمكن للوكيل الهدف الرد
بـ `REPLY_SKIP` للتوقف مبكرًا.

## مساعدات الحالة والتنسيق

تمثل `session_status` الأداة الخفيفة المكافئة لـ `/status` للجلسة الحالية
أو لجلسة أخرى مرئية. وهي تعرض الاستخدام، والوقت، وحالة النموذج/وقت التشغيل،
وسياق المهام في الخلفية المرتبط عند وجوده. ومثل `/status`، يمكنها استكمال
عدادات الرموز/الذاكرة المخبئية القليلة من أحدث إدخال استخدام في النص الكامل،
كما أن `model=default` يمسح أي تجاوز لكل جلسة.

تنهي `sessions_yield` الدورة الحالية عمدًا بحيث تكون الرسالة التالية
هي حدث المتابعة الذي تنتظره. استخدمها بعد إنشاء الوكلاء الفرعيين عندما
تريد أن تصل نتائج الإكمال كرسالة تالية بدلًا من بناء حلقات استطلاع.

تمثل `subagents` مساعد مستوى التحكم للوكلاء الفرعيين الموجودين مسبقًا في OpenClaw.
وهي تدعم:

- `action: "list"` لفحص التشغيلات النشطة/الحديثة
- `action: "steer"` لإرسال توجيهات متابعة إلى جلسة فرعية قيد التشغيل
- `action: "kill"` لإيقاف جلسة فرعية واحدة أو `all`

## إنشاء الوكلاء الفرعيين

تنشئ `sessions_spawn` جلسة معزولة لمهمة في الخلفية. وهي دائمًا
غير حاجزة -- إذ تعود فورًا مع `runId` و`childSessionKey`.

الخيارات الأساسية:

- `runtime: "subagent"` (الافتراضي) أو `"acp"` لوكلاء harness الخارجيين.
- تجاوزات `model` و`thinking` للجلسة الفرعية.
- `thread: true` لربط الإنشاء بسلسلة دردشة (Discord أو Slack أو غيرهما).
- `sandbox: "require"` لفرض sandboxing على الجلسة الفرعية.

لا تحصل الوكلاء الفرعيون النهائيون افتراضيًا على أدوات الجلسة. وعندما
تكون `maxSpawnDepth >= 2`، تتلقى الوكلاء الفرعيون المنسقون عند العمق 1 بالإضافة إلى ذلك
الأدوات `sessions_spawn` و`subagents` و`sessions_list` و`sessions_history` حتى
تتمكن من إدارة أبنائها. ولا تزال التشغيلات النهائية لا تحصل على
أدوات تنسيق تكرارية.

بعد الإكمال، تنشر خطوة إعلان النتيجة إلى قناة الطالب.
يحافظ تسليم الإكمال على توجيه السلسلة/الموضوع المرتبط عند توفره،
وإذا كان أصل الإكمال يحدد قناة فقط، فلا يزال بإمكان OpenClaw إعادة استخدام
المسار المخزن في جلسة الطالب (`lastChannel` / `lastTo`) من أجل
التسليم المباشر.

للاطلاع على السلوك الخاص بـ ACP، راجع [وكلاء ACP](/ar/tools/acp-agents).

## الرؤية

تكون أدوات الجلسة محددة النطاق للحد مما يمكن للوكيل رؤيته:

| المستوى | النطاق                                  |
| ------- | --------------------------------------- |
| `self`  | الجلسة الحالية فقط                      |
| `tree`  | الجلسة الحالية + الوكلاء الفرعيون الذين تم إنشاؤهم |
| `agent` | جميع الجلسات لهذا الوكيل                |
| `all`   | جميع الجلسات (عبر الوكلاء إذا كان ذلك مضبوطًا) |

الافتراضي هو `tree`. وتُقيَّد الجلسات المعزولة في sandbox إلى `tree` بغض النظر عن
الإعداد.
