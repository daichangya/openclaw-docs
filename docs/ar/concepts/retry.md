# سياسة إعادة المحاولة

## الأهداف

- تنفيذ إعادة المحاولة لكل طلب HTTP، وليس لكل تدفق متعدد الخطوات.
- الحفاظ على الترتيب عبر إعادة محاولة الخطوة الحالية فقط.
- تجنب تكرار العمليات غير المتماثلة.

## القيم الافتراضية

- عدد المحاولات: 3
- الحد الأقصى للتأخير: 30000 ms
- التذبذب: 0.1 ‏(10 بالمئة)
- القيم الافتراضية حسب المزوّد:
  - الحد الأدنى للتأخير في Telegram: ‏400 ms
  - الحد الأدنى للتأخير في Discord: ‏500 ms

## السلوك

### مزوّدو النماذج

- يتيح OpenClaw لحزم SDK الخاصة بالمزوّدات التعامل مع إعادات المحاولة القصيرة العادية.
- بالنسبة إلى حزم SDK المبنية على Stainless مثل Anthropic وOpenAI، يمكن أن تتضمن
  الاستجابات القابلة لإعادة المحاولة (`408` و`409` و`429` و`5xx`) القيم `retry-after-ms` أو
  `retry-after`. وعندما يكون وقت الانتظار هذا أطول من 60 ثانية، يحقن OpenClaw
  القيمة `x-should-retry: false` بحيث تعرض SDK الخطأ فورًا ويمكن لآلية
  التبديل بين النماذج أن تنتقل إلى auth profile آخر أو نموذج احتياطي.
- تجاوز الحد الأقصى عبر `OPENCLAW_SDK_RETRY_MAX_WAIT_SECONDS=<seconds>`.
  اضبطه على `0` أو `false` أو `off` أو `none` أو `disabled` للسماح لحزم SDK باحترام فترات
  السكون الطويلة `Retry-After` داخليًا.

### Discord

- يعيد المحاولة فقط عند أخطاء حد المعدّل (HTTP 429).
- يستخدم Discord `retry_after` عند توفره، وإلا فيستخدم exponential backoff.

### Telegram

- يعيد المحاولة عند الأخطاء المؤقتة (429، والمهلة، والاتصال/إعادة الضبط/الإغلاق، وعدم التوفر المؤقت).
- يستخدم `retry_after` عند توفره، وإلا فيستخدم exponential backoff.
- لا تُعاد محاولة أخطاء تحليل Markdown؛ بل تعود إلى نص عادي.

## الإعداد

اضبط سياسة إعادة المحاولة لكل مزوّد في `~/.openclaw/openclaw.json`:

```json5
{
  channels: {
    telegram: {
      retry: {
        attempts: 3,
        minDelayMs: 400,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
    discord: {
      retry: {
        attempts: 3,
        minDelayMs: 500,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
  },
}
```

## ملاحظات

- تنطبق إعادات المحاولة لكل طلب على حدة (إرسال رسالة، أو رفع وسائط، أو تفاعل، أو استطلاع، أو ملصق).
- لا تعيد التدفقات المركبة محاولة الخطوات المكتملة.
