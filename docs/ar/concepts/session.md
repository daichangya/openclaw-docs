# إدارة الجلسات

ينظم OpenClaw المحادثات ضمن **جلسات**. تُوجَّه كل رسالة إلى
جلسة بناءً على مصدرها -- الرسائل الخاصة، ودردشات المجموعات، ومهام Cron، وغير ذلك.

## كيفية توجيه الرسائل

| المصدر          | السلوك                  |
| --------------- | ------------------------- |
| الرسائل الخاصة | جلسة مشتركة افتراضيًا |
| دردشات المجموعات     | معزولة لكل مجموعة        |
| الغرف/القنوات  | معزولة لكل غرفة         |
| مهام Cron       | جلسة جديدة لكل تشغيل     |
| Webhook        | معزولة لكل hook         |

## عزل الرسائل الخاصة

افتراضيًا، تشترك جميع الرسائل الخاصة في جلسة واحدة من أجل الاستمرارية. وهذا مناسب
لإعدادات المستخدم الواحد.

<Warning>
إذا كان بإمكان عدة أشخاص مراسلة وكيلك، ففعّل عزل الرسائل الخاصة. من دونه، يشترك جميع
المستخدمين في سياق المحادثة نفسه -- وستكون رسائل Alice الخاصة مرئية لـ Bob.
</Warning>

**الحل:**

```json5
{
  session: {
    dmScope: "per-channel-peer", // عزل حسب القناة + المرسل
  },
}
```

خيارات أخرى:

- `main` (الافتراضي) -- تشترك جميع الرسائل الخاصة في جلسة واحدة.
- `per-peer` -- عزل حسب المرسل (عبر القنوات).
- `per-channel-peer` -- عزل حسب القناة + المرسل (موصى به).
- `per-account-channel-peer` -- عزل حسب الحساب + القناة + المرسل.

<Tip>
إذا تواصل الشخص نفسه معك من قنوات متعددة، فاستخدم
`session.identityLinks` لربط هوياته بحيث يشترك في جلسة واحدة.
</Tip>

تحقق من إعدادك باستخدام `openclaw security audit`.

## دورة حياة الجلسة

تُعاد استخدام الجلسات حتى تنتهي صلاحيتها:

- **إعادة تعيين يومية** (افتراضيًا) -- جلسة جديدة الساعة 4:00 صباحًا بالتوقيت المحلي على مضيف
  gateway.
- **إعادة تعيين عند الخمول** (اختياريًا) -- جلسة جديدة بعد فترة من عدم النشاط. اضبط
  `session.reset.idleMinutes`.
- **إعادة تعيين يدوية** -- اكتب `/new` أو `/reset` في الدردشة. كما أن `/new <model>`
  يبدّل النموذج أيضًا.

عند ضبط كل من إعادة التعيين اليومية وعند الخمول، فإن أول ما تنتهي صلاحيته هو الذي يسري.

لا تُقطع الجلسات التي تملك جلسة CLI نشطة مملوكة للمزوّد بواسطة الافتراضي اليومي الضمني.
استخدم `/reset` أو اضبط `session.reset` صراحةً عندما ينبغي أن تنتهي صلاحية تلك الجلسات وفق مؤقت.

## أين توجد الحالة

تعود ملكية كل حالة الجلسة إلى **gateway**. وتستعلم عملاء UI من gateway عن
بيانات الجلسة.

- **المخزن:** `~/.openclaw/agents/<agentId>/sessions/sessions.json`
- **النصوص:** `~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`

## صيانة الجلسة

يضع OpenClaw حدودًا تلقائية لتخزين الجلسات بمرور الوقت. ويعمل افتراضيًا
في وضع `warn` (يبلغ عما سيُنظف). اضبط `session.maintenance.mode`
على `"enforce"` من أجل التنظيف التلقائي:

```json5
{
  session: {
    maintenance: {
      mode: "enforce",
      pruneAfter: "30d",
      maxEntries: 500,
    },
  },
}
```

عاين ذلك باستخدام `openclaw sessions cleanup --dry-run`.

## فحص الجلسات

- `openclaw status` -- مسار مخزن الجلسة وآخر نشاط.
- `openclaw sessions --json` -- جميع الجلسات (رشّح باستخدام `--active <minutes>`).
- `/status` في الدردشة -- استخدام السياق، والنموذج، والمفاتيح.
- `/context list` -- ما الموجود في مطالبة النظام.
