# `openclaw cron`

إدارة وظائف Cron لجدولة Gateway.

ذو صلة:

- وظائف Cron: [وظائف Cron](/ar/automation/cron-jobs)

نصيحة: شغّل `openclaw cron --help` للحصول على واجهة الأوامر الكاملة.

ملاحظة: يعرض `openclaw cron list` و`openclaw cron show <job-id>` معاينة
لمسار التسليم بعد حله. بالنسبة إلى `channel: "last"`، تعرض المعاينة ما إذا كان
المسار قد تم حله من الجلسة الرئيسية/الحالية أو سيفشل في وضع الإغلاق الآمن.

ملاحظة: تستخدم وظائف `cron add` المعزولة افتراضيًا تسليم `--announce`. استخدم `--no-deliver` للإبقاء على
المخرجات داخلية. يظل `--deliver` اسمًا مستعارًا قديمًا لـ `--announce`.

ملاحظة: تسليم دردشة Cron المعزول مشترك. يمثل `--announce` تسليمًا احتياطيًا
من المشغّل للرد النهائي؛ ويعطّل `--no-deliver` هذا الاحتياطي لكنه
لا يزيل أداة `message` الخاصة بالوكيل عندما يكون مسار الدردشة متاحًا.

ملاحظة: تُحذف وظائف المرة الواحدة (`--at`) بعد النجاح افتراضيًا. استخدم `--keep-after-run` للاحتفاظ بها.

ملاحظة: يدعم `--session` القيم `main` و`isolated` و`current` و`session:<id>`.
استخدم `current` للربط بالجلسة النشطة وقت الإنشاء، أو `session:<id>` من أجل
مفتاح جلسة دائم صريح.

ملاحظة: ينشئ `--session isolated` نصًا/معرّف جلسة جديدًا لكل تشغيل.
يمكن نقل التفضيلات الآمنة وعمليات تجاوز النموذج/المصادقة التي اختارها المستخدم صراحةً، لكن
سياق المحادثة المحيط لا ينتقل: إذ يُعاد تعيين توجيه القناة/المجموعة، وسياسة الإرسال/الاصطفاف،
والتصعيد، والمصدر، وربط وقت تشغيل ACP للتشغيل المعزول الجديد.

ملاحظة: بالنسبة إلى وظائف CLI أحادية التشغيل، تُعامل تواريخ `--at` بدون إزاحة زمنية على أنها UTC ما لم تمرر أيضًا
`--tz <iana>`، الذي يفسّر ذلك الوقت المحلي في المنطقة الزمنية المحددة.

ملاحظة: تستخدم الوظائف المتكررة الآن تراجع إعادة المحاولة الأسي بعد الأخطاء المتتالية (30s ← 1m ← 5m ← 15m ← 60m)، ثم تعود إلى الجدول الطبيعي بعد التشغيل الناجح التالي.

ملاحظة: يعيد `openclaw cron run` الآن الاستجابة بمجرد إدراج التشغيل اليدوي في قائمة التنفيذ. تتضمن
الاستجابات الناجحة `{ ok: true, enqueued: true, runId }`؛ استخدم `openclaw cron runs --id <job-id>` لمتابعة النتيجة النهائية.

ملاحظة: يفرض `openclaw cron run <job-id>` التشغيل افتراضيًا. استخدم `--due` للاحتفاظ
بالسلوك الأقدم "شغّل فقط إذا كان مستحقًا".

ملاحظة: تمنع دورات Cron المعزولة الردود المتقادمة الخاصة بالإقرار فقط. إذا كانت
النتيجة الأولى مجرد تحديث حالة مؤقت ولم يكن هناك تشغيل وكيل فرعي تابع
مسؤول عن الإجابة النهائية، فإن Cron يعيد المطالبة مرة واحدة للحصول على النتيجة الفعلية قبل
التسليم.

ملاحظة: إذا أعاد تشغيل Cron المعزول الرمز الصامت فقط (`NO_REPLY` /
`no_reply`)، فإن Cron يمنع التسليم المباشر إلى الخارج وكذلك مسار الملخص
الاحتياطي في قائمة الانتظار، لذلك لا يتم نشر أي شيء مرة أخرى في الدردشة.

ملاحظة: يستخدم `cron add|edit --model ...` ذلك النموذج المسموح المحدد للوظيفة.
إذا لم يكن النموذج مسموحًا، فسيحذّر Cron ويعود إلى اختيار نموذج الوكيل/الافتراضي
للوظيفة بدلًا من ذلك. تظل سلاسل الرجوع الاحتياطي المضبوطة مطبقة، لكن تجاوز
نموذج عادي بدون قائمة رجوع احتياطي صريحة لكل وظيفة لم يعد يضيف النموذج الأساسي
للوكيل كهدف إعادة محاولة إضافي مخفي.

ملاحظة: أولوية النموذج في Cron المعزول هي: تجاوز Gmail-hook أولًا، ثم
`--model` لكل وظيفة، ثم أي تجاوز مخزّن للنموذج الخاص بجلسة Cron اختاره المستخدم، ثم
الاختيار العادي للوكيل/الافتراضي.

ملاحظة: يتبع الوضع السريع في Cron المعزول اختيار النموذج الحي بعد حله. يُطبّق
`params.fastMode` من إعدادات النموذج افتراضيًا، لكن تجاوز `fastMode` المخزّن
في الجلسة يظل أعلى أولوية من الإعدادات.

ملاحظة: إذا أطلق تشغيل معزول الخطأ `LiveSessionModelSwitchError`، فإن Cron يحفظ
المزوّد/النموذج المحوّلين (وتجاوز ملف تعريف المصادقة المحوّل عند وجوده) للتشغيل
النشط قبل إعادة المحاولة. تكون حلقة إعادة المحاولة الخارجية محدودة بمحاولتي تبديل
بعد المحاولة الأولية، ثم تُجهض بدلًا من الدوران إلى ما لا نهاية.

ملاحظة: تستخدم إشعارات الفشل `delivery.failureDestination` أولًا، ثم
`cron.failureDestination` العام، ثم تعود أخيرًا إلى هدف
الإعلان الأساسي للوظيفة عندما لا تكون هناك وجهة فشل صريحة مضبوطة.

ملاحظة: يتم التحكم في الاحتفاظ/التقليم في الإعدادات:

- `cron.sessionRetention` (الافتراضي `24h`) يقلّم جلسات التشغيلات المعزولة المكتملة.
- `cron.runLog.maxBytes` + `cron.runLog.keepLines` يقلّمان `~/.openclaw/cron/runs/<jobId>.jsonl`.

ملاحظة الترقية: إذا كانت لديك وظائف Cron أقدم من تنسيق التسليم/التخزين الحالي، فشغّل
`openclaw doctor --fix`. يقوم Doctor الآن بتسوية حقول Cron القديمة (`jobId` و`schedule.cron`،
وحقول التسليم ذات المستوى الأعلى بما فيها `threadId` القديم، وأسماء التسليم البديلة `provider` في الحمولة) وينقل
وظائف الرجوع الاحتياطي لـ Webhook البسيطة `notify: true` إلى تسليم Webhook صريح عندما يكون
`cron.webhook` مضبوطًا.

## تعديلات شائعة

حدّث إعدادات التسليم بدون تغيير الرسالة:

```bash
openclaw cron edit <job-id> --announce --channel telegram --to "123456789"
```

تعطيل التسليم لوظيفة معزولة:

```bash
openclaw cron edit <job-id> --no-deliver
```

تمكين سياق تمهيدي خفيف لوظيفة معزولة:

```bash
openclaw cron edit <job-id> --light-context
```

الإعلان إلى قناة محددة:

```bash
openclaw cron edit <job-id> --announce --channel slack --to "channel:C1234567890"
```

إنشاء وظيفة معزولة بسياق تمهيدي خفيف:

```bash
openclaw cron add \
  --name "Lightweight morning brief" \
  --cron "0 7 * * *" \
  --session isolated \
  --message "Summarize overnight updates." \
  --light-context \
  --no-deliver
```

ينطبق `--light-context` على وظائف دور الوكيل المعزولة فقط. بالنسبة إلى تشغيلات Cron، يبقي الوضع الخفيف سياق التمهيد فارغًا بدلًا من حقن مجموعة التمهيد الكاملة لمساحة العمل.

ملاحظة ملكية التسليم:

- تسليم دردشة Cron المعزول مشترك. يمكن للوكيل الإرسال مباشرة باستخدام
  أداة `message` عندما يكون مسار دردشة متاحًا.
- يسلّم `announce` الرد النهائي احتياطيًا فقط عندما لا يرسل الوكيل
  مباشرة إلى الهدف المحلول. ويقوم `webhook` بإرسال الحمولة المكتملة إلى URL.
  ويعطّل `none` تسليم الرجوع الاحتياطي للمشغّل.

## أوامر إدارية شائعة

تشغيل يدوي:

```bash
openclaw cron list
openclaw cron show <job-id>
openclaw cron run <job-id>
openclaw cron run <job-id> --due
openclaw cron runs --id <job-id> --limit 50
```

تتضمن إدخالات `cron runs` بيانات تشخيص التسليم مع هدف Cron المقصود،
والهدف المحلول، وعمليات الإرسال عبر أداة الرسائل، واستخدام الاحتياطي، وحالة التسليم.

إعادة توجيه الوكيل/الجلسة:

```bash
openclaw cron edit <job-id> --agent ops
openclaw cron edit <job-id> --clear-agent
openclaw cron edit <job-id> --session current
openclaw cron edit <job-id> --session "session:daily-brief"
```

تعديلات التسليم:

```bash
openclaw cron edit <job-id> --announce --channel slack --to "channel:C1234567890"
openclaw cron edit <job-id> --best-effort-deliver
openclaw cron edit <job-id> --no-best-effort-deliver
openclaw cron edit <job-id> --no-deliver
```

ملاحظة تسليم الفشل:

- `delivery.failureDestination` مدعوم للوظائف المعزولة.
- يمكن لوظائف الجلسة الرئيسية استخدام `delivery.failureDestination` فقط عندما
  يكون وضع التسليم الأساسي هو `webhook`.
- إذا لم تضبط أي وجهة فشل وكانت الوظيفة تعلن بالفعل إلى
  قناة، فإن إشعارات الفشل تعيد استخدام هدف الإعلان نفسه.
