# `openclaw status`

تشخيصات للقنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` عمليات فحص حية (`probes`) لكل من WhatsApp Web وTelegram وDiscord وSlack وSignal.
- يطبع `--usage` نوافذ استخدام الموفّر الموحّدة على شكل `X% left`.
- يفصل خرج حالة الجلسة بين `Execution:` و`Runtime:`. يشير `Execution` إلى مسار sandbox (`direct`, `docker/*`)، بينما يوضّح `Runtime` ما إذا كانت الجلسة تستخدم `OpenClaw Pi Default` أو `OpenAI Codex` أو backend خاصًا بـ CLI أو backend من ACP مثل `codex (acp/acpx)`. راجع [Agent runtimes](/ar/concepts/agent-runtimes) للاطلاع على الفرق بين الموفّر/النموذج/بيئة التشغيل.
- حقلا `usage_percent` / `usagePercent` الخامّان في MiniMax يمثلان الحصة المتبقية، لذلك يعكسهما OpenClaw قبل العرض؛ وتُعطى الحقول المعتمدة على العدد أولوية عند توفرها. وتفضّل استجابات `model_remains` إدخال chat-model، وتستنتج تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية قليلة البيانات، يمكن لـ `/status` استكمال عدّادات الرموز المميزة وcache من أحدث سجل استخدام في transcript. وتبقى القيم الحية غير الصفرية الموجودة لها الأولوية على قيم الاستكمال من transcript.
- يمكن للاستكمال من transcript أيضًا استعادة تسمية نموذج بيئة التشغيل النشطة عندما تكون مفقودة من إدخال الجلسة الحية. وإذا كان نموذج transcript مختلفًا عن النموذج المحدد، فستحل الحالة نافذة السياق وفق نموذج بيئة التشغيل المستعاد بدلًا من النموذج المحدد.
- بالنسبة إلى احتساب حجم المطالبة، يفضّل الاستكمال من transcript الإجمالي الأكبر الموجّه للمطالبة عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، حتى لا تنخفض جلسات الموفّر المخصص إلى عرض `0` رمز مميز.
- يتضمن الخرج مخازن جلسات لكل وكيل عند تكوين عدة وكلاء.
- تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة Gateway + مضيف node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + Git SHA (في حالات السحب من المصدر).
- تظهر معلومات التحديث في النظرة العامة؛ وإذا كان هناك تحديث متاح، تطبع الحالة تلميحًا لتشغيل `openclaw update` (راجع [Updating](/ar/install/updating)).
- تقوم أسطح الحالة للقراءة فقط (`status`, `status --json`, `status --all`) بحل SecretRefs المدعومة لمسارات الإعداد المستهدفة عند الإمكان.
- إذا تم تكوين SecretRef مدعوم لقناة ما لكنه غير متاح في مسار الأمر الحالي، تبقى الحالة للقراءة فقط وتبلّغ عن خرج متدهور بدلًا من التعطل. يعرض الخرج المخصص للبشر تحذيرات مثل “configured token unavailable in this command path”، ويتضمن خرج JSON الحقل `secretDiagnostics`.
- عندما ينجح حل SecretRef المحلي للأمر، تفضّل الحالة اللقطة المحلولة وتمحو مؤشرات القنوات المؤقتة الخاصة بـ “secret unavailable” من الخرج النهائي.
- يتضمن `status --all` صفًا لنظرة عامة على Secrets وقسمًا للتشخيص يلخص `secretDiagnostics` (مقتطعًا لسهولة القراءة) من دون إيقاف توليد التقرير.
