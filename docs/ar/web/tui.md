## البدء السريع

### وضع Gateway

1. ابدأ Gateway.

```bash
openclaw gateway
```

2. افتح TUI.

```bash
openclaw tui
```

3. اكتب رسالة واضغط Enter.

Gateway البعيدة:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

استخدم `--password` إذا كانت Gateway لديك تستخدم مصادقة كلمة المرور.

### الوضع المحلي

شغّل TUI من دون Gateway:

```bash
openclaw chat
# أو
openclaw tui --local
```

ملاحظات:

- يمثل `openclaw chat` و`openclaw terminal` اسمين مستعارين لـ `openclaw tui --local`.
- لا يمكن الجمع بين `--local` و`--url` أو `--token` أو `--password`.
- يستخدم الوضع المحلي وقت تشغيل العامل المضمّن مباشرةً. تعمل معظم الأدوات المحلية، لكن الميزات الخاصة بـ Gateway غير متاحة.
- يستخدم `openclaw` و`openclaw crestodian` أيضًا غلاف TUI هذا، مع Crestodian كواجهة محادثة محلية للإعداد والإصلاح.

## ما الذي تراه

- الرأس: عنوان URL للاتصال، والعامل الحالي، والجلسة الحالية.
- سجل المحادثة: رسائل المستخدم، وردود المساعد، وإشعارات النظام، وبطاقات الأدوات.
- سطر الحالة: حالة الاتصال/التشغيل (جارٍ الاتصال، جارٍ التشغيل، بث متدفق، خامل، خطأ).
- التذييل: حالة الاتصال + العامل + الجلسة + النموذج + التفكير/السريع/المفصل/التتبع/reasoning + عدد التوكنات + التسليم.
- الإدخال: محرر نصي مع الإكمال التلقائي.

## النموذج الذهني: العوامل + الجلسات

- العوامل هي slugs فريدة (مثل `main` و`research`). وتعرض Gateway القائمة.
- تنتمي الجلسات إلى العامل الحالي.
- تُخزن مفاتيح الجلسات بالشكل `agent:<agentId>:<sessionKey>`.
  - إذا كتبت `/session main`، يوسعها TUI إلى `agent:<currentAgent>:main`.
  - إذا كتبت `/session agent:other:main`، فإنك تنتقل إلى جلسة ذلك العامل صراحةً.
- نطاق الجلسة:
  - `per-sender` (الافتراضي): لكل عامل جلسات متعددة.
  - `global`: يستخدم TUI دائمًا الجلسة `global` (وقد يكون المنتقي فارغًا).
- يكون العامل الحالي + الجلسة الحالية ظاهرين دائمًا في التذييل.

## الإرسال + التسليم

- تُرسل الرسائل إلى Gateway؛ ويكون التسليم إلى المزودات معطّلًا افتراضيًا.
- فعّل تسليم الأدوار:
  - `/deliver on`
  - أو لوحة Settings
  - أو ابدأ باستخدام `openclaw tui --deliver`

## المنتقيات + الطبقات العلوية

- منتقي النماذج: يسرد النماذج المتاحة ويضبط تجاوز الجلسة.
- منتقي العوامل: اختر عاملًا مختلفًا.
- منتقي الجلسات: يعرض فقط جلسات العامل الحالي.
- Settings: بدّل التسليم، وتوسيع مخرجات الأدوات، وظهور التفكير.

## اختصارات لوحة المفاتيح

- Enter: إرسال الرسالة
- Esc: إيقاف التشغيل النشط
- Ctrl+C: مسح الإدخال (اضغط مرتين للخروج)
- Ctrl+D: خروج
- Ctrl+L: منتقي النماذج
- Ctrl+G: منتقي العوامل
- Ctrl+P: منتقي الجلسات
- Ctrl+O: تبديل توسيع مخرجات الأدوات
- Ctrl+T: تبديل ظهور التفكير (يعيد تحميل السجل)

## أوامر Slash

الأساسية:

- `/help`
- `/status`
- `/agent <id>` (أو `/agents`)
- `/session <key>` (أو `/sessions`)
- `/model <provider/model>` (أو `/models`)

عناصر التحكم في الجلسة:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` (اسم مستعار: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

دورة حياة الجلسة:

- `/new` أو `/reset` (إعادة ضبط الجلسة)
- `/abort` (إيقاف التشغيل النشط)
- `/settings`
- `/exit`

الوضع المحلي فقط:

- `/auth [provider]` يفتح تدفق مصادقة/تسجيل دخول المزود داخل TUI.

تُمرر أوامر Slash الأخرى الخاصة بـ Gateway (مثل `/context`) إلى Gateway وتُعرض كمخرجات نظام. راجع [أوامر Slash](/ar/tools/slash-commands).

## أوامر الغلاف المحلية

- ابدأ السطر بالبادئة `!` لتشغيل أمر غلاف محلي على المضيف الذي يعمل عليه TUI.
- يطلب TUI الموافقة مرة واحدة لكل جلسة للسماح بالتنفيذ المحلي؛ وإذا رفضت، يبقى `!` معطّلًا لتلك الجلسة.
- تعمل الأوامر في غلاف جديد غير تفاعلي داخل دليل عمل TUI (من دون `cd`/بيئة مستمرة).
- تستقبل أوامر الغلاف المحلية `OPENCLAW_SHELL=tui-local` ضمن بيئتها.
- تُرسل `!` المفردة كرسالة عادية؛ ولا تؤدي المسافات البادئة إلى تشغيل التنفيذ المحلي.

## إصلاح الإعدادات من TUI المحلي

استخدم الوضع المحلي عندما تكون الإعدادات الحالية صالحة بالفعل وتريد أن يقوم
العامل المضمّن بفحصها على الجهاز نفسه، ومقارنتها بالوثائق،
والمساعدة في إصلاح الانحراف من دون الاعتماد على Gateway قيد التشغيل.

إذا كان `openclaw config validate` يفشل بالفعل، فابدأ أولًا بـ `openclaw configure`
أو `openclaw doctor --fix`. لا يتجاوز `openclaw chat` حاجز
الإعدادات غير الصالحة.

حلقة العمل النموذجية:

1. ابدأ الوضع المحلي:

```bash
openclaw chat
```

2. اطلب من العامل ما الذي تريد التحقق منه، على سبيل المثال:

```text
قارن إعدادات مصادقة gateway لدي بالوثائق واقترح أصغر إصلاح.
```

3. استخدم أوامر الغلاف المحلية للحصول على دليل دقيق والتحقق:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. طبّق تغييرات ضيقة باستخدام `openclaw config set` أو `openclaw configure`، ثم أعد تشغيل `!openclaw config validate`.
5. إذا أوصى Doctor بترحيل أو إصلاح تلقائي، فراجعه وشغّل `!openclaw doctor --fix`.

نصائح:

- فضّل `openclaw config set` أو `openclaw configure` على التحرير اليدوي لـ `openclaw.json`.
- يبحث `openclaw docs "<query>"` في فهرس الوثائق المباشر من الجهاز نفسه.
- يفيد `openclaw config validate --json` عندما تريد أخطاء منظمة للمخطط وSecretRef/إمكانية التحليل.

## مخرجات الأدوات

- تظهر استدعاءات الأدوات كبطاقات تحتوي على الوسيطات + النتائج.
- يبدّل Ctrl+O بين العرض المطوي/الموسع.
- أثناء تشغيل الأدوات، تتدفق التحديثات الجزئية إلى البطاقة نفسها.

## ألوان الطرفية

- يحافظ TUI على نص جسم المساعد بلون المقدمة الافتراضي في الطرفية لديك حتى تبقى الطرفيات الداكنة والفاتحة مقروءة.
- إذا كانت طرفيتك تستخدم خلفية فاتحة وكان الاكتشاف التلقائي خاطئًا، فاضبط `OPENCLAW_THEME=light` قبل تشغيل `openclaw tui`.
- لفرض اللوحة الداكنة الأصلية بدلًا من ذلك، اضبط `OPENCLAW_THEME=dark`.

## السجل + البث المتدفق

- عند الاتصال، يحمّل TUI أحدث السجل (الافتراضي 200 رسالة).
- تُحدَّث الردود المتدفقة في مكانها إلى أن تُنهى.
- يستمع TUI أيضًا إلى أحداث أدوات العامل للحصول على بطاقات أدوات أكثر غنى.

## تفاصيل الاتصال

- يسجّل TUI نفسه لدى Gateway باعتباره `mode: "tui"`.
- تعرض عمليات إعادة الاتصال رسالة نظام؛ كما تظهر فجوات الأحداث في السجل.

## الخيارات

- `--local`: التشغيل مقابل وقت تشغيل العامل المضمّن المحلي
- `--url <url>`: عنوان URL لـ WebSocket الخاص بـ Gateway (الافتراضي من الإعدادات أو `ws://127.0.0.1:<port>`)
- `--token <token>`: رمز Gateway (إذا كان مطلوبًا)
- `--password <password>`: كلمة مرور Gateway (إذا كانت مطلوبة)
- `--session <key>`: مفتاح الجلسة (الافتراضي: `main`، أو `global` عندما يكون النطاق عامًا)
- `--deliver`: تسليم ردود المساعد إلى المزود (معطّل افتراضيًا)
- `--thinking <level>`: تجاوز مستوى التفكير للإرسال
- `--message <text>`: إرسال رسالة أولية بعد الاتصال
- `--timeout-ms <ms>`: مهلة العامل بالمللي ثانية (الافتراضي من `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: عدد إدخالات السجل المطلوب تحميلها (الافتراضي `200`)

ملاحظة: عند ضبط `--url`، لا يعود TUI إلى بيانات الاعتماد الموجودة في الإعدادات أو البيئة.
مرّر `--token` أو `--password` صراحةً. ويُعد غياب بيانات الاعتماد الصريحة خطأً.
في الوضع المحلي، لا تمرر `--url` أو `--token` أو `--password`.

## استكشاف الأخطاء وإصلاحها

لا توجد مخرجات بعد إرسال رسالة:

- شغّل `/status` داخل TUI للتأكد من أن Gateway متصلة وفي حالة خمول/انشغال.
- تحقق من سجلات Gateway: `openclaw logs --follow`.
- أكد أن العامل يمكنه التشغيل: `openclaw status` و`openclaw models status`.
- إذا كنت تتوقع رسائل في قناة دردشة، ففعّل التسليم (`/deliver on` أو `--deliver`).

## استكشاف أخطاء الاتصال وإصلاحها

- `disconnected`: تأكد من أن Gateway قيد التشغيل وأن `--url/--token/--password` صحيحة.
- لا توجد عوامل في المنتقي: تحقق من `openclaw agents list` وإعدادات التوجيه.
- منتقي الجلسات فارغ: قد تكون في النطاق العام أو لا توجد جلسات بعد.
