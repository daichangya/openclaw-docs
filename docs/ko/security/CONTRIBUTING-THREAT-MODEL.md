# OpenClaw 위협 모델에 기여하기

OpenClaw를 더 안전하게 만드는 데 도움을 주셔서 감사합니다. 이 위협 모델은 살아 있는 문서이며, 보안 전문가가 아니어도 누구나 기여할 수 있습니다.

## 기여 방법

### 위협 추가하기

아직 다루지 않은 공격 벡터나 위험을 발견하셨나요? [openclaw/trust](https://github.com/openclaw/trust/issues)에 이슈를 열고 자신의 말로 설명해 주세요. 어떤 프레임워크를 알 필요도 없고, 모든 항목을 채울 필요도 없습니다. 시나리오만 설명해 주시면 됩니다.

**포함하면 도움이 되지만 필수는 아닌 내용:**

- 공격 시나리오와 악용 방식
- OpenClaw의 어떤 부분이 영향을 받는지(CLI, gateway, channels, ClawHub, MCP 서버 등)
- 심각도를 어떻게 생각하는지(low / medium / high / critical)
- 관련 연구, CVE, 실제 사례 링크

ATLAS 매핑, 위협 ID, 위험 평가는 검토 과정에서 저희가 처리합니다. 이러한 세부 사항까지 포함해 주시면 좋지만, 기대하는 것은 아닙니다.

> **이것은 위협 모델에 내용을 추가하기 위한 경로이지, 실제 취약점을 신고하는 경로가 아닙니다.** 악용 가능한 취약점을 발견했다면 책임 있는 공개 지침은 [Trust 페이지](https://trust.openclaw.ai)를 참조하세요.

### 완화책 제안하기

기존 위협을 해결하는 방법에 대한 아이디어가 있나요? 해당 위협을 참조하는 이슈나 PR을 열어 주세요. 유용한 완화책은 구체적이고 실행 가능해야 합니다. 예를 들어 "rate limiting을 구현"보다 "gateway에서 발신자별 분당 10개 메시지 rate limiting"이 더 좋습니다.

### 공격 체인 제안하기

공격 체인은 여러 위협이 결합되어 현실적인 공격 시나리오를 만드는 방식을 보여줍니다. 위험한 조합이 보인다면 단계와 공격자가 그것들을 어떻게 연결할지 설명해 주세요. 실전에서 공격이 어떻게 전개되는지에 대한 짧은 서술이 형식적인 템플릿보다 더 가치 있습니다.

### 기존 내용 수정 또는 개선하기

오탈자, 설명 보완, 오래된 정보, 더 좋은 예시 등 — PR은 언제나 환영이며 이슈는 필요 없습니다.

## 우리가 사용하는 것

### MITRE ATLAS

이 위협 모델은 [MITRE ATLAS](https://atlas.mitre.org/)(AI 시스템을 위한 적대적 위협 환경)를 기반으로 합니다. 이는 프롬프트 인젝션, 도구 오용, 에이전트 악용과 같은 AI/ML 위협을 위해 특별히 설계된 프레임워크입니다. 기여를 위해 ATLAS를 알 필요는 없습니다. 제출된 내용은 검토 중에 저희가 프레임워크에 매핑합니다.

### 위협 ID

각 위협에는 `T-EXEC-003` 같은 ID가 부여됩니다. 카테고리는 다음과 같습니다.

| Code    | Category                                  |
| ------- | ----------------------------------------- |
| RECON   | 정찰 - 정보 수집                          |
| ACCESS  | 초기 접근 - 진입 확보                     |
| EXEC    | 실행 - 악의적 동작 실행                   |
| PERSIST | 지속성 - 접근 유지                        |
| EVADE   | 방어 회피 - 탐지 회피                     |
| DISC    | 탐색 - 환경 파악                          |
| EXFIL   | 유출 - 데이터 탈취                        |
| IMPACT  | 영향 - 피해 또는 서비스 중단              |

ID는 검토 중에 유지보수자가 부여합니다. 직접 고를 필요는 없습니다.

### 위험 수준

| Level        | Meaning                                                           |
| ------------ | ----------------------------------------------------------------- |
| **Critical** | 전체 시스템 장악, 또는 높은 가능성 + 치명적 영향                 |
| **High**     | 상당한 피해 가능성 높음, 또는 중간 가능성 + 치명적 영향          |
| **Medium**   | 중간 수준의 위험, 또는 낮은 가능성 + 높은 영향                   |
| **Low**      | 가능성이 낮고 영향이 제한적임                                    |

위험 수준이 확실하지 않다면 영향만 설명해 주세요. 저희가 평가하겠습니다.

## 검토 절차

1. **초기 분류** - 새 제출은 48시간 이내에 검토합니다.
2. **평가** - 실현 가능성을 확인하고, ATLAS 매핑과 위협 ID를 부여하며, 위험 수준을 검증합니다.
3. **문서화** - 모든 내용이 올바른 형식이고 완전한지 확인합니다.
4. **병합** - 위협 모델과 시각화에 추가합니다.
