세션 가지치기는 각 LLM 호출 전에 컨텍스트에서 **오래된 도구 결과**를 정리합니다. 일반 대화 텍스트는 다시 쓰지 않고도 누적된 도구 출력(exec 결과, 파일 읽기, 검색 결과)로 인한 컨텍스트 팽창을 줄입니다.

<Info>
가지치기는 메모리 내에서만 수행됩니다 -- 디스크에 있는 세션 전사는 수정하지 않습니다.
전체 기록은 항상 보존됩니다.
</Info>

## 왜 중요한가

긴 세션은 컨텍스트 윈도를 부풀리는 도구 출력을 축적합니다. 이로 인해 비용이 증가하고 필요 이상으로 빨리 [Compaction](/ko/concepts/compaction)이 강제될 수 있습니다.

가지치기는 특히 **Anthropic 프롬프트 캐싱**에 중요합니다. 캐시 TTL이 만료되면 다음 요청이 전체 프롬프트를 다시 캐시합니다. 가지치기는 캐시 쓰기 크기를 줄여 비용을 직접 낮춥니다.

## 작동 방식

1. 캐시 TTL이 만료될 때까지 기다립니다(기본값 5분).
2. 일반 가지치기를 위해 오래된 도구 결과를 찾습니다(대화 텍스트는 그대로 둠).
3. 과도하게 큰 결과를 **소프트 트림**합니다 -- 앞부분과 뒷부분은 유지하고 `...`를 삽입합니다.
4. 나머지는 **하드 클리어**합니다 -- 플레이스홀더로 대체합니다.
5. 후속 요청이 새 캐시를 재사용하도록 TTL을 재설정합니다.

## 레거시 이미지 정리

OpenClaw는 기록에 원시 이미지 블록을 유지하던 오래된 레거시 세션에 대해 별도의 멱등 정리도 수행합니다.

- 최근 후속 요청의 프롬프트 캐시 접두사가 안정적으로 유지되도록 **가장 최근 완료된 3개의 턴**은 바이트 단위로 그대로 보존합니다.
- 이미 처리된 오래된 `user` 또는 `toolResult` 기록의 이미지 블록은 `[image data removed - already processed by model]`로 대체될 수 있습니다.
- 이는 일반 캐시 TTL 가지치기와는 별개입니다. 이후 턴에서 반복되는 이미지 payload가 프롬프트 캐시를 깨뜨리는 것을 막기 위해 존재합니다.

## 스마트 기본값

OpenClaw는 Anthropic 프로필에 대해 가지치기를 자동으로 활성화합니다:

| 프로필 유형                                             | 가지치기 활성화 | Heartbeat |
| ------------------------------------------------------- | --------------- | --------- |
| Anthropic OAuth/토큰 인증(Claude CLI 재사용 포함)      | 예              | 1시간     |
| API 키                                                  | 예              | 30분      |

명시적인 값을 설정하면 OpenClaw는 이를 재정의하지 않습니다.

## 활성화 또는 비활성화

Anthropic이 아닌 provider에서는 기본적으로 가지치기가 꺼져 있습니다. 활성화하려면:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

비활성화하려면 `mode: "off"`로 설정하세요.

## 가지치기와 Compaction

|            | 가지치기             | Compaction             |
| ---------- | -------------------- | ---------------------- |
| **무엇**   | 도구 결과를 정리     | 대화를 요약            |
| **저장됨?** | 아니요(요청별)      | 예(전사에 저장)        |
| **범위**   | 도구 결과만          | 전체 대화              |

둘은 서로를 보완합니다 -- 가지치기는 Compaction 주기 사이에 도구 출력을 가볍게 유지합니다.
