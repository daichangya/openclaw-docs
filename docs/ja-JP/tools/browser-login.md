# ブラウザーログイン + X/Twitter 投稿

## 手動ログイン（推奨）

サイトでログインが必要な場合は、**ホスト**のブラウザープロファイル（openclaw browser）で**手動でサインイン**してください。

認証情報をモデルに渡してはいけません。自動ログインはしばしばアンチボット防御を発動させ、アカウントがロックされる可能性があります。

ブラウザーに関するメインドキュメントに戻る: [Browser](/tools/browser)。

## どの Chrome プロファイルが使われますか？

OpenClaw は**専用の Chrome プロファイル**（`openclaw` という名前、オレンジ色がかった UI）を操作します。これは日常的に使うブラウザープロファイルとは別です。

エージェントのブラウザーツール呼び出しでは:

- デフォルトの選択: エージェントは隔離された `openclaw` ブラウザーを使うべきです。
- 既存のログイン済みセッションが重要で、ユーザーがその場でアタッチ確認のプロンプトをクリック/承認できる場合にのみ `profile="user"` を使ってください。
- ユーザーブラウザープロファイルが複数ある場合は、推測せずに明示的に指定してください。

これにアクセスする簡単な方法は 2 つあります。

1. **エージェントにブラウザーを開かせる**。その後、自分でログインします。
2. **CLI 経由で開く**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile <name>` を渡してください（デフォルトは `openclaw` です）。

## X/Twitter: 推奨フロー

- **読む/検索する/スレッド:** **ホスト**のブラウザーを使う（手動ログイン）。
- **更新を投稿する:** **ホスト**のブラウザーを使う（手動ログイン）。

## サンドボックス化 + ホストブラウザーアクセス

サンドボックス化されたブラウザーセッションは、ボット検知を引き起こす可能性が**より高く**なります。X/Twitter（および他の厳格なサイト）では、**ホスト**のブラウザーを優先してください。

エージェントがサンドボックス化されている場合、ブラウザーツールはデフォルトでサンドボックスを対象にします。ホスト制御を許可するには:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

その後、ホストブラウザーを対象にします:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、更新を投稿するエージェントではサンドボックス化を無効にしてください。
