# 設定例

以下の例は、現在の設定スキーマに沿っています。完全なリファレンスと各フィールドの注記については、[Configuration](/gateway/configuration) を参照してください。

## クイックスタート

### 必要最小限

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

これを `~/.openclaw/openclaw.json` に保存すると、その番号からボットに DM を送れます。

### 推奨スターター構成

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-6" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## 一般的なパターン

### 共有 Skills ベースラインに 1 つの上書きを加える

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      skills: ["github", "weather"],
    },
    list: [
      { id: "main", default: true },
      { id: "docs", workspace: "~/.openclaw/workspace-docs", skills: ["docs-search"] },
    ],
  },
}
```

- `agents.defaults.skills` は共有ベースラインです。
- `agents.list[].skills` は、その 1 エージェントに対してベースラインを置き換えます。
- エージェントに Skills を見せたくない場合は `skills: []` を使います。

### マルチプラットフォーム構成

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### セキュア DM モード（共有受信箱 / 複数ユーザー DM）

複数人がボットに DM できる場合（`allowFrom` に複数エントリーがある、複数人へのペアリング承認がある、または `dmPolicy: "open"` の場合）は、**セキュア DM モード**を有効にしてください。これにより、異なる送信者からの DM がデフォルトで 1 つのコンテキストを共有しなくなります。

```json5
{
  // セキュア DM モード（複数ユーザーまたは機微な DM エージェントに推奨）
  session: { dmScope: "per-channel-peer" },

  channels: {
    // 例: WhatsApp の複数ユーザー受信箱
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // 例: Discord の複数ユーザー受信箱
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

Discord / Slack / Google Chat / Microsoft Teams / Mattermost / IRC では、送信者認可はデフォルトで ID 優先です。
各チャンネルの `dangerouslyAllowNameMatching: true` を有効にして、変更可能な名前 / メール / ニックの一致を許可するのは、そのリスクを明示的に受け入れる場合だけにしてください。

## ヒント

- `dmPolicy: "open"` を設定する場合、対応する `allowFrom` リストには `"*"` を含める必要があります。
- プロバイダー ID は異なります（電話番号、ユーザー ID、チャンネル ID など）。形式の確認には各プロバイダーのドキュメントを使ってください。
- 後から追加できる任意セクション: `web`、`browser`、`ui`、`discovery`、`canvasHost`、`talk`、`signal`、`imessage`。
- より詳しいセットアップの注記については、[Providers](/providers) と [Troubleshooting](/gateway/troubleshooting) を参照してください。
