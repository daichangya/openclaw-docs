ワンクリックテンプレートで OpenClaw を Northflank にデプロイし、Web Control UI 経由でアクセスします。
これは「サーバー上でターミナル不要」のもっとも簡単な経路です。Northflank が Gateway を代わりに実行します。

## はじめ方

1. [Deploy OpenClaw](https://northflank.com/stacks/deploy-openclaw) をクリックしてテンプレートを開きます。
2. まだアカウントを持っていない場合は、[Northflank のアカウント](https://app.northflank.com/signup) を作成します。
3. **Deploy OpenClaw now** をクリックします。
4. 必須の環境変数 `OPENCLAW_GATEWAY_TOKEN` を設定します（強力なランダム値を使ってください）。
5. **Deploy stack** をクリックして、OpenClaw テンプレートをビルドして実行します。
6. デプロイが完了するまで待ち、その後 **View resources** をクリックします。
7. OpenClaw サービスを開きます。
8. `/openclaw` の公開 OpenClaw URL を開き、設定した shared secret を使って接続します。このテンプレートではデフォルトで `OPENCLAW_GATEWAY_TOKEN` を使用します。password auth に置き換えた場合は、代わりにその password を使ってください。

## 得られるもの

- ホスト型 OpenClaw Gateway + Control UI
- Northflank Volume（`/data`）による永続ストレージ。これにより `openclaw.json`、
  エージェントごとの `auth-profiles.json`、チャネル/プロバイダー状態、セッション、
  およびワークスペースは再デプロイ後も保持されます

## チャネルを接続する

チャネル設定手順については、`/openclaw` の Control UI を使うか、SSH 経由で `openclaw onboard` を実行します:

- [Telegram](/ja-JP/channels/telegram)（最速。bot token だけで済みます）
- [Discord](/ja-JP/channels/discord)
- [All channels](/ja-JP/channels)
