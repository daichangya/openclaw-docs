# セッションツール

OpenClawは、エージェントに対してセッションをまたいで作業し、ステータスを確認し、
サブエージェントをオーケストレーションするためのツールを提供します。

## 利用可能なツール

| ツール               | 機能                                                                        |
| -------------------- | --------------------------------------------------------------------------- |
| `sessions_list`    | オプションのフィルター（種類、ラベル、エージェント、最近のアクティビティ、プレビュー）付きでセッションを一覧表示 |
| `sessions_history` | 特定のセッションのトランスクリプトを読み取る                                   |
| `sessions_send`    | 別のセッションにメッセージを送信し、必要に応じて待機する                       |
| `sessions_spawn`   | バックグラウンド作業用に分離されたサブエージェントセッションを起動する                     |
| `sessions_yield`   | 現在のターンを終了し、後続のサブエージェント結果を待機する               |
| `subagents`        | このセッションで起動されたサブエージェントを一覧表示、誘導、または停止する                    |
| `session_status`   | `/status` 形式のカードを表示し、必要に応じてセッションごとのモデルオーバーライドを設定する |

## セッションの一覧表示と読み取り

`sessions_list` は、セッションキー、agentId、種類、チャンネル、モデル、
トークン数、およびタイムスタンプを含むセッションを返します。種類（`main`、`group`、`cron`、`hook`、
`node`）、完全一致の `label`、完全一致の `agentId`、検索テキスト、または最近のアクティビティ
（`activeMinutes`）でフィルターできます。メールボックス形式のトリアージが必要な場合は、
派生タイトル、最新メッセージのプレビュー、または件数制限付きの最近のメッセージも取得できます。
プレビュー用のトランスクリプト読み取りは、設定されたセッションツール可視性ポリシーの下で
参照可能なセッションに限定されます。

`sessions_history` は、特定のセッションの会話トランスクリプトを取得します。
デフォルトではツール結果は除外されます。表示するには `includeTools: true` を指定してください。
返されるビューは、意図的に制限され、安全性フィルターが適用されています。

- アシスタントのテキストは想起前に正規化されます:
  - thinking タグは削除されます
  - `<relevant-memories>` / `<relevant_memories>` の足場ブロックは削除されます
  - `<tool_call>...</tool_call>`、`<function_call>...</function_call>`、`<tool_calls>...</tool_calls>`、および
    `<function_calls>...</function_calls>` のようなプレーンテキストのツール呼び出し XML ペイロードブロックは、
    正常に閉じられない切り詰められたペイロードを含めて削除されます
  - `[Tool Call: ...]`、
    `[Tool Result ...]`、および `[Historical context ...]` のような
    格下げされたツール呼び出し/結果の足場は削除されます
  - `<|assistant|>` のような漏れたモデル制御トークン、その他の ASCII
    `<|...|>` トークン、および全角の `<｜...｜>` バリアントは削除されます
  - `<invoke ...>` /
    `</minimax:tool_call>` のような不正な MiniMax ツール呼び出し XML は削除されます
- 認証情報/トークンのようなテキストは返される前に秘匿化されます
- 長いテキストブロックは切り詰められます
- 非常に大きな履歴では、古い行が削除されたり、過大な行が
  `[sessions_history omitted: message too large]` に置き換えられたりする場合があります
- このツールは `truncated`、`droppedMessages`、
  `contentTruncated`、`contentRedacted`、`bytes` のような要約フラグを報告します

どちらのツールも、**session key**（`"main"` など）または以前の一覧呼び出しで得た
**session ID** のどちらでも受け付けます。

完全にバイト単位で一致するトランスクリプトが必要な場合は、
`sessions_history` を生ダンプとして扱うのではなく、ディスク上のトランスクリプトファイルを確認してください。

## セッション間メッセージの送信

`sessions_send` は別のセッションにメッセージを配信し、必要に応じて
応答を待機します。

- **送信のみ:** `timeoutSeconds: 0` を設定すると、キューに追加して
  即座に返ります。
- **返信を待機:** タイムアウトを設定すると、応答をインラインで取得できます。

対象が応答した後、OpenClawは**reply-back loop** を実行でき、
エージェント同士が交互にメッセージを送ります（最大 5 ターン）。対象エージェントは
`REPLY_SKIP` を返して早期終了できます。

## ステータスおよびオーケストレーションの補助

`session_status` は、現在または参照可能な別セッションに対する軽量な
`/status` 相当ツールです。使用量、時刻、モデル/ランタイム状態、および存在する場合は
関連付けられたバックグラウンドタスクのコンテキストを報告します。`/status` と同様に、
最新のトランスクリプト使用量エントリからまばらなトークン/キャッシュカウンターを補完でき、
`model=default` でセッションごとのオーバーライドを解除します。

`sessions_yield` は、現在のターンを意図的に終了し、待機しているフォローアップイベントが
次のメッセージになるようにします。サブエージェントを起動した後、
完了結果をポーリングループで組み立てるのではなく次のメッセージとして受け取りたい場合に使用します。

`subagents` は、すでに起動済みの OpenClaw
サブエージェント用のコントロールプレーン補助です。以下をサポートします。

- `action: "list"` でアクティブ/最近の実行を確認
- `action: "steer"` で実行中の子にフォローアップ指示を送信
- `action: "kill"` で 1 つの子または `all` を停止

## サブエージェントの起動

`sessions_spawn` は、バックグラウンドタスク用に分離されたセッションを作成します。これは常に
非ブロッキングです。`runId` と `childSessionKey` を即座に返します。

主なオプション:

- `runtime: "subagent"`（デフォルト）または外部ハーネスエージェント用の `"acp"`。
- 子セッション用の `model` および `thinking` オーバーライド。
- `thread: true` で起動をチャットスレッド（Discord、Slack など）にバインド。
- `sandbox: "require"` で子にサンドボックス化を強制。

デフォルトのリーフサブエージェントにはセッションツールは付与されません。
`maxSpawnDepth >= 2` の場合、深さ 1 のオーケストレーターサブエージェントには追加で
`sessions_spawn`、`subagents`、`sessions_list`、`sessions_history` が付与され、
自身の子を管理できます。リーフ実行には引き続き再帰的な
オーケストレーションツールは付与されません。

完了後、通知ステップがリクエスターのチャンネルに結果を投稿します。
完了配信では、利用可能な場合はバインド済みのスレッド/トピックルーティングが保持され、
完了元がチャンネルのみを識別している場合でも、OpenClawは
リクエスターセッションに保存されたルート（`lastChannel` / `lastTo`）を再利用して
直接配信できます。

ACP 固有の動作については、[ACP Agents](/ja-JP/tools/acp-agents)を参照してください。

## 可視性

セッションツールは、エージェントが見られる範囲を制限するようにスコープ設定されています。

| レベル   | 範囲                                    |
| ------- | ---------------------------------------- |
| `self`  | 現在のセッションのみ                 |
| `tree`  | 現在のセッション + 起動されたサブエージェント     |
| `agent` | このエージェントのすべてのセッション              |
| `all`   | すべてのセッション（設定されていればエージェント横断） |

デフォルトは `tree` です。サンドボックス化されたセッションは、設定に関係なく
`tree` に固定されます。
