Déployez OpenClaw sur Railway avec un modèle en un clic et accédez-y via l’interface de contrôle web.
C’est la voie la plus simple « sans terminal sur le serveur » : Railway exécute la passerelle pour vous.

## Checklist rapide (nouveaux utilisateurs)

1. Cliquez sur **Deploy on Railway** (ci-dessous).
2. Ajoutez un **Volume** monté sur `/data`.
3. Définissez les **Variables** requises (au minimum `OPENCLAW_GATEWAY_PORT` et `OPENCLAW_GATEWAY_TOKEN`).
4. Activez **HTTP Proxy** sur le port `8080`.
5. Ouvrez `https://<your-railway-domain>/openclaw` et connectez-vous avec le secret partagé configuré. Ce modèle utilise `OPENCLAW_GATEWAY_TOKEN` par défaut ; si vous le remplacez par une auth par mot de passe, utilisez ce mot de passe à la place.

## Déploiement en un clic

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

Après le déploiement, trouvez votre URL publique dans **Railway → votre service → Settings → Domains**.

Railway :

- vous donnera soit un domaine généré (souvent `https://<something>.up.railway.app`), soit
- utilisera votre domaine personnalisé si vous en avez attaché un.

Puis ouvrez :

- `https://<your-railway-domain>/openclaw` — interface de contrôle

## Ce que vous obtenez

- Passerelle OpenClaw hébergée + interface de contrôle
- Stockage persistant via Railway Volume (`/data`) afin que `openclaw.json`,
  `auth-profiles.json` par agent, l’état des canaux/fournisseurs, les sessions et
  le workspace survivent aux redéploiements

## Paramètres Railway requis

### Réseau public

Activez **HTTP Proxy** pour le service.

- Port : `8080`

### Volume (obligatoire)

Attachez un volume monté sur :

- `/data`

### Variables

Définissez ces variables sur le service :

- `OPENCLAW_GATEWAY_PORT=8080` (obligatoire — doit correspondre au port de Public Networking)
- `OPENCLAW_GATEWAY_TOKEN` (obligatoire ; à traiter comme un secret administrateur)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recommandé)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recommandé)

## Connecter un canal

Utilisez l’interface de contrôle à `/openclaw` ou exécutez `openclaw onboard` via le shell Railway pour obtenir les instructions de configuration des canaux :

- [Telegram](/channels/telegram) (le plus rapide — juste un jeton de bot)
- [Discord](/channels/discord)
- [Tous les canaux](/channels)

## Sauvegardes et migration

Exportez votre état, votre configuration, vos profils d’authentification et votre workspace :

```bash
openclaw backup create
```

Cela crée une archive de sauvegarde portable avec l’état OpenClaw ainsi que tout
workspace configuré. Consultez [Backup](/cli/backup) pour plus de détails.
