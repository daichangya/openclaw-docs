# ข้อความกลุ่ม (ช่องทาง WhatsApp web)

เป้าหมาย: ให้ Clawd อยู่ในกลุ่ม WhatsApp ตื่นขึ้นมาเฉพาะเมื่อถูกเรียก และแยกเธรดนั้นออกจากเซสชัน DM ส่วนตัว

หมายเหตุ: ตอนนี้ `agents.list[].groupChat.mentionPatterns` ถูกใช้โดย Telegram/Discord/Slack/iMessage ด้วยเช่นกัน; เอกสารนี้มุ่งเน้นพฤติกรรมเฉพาะของ WhatsApp สำหรับการตั้งค่าหลายเอเจนต์ ให้ตั้ง `agents.list[].groupChat.mentionPatterns` แยกตามเอเจนต์ (หรือใช้ `messages.groupChat.mentionPatterns` เป็นค่าทดแทนแบบส่วนกลาง)

## การทำงานปัจจุบัน (2025-12-03)

- โหมดการเปิดใช้งาน: `mention` (ค่าเริ่มต้น) หรือ `always` โดย `mention` ต้องมีการเรียก (การกล่าวถึง @ จริงของ WhatsApp ผ่าน `mentionedJids`, รูปแบบ regex ที่ปลอดภัย, หรือหมายเลข E.164 ของบอตที่อยู่ที่ใดก็ได้ในข้อความ) ส่วน `always` จะปลุกเอเจนต์ในทุกข้อความ แต่ควรตอบกลับเฉพาะเมื่อสามารถเพิ่มคุณค่าที่มีความหมายได้เท่านั้น; มิฉะนั้นจะคืนค่าโทเค็นเงียบแบบตรงตัว `NO_REPLY` / `no_reply` สามารถตั้งค่าเริ่มต้นได้ใน config (`channels.whatsapp.groups`) และ override แยกตามกลุ่มผ่าน `/activation` เมื่อมีการตั้งค่า `channels.whatsapp.groups` ค่านี้จะทำหน้าที่เป็น allowlist ของกลุ่มด้วย (ใส่ `"*"` เพื่ออนุญาตทุกกลุ่ม)
- นโยบายกลุ่ม: `channels.whatsapp.groupPolicy` ควบคุมว่าจะยอมรับข้อความกลุ่มหรือไม่ (`open|disabled|allowlist`) โดย `allowlist` ใช้ `channels.whatsapp.groupAllowFrom` (ค่าทดแทน: `channels.whatsapp.allowFrom` ที่ระบุไว้อย่างชัดเจน) ค่าเริ่มต้นคือ `allowlist` (จะถูกบล็อกจนกว่าคุณจะเพิ่มผู้ส่ง)
- เซสชันแยกตามกลุ่ม: คีย์เซสชันจะมีลักษณะเป็น `agent:<agentId>:whatsapp:group:<jid>` ดังนั้นคำสั่งอย่าง `/verbose on`, `/trace on`, หรือ `/think high` (ที่ส่งเป็นข้อความเดี่ยว) จะมีขอบเขตเฉพาะกลุ่มนั้น; สถานะ DM ส่วนตัวจะไม่ถูกแตะต้อง และ Heartbeat จะถูกข้ามสำหรับเธรดกลุ่ม
- การฉีดบริบท: ข้อความกลุ่มแบบ **pending-only** (ค่าเริ่มต้น 50 ข้อความ) ที่ _ไม่ได้_ ทริกเกอร์การรัน จะถูกใส่นำหน้าภายใต้ `[Chat messages since your last reply - for context]` โดยบรรทัดที่เป็นตัวทริกเกอร์จะอยู่ภายใต้ `[Current message - respond to this]` ข้อความที่อยู่ในเซสชันแล้วจะไม่ถูกฉีดซ้ำ
- การแสดงผู้ส่ง: ตอนนี้ทุกแบตช์ของกลุ่มจะลงท้ายด้วย `[from: Sender Name (+E164)]` เพื่อให้ Pi รู้ว่าใครกำลังพูด
- ข้อความชั่วคราว/view-once: เราจะแกะห่อสิ่งเหล่านั้นก่อนดึงข้อความ/การกล่าวถึง ดังนั้นการเรียกที่อยู่ภายในก็ยังสามารถทริกเกอร์ได้
- system prompt ของกลุ่ม: ในเทิร์นแรกของเซสชันกลุ่ม (และทุกครั้งที่ `/activation` เปลี่ยนโหมด) เราจะฉีดข้อความสั้น ๆ เข้าไปใน system prompt เช่น `You are replying inside the WhatsApp group "<subject>". Group members: Alice (+44...), Bob (+43...), … Activation: trigger-only … Address the specific sender noted in the message context.` หากไม่มี metadata เราจะยังบอกเอเจนต์ว่าเป็นแชตกลุ่มอยู่ดี

## ตัวอย่าง config (WhatsApp)

เพิ่มบล็อก `groupChat` ลงใน `~/.openclaw/openclaw.json` เพื่อให้การเรียกด้วยชื่อที่แสดงผลทำงานได้ แม้ WhatsApp จะตัด `@` ที่มองเห็นได้ออกจากเนื้อความ:

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true },
      },
    },
  },
  agents: {
    list: [
      {
        id: "main",
        groupChat: {
          historyLimit: 50,
          mentionPatterns: ["@?openclaw", "\\+?15555550123"],
        },
      },
    ],
  },
}
```

หมายเหตุ:

- regex เหล่านี้ไม่แยกตัวพิมพ์เล็ก-ใหญ่ และใช้ guardrail ของ safe-regex แบบเดียวกับพื้นผิว regex อื่น ๆ ใน config; รูปแบบที่ไม่ถูกต้องและการทำซ้ำซ้อนที่ไม่ปลอดภัยจะถูกเพิกเฉย
- WhatsApp ยังส่งการกล่าวถึงแบบ canonical ผ่าน `mentionedJids` เมื่อมีคนแตะที่รายชื่อติดต่อ ดังนั้นค่าทดแทนด้วยหมายเลขจึงแทบไม่จำเป็น แต่ก็เป็นตาข่ายนิรภัยที่มีประโยชน์

### คำสั่งการเปิดใช้งาน (เจ้าของเท่านั้น)

ใช้คำสั่งในแชตกลุ่ม:

- `/activation mention`
- `/activation always`

มีเพียงหมายเลขของเจ้าของเท่านั้น (จาก `channels.whatsapp.allowFrom` หรือหมายเลข E.164 ของบอตเองเมื่อไม่ได้ตั้งค่าไว้) ที่สามารถเปลี่ยนแปลงสิ่งนี้ได้ ส่ง `/status` เป็นข้อความเดี่ยวในกลุ่มเพื่อดูโหมดการเปิดใช้งานปัจจุบัน

## วิธีใช้งาน

1. เพิ่มบัญชี WhatsApp ของคุณ (บัญชีที่รัน OpenClaw) เข้าไปในกลุ่ม
2. พูดว่า `@openclaw …` (หรือใส่หมายเลขไว้) เฉพาะผู้ส่งที่อยู่ใน allowlist เท่านั้นที่สามารถทริกเกอร์ได้ เว้นแต่คุณจะตั้ง `groupPolicy: "open"`
3. พรอมป์ของเอเจนต์จะมีบริบทกลุ่มล่าสุดรวมถึงเครื่องหมาย `[from: …]` ที่ต่อท้าย เพื่อให้ตอบไปยังบุคคลที่ถูกต้องได้
4. คำสั่งระดับเซสชัน (`/verbose on`, `/trace on`, `/think high`, `/new` หรือ `/reset`, `/compact`) จะมีผลเฉพาะกับเซสชันของกลุ่มนั้นเท่านั้น; ส่งเป็นข้อความเดี่ยวเพื่อให้ระบบลงทะเบียน เซสชัน DM ส่วนตัวของคุณยังคงแยกเป็นอิสระ

## การทดสอบ / การตรวจสอบยืนยัน

- การทดสอบแบบ manual smoke:
  - ส่งการเรียก `@openclaw` ในกลุ่ม และยืนยันว่ามีการตอบกลับที่อ้างอิงถึงชื่อผู้ส่ง
  - ส่งการเรียกครั้งที่สอง และตรวจสอบว่าบล็อกประวัติถูกใส่เข้ามาแล้วจากนั้นถูกล้างในเทิร์นถัดไป
- ตรวจสอบบันทึก Gateway (รันด้วย `--verbose`) เพื่อดูรายการ `inbound web message` ที่แสดง `from: <groupJid>` และส่วนต่อท้าย `[from: …]`

## ข้อควรพิจารณาที่ทราบ

- Heartbeat จะถูกข้ามโดยตั้งใจสำหรับกลุ่ม เพื่อหลีกเลี่ยงการกระจายข้อความที่รบกวน
- การกดการสะท้อนกลับใช้สตริงแบตช์รวม; หากคุณส่งข้อความเดียวกันซ้ำสองครั้งโดยไม่มีการกล่าวถึง จะมีเพียงครั้งแรกเท่านั้นที่ได้รับการตอบกลับ
- รายการในที่เก็บเซสชันจะปรากฏเป็น `agent:<agentId>:whatsapp:group:<jid>` ในที่เก็บเซสชัน (`~/.openclaw/agents/<agentId>/sessions/sessions.json` โดยค่าเริ่มต้น); หากไม่มีรายการ ก็เพียงหมายความว่ากลุ่มนั้นยังไม่เคยทริกเกอร์การรัน
- ตัวบ่งชี้การพิมพ์ในกลุ่มจะเป็นไปตาม `agents.defaults.typingMode` (ค่าเริ่มต้น: `message` เมื่อไม่มีการกล่าวถึง)
