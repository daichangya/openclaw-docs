เอกสารนี้อธิบายขั้นตอนการตั้งค่าครั้งแรก **ปัจจุบัน** เป้าหมายคือประสบการณ์ “วันแรก” ที่ราบรื่น: เลือกว่าจะให้ Gateway รันที่ไหน เชื่อมต่อการยืนยันตัวตน เรียกใช้วิซาร์ด และปล่อยให้เอเจนต์บูตสแตรปตัวเอง
สำหรับภาพรวมทั่วไปของเส้นทางการเริ่มต้นใช้งาน ดู [Onboarding Overview](/th/start/onboarding-overview)

โมเดลความไว้วางใจด้านความปลอดภัย:

- โดยค่าเริ่มต้น OpenClaw เป็นเอเจนต์ส่วนบุคคล: มีขอบเขตผู้ปฏิบัติงานที่เชื่อถือได้เพียงหนึ่งเดียว
- การตั้งค่าแบบใช้ร่วมกัน/หลายผู้ใช้ต้องมีการล็อกดาวน์ (แยกขอบเขตความไว้วางใจ จำกัดการเข้าถึงเครื่องมือให้เหลือน้อยที่สุด และปฏิบัติตาม [Security](/th/gateway/security))
- การเริ่มต้นใช้งานแบบ local ตอนนี้ตั้งค่า config ใหม่เป็น `tools.profile: "coding"` โดยค่าเริ่มต้น เพื่อให้การตั้งค่า local ใหม่ยังคงมีเครื่องมือระบบไฟล์/รันไทม์ โดยไม่ต้องบังคับใช้โปรไฟล์ `full` ที่ไม่จำกัด
- หากเปิดใช้ hooks/webhooks หรือฟีดเนื้อหาที่ไม่น่าเชื่อถืออื่นๆ ให้ใช้โมเดลระดับสูงที่ทันสมัยและคงนโยบายเครื่องมือ/การทำ sandboxing ที่เข้มงวด

<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />

**Gateway** รันอยู่ที่ไหน?

- **Mac เครื่องนี้ (Local only):** การเริ่มต้นใช้งานสามารถตั้งค่าการยืนยันตัวตนและเขียนข้อมูลรับรองไว้ในเครื่องได้
- **Remote (ผ่าน SSH/Tailnet):** การเริ่มต้นใช้งานจะ **ไม่** ตั้งค่าการยืนยันตัวตนในเครื่อง; ข้อมูลรับรองต้องมีอยู่บนโฮสต์ gateway
- **ตั้งค่าภายหลัง:** ข้ามการตั้งค่าและปล่อยให้แอปยังไม่ได้รับการกำหนดค่า

<Tip>
**เคล็ดลับเรื่องการยืนยันตัวตนของ Gateway:**

- ตอนนี้วิซาร์ดจะสร้าง **token** แม้สำหรับ loopback ดังนั้นไคลเอนต์ WS แบบ local ต้องยืนยันตัวตน
- หากคุณปิดการยืนยันตัวตน โปรเซสใดๆ ในเครื่องก็สามารถเชื่อมต่อได้; ใช้แบบนั้นเฉพาะบนเครื่องที่เชื่อถือได้ทั้งหมดเท่านั้น
- ใช้ **token** สำหรับการเข้าถึงจากหลายเครื่องหรือการ bind ที่ไม่ใช่ loopback

</Tip>
</Step>

<img src="/assets/macos-onboarding/05-permissions.png" alt="" />

การเริ่มต้นใช้งานจะขอสิทธิ์ TCC ที่จำเป็นสำหรับ:

- การทำงานอัตโนมัติ (AppleScript)
- การแจ้งเตือน
- การช่วยการเข้าถึง
- การบันทึกหน้าจอ
- ไมโครโฟน
- การรู้จำเสียงพูด
- กล้อง
- ตำแหน่งที่ตั้ง

</Step>
<Step title="CLI">
  <Info>ขั้นตอนนี้เป็นทางเลือก</Info>
  แอปสามารถติดตั้ง CLI `openclaw` แบบ global ผ่าน npm, pnpm หรือ bun
  โดยจะเลือก npm ก่อน จากนั้น pnpm แล้วจึงเป็น bun หากเป็นตัวจัดการแพ็กเกจที่ตรวจพบเพียงตัวเดียว
  สำหรับรันไทม์ Gateway นั้น Node ยังคงเป็นแนวทางที่แนะนำ
</Step>
<Step title="แชตการเริ่มต้นใช้งาน (เซสชันเฉพาะ)">
  หลังการตั้งค่า แอปจะเปิดเซสชันแชตสำหรับการเริ่มต้นใช้งานโดยเฉพาะ เพื่อให้เอเจนต์
  แนะนำตัวเองและแนะนำขั้นตอนถัดไป วิธีนี้ช่วยแยกคำแนะนำสำหรับการใช้งานครั้งแรก
  ออกจากการสนทนาปกติของคุณ ดู [Bootstrapping](/th/start/bootstrapping) สำหรับ
  สิ่งที่จะเกิดขึ้นบนโฮสต์ gateway ระหว่างการรันเอเจนต์ครั้งแรก
</Step>
</Steps>
