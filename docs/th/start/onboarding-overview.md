OpenClaw มีเส้นทางการเริ่มต้นใช้งานอยู่สองแบบ ทั้งสองแบบจะกำหนดค่าการยืนยันตัวตน Gateway และ
ช่องทางแชตเพิ่มเติมตามต้องการ — ต่างกันเพียงวิธีที่คุณโต้ตอบกับการตั้งค่าเท่านั้น

## ฉันควรใช้เส้นทางไหน?

|                | การเริ่มต้นใช้งานผ่าน CLI             | การเริ่มต้นใช้งานผ่านแอป macOS |
| -------------- | -------------------------------------- | ------------------------------ |
| **แพลตฟอร์ม** | macOS, Linux, Windows (เนทีฟหรือ WSL2) | macOS เท่านั้น                 |
| **อินเทอร์เฟซ** | วิซาร์ดในเทอร์มินัล                   | UI แบบมีคำแนะนำในแอป          |
| **เหมาะที่สุดสำหรับ** | เซิร์ฟเวอร์, แบบ headless, การควบคุมเต็มรูปแบบ | เดสก์ท็อป Mac, การตั้งค่าแบบภาพ |
| **ระบบอัตโนมัติ** | `--non-interactive` สำหรับสคริปต์    | ด้วยตนเองเท่านั้น              |
| **คำสั่ง**     | `openclaw onboard`                     | เปิดแอป                        |

ผู้ใช้ส่วนใหญ่ควรเริ่มด้วย **การเริ่มต้นใช้งานผ่าน CLI** — ใช้งานได้ทุกที่และให้
การควบคุมมากที่สุด

## สิ่งที่การเริ่มต้นใช้งานจะกำหนดค่า

ไม่ว่าคุณจะเลือกเส้นทางใด การเริ่มต้นใช้งานจะตั้งค่าสิ่งต่อไปนี้:

1. **ผู้ให้บริการโมเดลและการยืนยันตัวตน** — API key, OAuth หรือ setup token สำหรับผู้ให้บริการที่คุณเลือก
2. **Workspace** — ไดเรกทอรีสำหรับไฟล์เอเจนต์ เทมเพลต bootstrap และหน่วยความจำ
3. **Gateway** — พอร์ต, bind address, โหมดการยืนยันตัวตน
4. **ช่องทาง** (ไม่บังคับ) — ช่องทางแชตในตัวและแบบ bundled เช่น
   BlueBubbles, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams,
   Telegram, WhatsApp และอื่น ๆ
5. **Daemon** (ไม่บังคับ) — บริการเบื้องหลังเพื่อให้ Gateway เริ่มทำงานโดยอัตโนมัติ

## การเริ่มต้นใช้งานผ่าน CLI

รันในเทอร์มินัลใดก็ได้:

```bash
openclaw onboard
```

เพิ่ม `--install-daemon` เพื่อให้ติดตั้งบริการเบื้องหลังในขั้นตอนเดียวด้วย

ข้อมูลอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (CLI)](/th/start/wizard)
เอกสารคำสั่ง CLI: [`openclaw onboard`](/th/cli/onboard)

## การเริ่มต้นใช้งานผ่านแอป macOS

เปิดแอป OpenClaw วิซาร์ดเมื่อเปิดใช้งานครั้งแรกจะพาคุณผ่านขั้นตอนเดียวกัน
ด้วยอินเทอร์เฟซแบบภาพ

ข้อมูลอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (แอป macOS)](/th/start/onboarding)

## ผู้ให้บริการแบบกำหนดเองหรือที่ไม่อยู่ในรายการ

หากผู้ให้บริการของคุณไม่อยู่ในรายการการเริ่มต้นใช้งาน ให้เลือก **Custom Provider** และ
ป้อนข้อมูลดังนี้:

- โหมดความเข้ากันได้ของ API (เข้ากันได้กับ OpenAI, เข้ากันได้กับ Anthropic หรือ auto-detect)
- Base URL และ API key
- Model ID และ alias เพิ่มเติมตามต้องการ

endpoint แบบกำหนดเองหลายรายการสามารถอยู่ร่วมกันได้ — แต่ละรายการจะมี endpoint ID ของตนเอง
