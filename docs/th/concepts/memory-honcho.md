# หน่วยความจำ Honcho

[Honcho](https://honcho.dev) เพิ่มหน่วยความจำแบบ AI-native ให้กับ OpenClaw โดยจะบันทึกบทสนทนาไปยังบริการเฉพาะ และสร้างแบบจำลองผู้ใช้และเอเจนต์เมื่อเวลาผ่านไป ทำให้เอเจนต์ของคุณมีบริบทข้ามเซสชันที่ไปไกลกว่าไฟล์ Markdown ใน workspace

## สิ่งที่มีให้

- **หน่วยความจำข้ามเซสชัน** -- บทสนทนาจะถูกบันทึกหลังจากทุกเทิร์น ดังนั้นบริบทจึงคงอยู่ข้ามการรีเซ็ตเซสชัน, Compaction และการสลับช่องทาง
- **การสร้างแบบจำลองผู้ใช้** -- Honcho จะดูแลโปรไฟล์สำหรับผู้ใช้แต่ละคน (ค่ากำหนด ข้อเท็จจริง รูปแบบการสื่อสาร) และสำหรับเอเจนต์ (บุคลิกภาพ พฤติกรรมที่เรียนรู้)
- **การค้นหาเชิงความหมาย** -- ค้นหาจากข้อสังเกตในบทสนทนาที่ผ่านมา ไม่ใช่แค่เซสชันปัจจุบัน
- **การรับรู้หลายเอเจนต์** -- เอเจนต์แม่จะติดตาม sub-agent ที่สร้างขึ้นโดยอัตโนมัติ โดยเพิ่มเอเจนต์แม่เป็นผู้สังเกตการณ์ในเซสชันลูก

## เครื่องมือที่ใช้งานได้

Honcho ลงทะเบียนเครื่องมือที่เอเจนต์สามารถใช้ระหว่างการสนทนาได้:

**การดึงข้อมูล (เร็ว, ไม่เรียก LLM):**

| เครื่องมือ                    | สิ่งที่ทำ                                                  |
| ----------------------------- | ---------------------------------------------------------- |
| `honcho_context`              | ข้อมูลตัวแทนของผู้ใช้แบบครบถ้วนข้ามเซสชัน                |
| `honcho_search_conclusions`   | การค้นหาเชิงความหมายใน conclusions ที่จัดเก็บไว้          |
| `honcho_search_messages`      | ค้นหาข้อความข้ามเซสชัน (กรองตามผู้ส่ง วันที่)            |
| `honcho_session`              | ประวัติและสรุปของเซสชันปัจจุบัน                           |

**ถาม-ตอบ (ขับเคลื่อนด้วย LLM):**

| เครื่องมือ     | สิ่งที่ทำ                                                                         |
| --------------- | --------------------------------------------------------------------------------- |
| `honcho_ask`    | ถามเกี่ยวกับผู้ใช้ ใช้ `depth='quick'` สำหรับข้อเท็จจริง และ `'thorough'` สำหรับการสังเคราะห์ |

## เริ่มต้นใช้งาน

ติดตั้ง Plugin แล้วรันการตั้งค่า:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

คำสั่ง setup จะถามหาข้อมูลรับรอง API ของคุณ เขียนคอนฟิก และสามารถย้ายไฟล์หน่วยความจำ workspace ที่มีอยู่เดิมได้ตามตัวเลือก

<Info>
Honcho สามารถรันได้ทั้งหมดบนเครื่องของคุณเอง (self-hosted) หรือผ่าน managed API ที่
`api.honcho.dev` ตัวเลือก self-hosted ไม่ต้องมี dependency ภายนอก
</Info>

## การกำหนดค่า

การตั้งค่าจะอยู่ภายใต้ `plugins.entries["openclaw-honcho"].config`:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // ละไว้ได้สำหรับ self-hosted
          workspaceId: "openclaw", // การแยกหน่วยความจำ
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

สำหรับอินสแตนซ์ self-hosted ให้ชี้ `baseUrl` ไปยังเซิร์ฟเวอร์ภายในเครื่องของคุณ (เช่น
`http://localhost:8000`) และไม่ต้องใส่ API key

## การย้ายหน่วยความจำที่มีอยู่เดิม

หากคุณมีไฟล์หน่วยความจำใน workspace อยู่แล้ว (`USER.md`, `MEMORY.md`,
`IDENTITY.md`, `memory/`, `canvas/`) `openclaw honcho setup` จะตรวจพบและเสนอให้ย้ายข้อมูล

<Info>
การย้ายข้อมูลจะไม่ทำลายข้อมูลเดิม -- ไฟล์จะถูกอัปโหลดไปยัง Honcho โดยไฟล์ต้นฉบับจะไม่ถูกลบหรือย้าย
</Info>

## วิธีการทำงาน

หลังจากทุก AI turn บทสนทนาจะถูกบันทึกลงใน Honcho ทั้งข้อความของผู้ใช้และ
เอเจนต์จะถูกสังเกต เพื่อให้ Honcho สามารถสร้างและปรับปรุงแบบจำลองของตนเมื่อเวลาผ่านไป

ระหว่างการสนทนา เครื่องมือของ Honcho จะสอบถามบริการในช่วง `before_prompt_build`
โดยฉีดบริบทที่เกี่ยวข้องเข้ามาก่อนที่โมเดลจะเห็นพรอมป์ วิธีนี้ช่วยให้ขอบเขตของแต่ละ turn แม่นยำและเรียกคืนข้อมูลที่เกี่ยวข้องได้

## Honcho เทียบกับหน่วยความจำในตัว

|                   | ในตัว / QMD                    | Honcho                              |
| ----------------- | ------------------------------ | ----------------------------------- |
| **การจัดเก็บ**    | ไฟล์ Markdown ใน workspace     | บริการเฉพาะ (ในเครื่องหรือโฮสต์)  |
| **ข้ามเซสชัน**    | ผ่านไฟล์หน่วยความจำ            | อัตโนมัติ มีมาในตัว                |
| **การสร้างแบบจำลองผู้ใช้** | ทำเอง (เขียนลง `MEMORY.md`) | โปรไฟล์อัตโนมัติ                  |
| **การค้นหา**      | Vector + keyword (ไฮบริด)      | เชิงความหมายบนข้อสังเกต           |
| **หลายเอเจนต์**   | ไม่มีการติดตาม                 | รับรู้ความสัมพันธ์แม่/ลูก          |
| **Dependencies**  | ไม่มี (ในตัว) หรือไบนารี QMD   | ติดตั้ง Plugin                      |

Honcho และระบบหน่วยความจำในตัวสามารถทำงานร่วมกันได้ เมื่อมีการกำหนดค่า QMD
เครื่องมือเพิ่มเติมจะพร้อมใช้งานสำหรับการค้นหาไฟล์ Markdown ภายในเครื่องควบคู่ไปกับหน่วยความจำข้ามเซสชันของ Honcho

## คำสั่ง CLI

```bash
openclaw honcho setup                        # กำหนดค่า API key และย้ายไฟล์
openclaw honcho status                       # ตรวจสอบสถานะการเชื่อมต่อ
openclaw honcho ask <question>               # สอบถาม Honcho เกี่ยวกับผู้ใช้
openclaw honcho search <query> [-k N] [-d D] # ค้นหาเชิงความหมายในหน่วยความจำ
```
