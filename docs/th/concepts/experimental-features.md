ฟีเจอร์ experimental ใน OpenClaw คือ **พื้นผิวพรีวิวแบบต้องเลือกใช้เอง** ฟีเจอร์เหล่านี้
ถูกวางไว้หลังแฟล็กที่ต้องเปิดใช้อย่างชัดเจน เพราะยังต้องการการใช้งานจริงเพิ่มเติมก่อนที่จะ
สมควรได้รับค่าเริ่มต้นแบบเสถียรหรือสัญญาสาธารณะที่ใช้งานระยะยาว

ให้ปฏิบัติกับสิ่งเหล่านี้ต่างจาก config ปกติ:

- ให้ **ปิดไว้โดยค่าเริ่มต้น** เว้นแต่เอกสารที่เกี่ยวข้องจะบอกให้คุณลองใช้
- คาดว่า **รูปแบบและพฤติกรรมอาจเปลี่ยนแปลง** ได้เร็วกว่าคอนฟิกที่เสถียร
- ให้ใช้เส้นทางที่เสถียรก่อนเมื่อมีอยู่แล้ว
- หากคุณกำลังนำ OpenClaw ไปใช้งานในวงกว้าง ให้ทดสอบแฟล็ก experimental ใน
  สภาพแวดล้อมที่เล็กกว่าก่อนจะนำไปเป็นค่าพื้นฐานที่ใช้ร่วมกัน

## แฟล็กที่มีการจัดทำเอกสารไว้ในปัจจุบัน

| Surface                  | Key                                                       | Use it when                                                                                                    | More                                                                                          |
| ------------------------ | --------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| รันไทม์โมเดลในเครื่อง      | `agents.defaults.experimental.localModelLean`             | แบ็กเอนด์ในเครื่องที่เล็กกว่าหรือเข้มงวดกว่ามีปัญหากับพื้นผิวเครื่องมือค่าเริ่มต้นแบบเต็มของ OpenClaw | [Local Models](/th/gateway/local-models)                                                         |
| การค้นหา Memory            | `agents.defaults.memorySearch.experimental.sessionMemory` | คุณต้องการให้ `memory_search` สร้างดัชนี transcript ของเซสชันก่อนหน้าและยอมรับต้นทุนการจัดเก็บ/สร้างดัชนีเพิ่มเติม | [ข้อมูลอ้างอิงการกำหนดค่า Memory](/th/reference/memory-config#session-memory-search-experimental) |
| เครื่องมือวางแผนแบบมีโครงสร้าง | `tools.experimental.planTool`                             | คุณต้องการให้เครื่องมือ `update_plan` แบบมีโครงสร้างแสดงให้ใช้งานสำหรับการติดตามงานหลายขั้นตอนในรันไทม์และ UI ที่รองรับ | [ข้อมูลอ้างอิงการกำหนดค่า Gateway](/th/gateway/config-tools#toolsexperimental)                    |

## โหมด lean ของโมเดลในเครื่อง

`agents.defaults.experimental.localModelLean: true` เป็นวาล์วผ่อนแรง
สำหรับการตั้งค่าโมเดลในเครื่องที่อ่อนกำลังกว่า มันจะตัดเครื่องมือค่าเริ่มต้นที่มีน้ำหนักมาก เช่น
`browser`, `cron` และ `message` ออก เพื่อให้รูปแบบพรอมป์ต์เล็กลงและเปราะบางน้อยลง
สำหรับแบ็กเอนด์ที่เข้ากันได้กับ OpenAI ซึ่งมีบริบทขนาดเล็กหรือเข้มงวดกว่า

นี่ตั้งใจให้ **ไม่ใช่** เส้นทางปกติ หากแบ็กเอนด์ของคุณจัดการรันไทม์แบบเต็ม
ได้อย่างราบรื่น ก็ให้ปิดสิ่งนี้ไว้

## Experimental ไม่ได้แปลว่าซ่อนอยู่

หากฟีเจอร์ใดเป็น experimental OpenClaw ควรระบุเรื่องนั้นอย่างชัดเจนทั้งในเอกสารและใน
พาธ config เอง สิ่งที่ **ไม่ควร** ทำคือแอบยัดพฤติกรรมพรีวิวเข้าไปในปุ่มควบคุมที่ดูเสถียร
และแสร้งทำว่าเป็นเรื่องปกติ นั่นคือวิธีที่พื้นผิวของ config
จะยุ่งเหยิง
