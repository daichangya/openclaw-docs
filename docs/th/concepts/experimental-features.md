# ฟีเจอร์ Experimental

ฟีเจอร์ Experimental ใน OpenClaw คือ **พื้นผิวพรีวิวแบบเลือกเปิดใช้เอง** โดยจะอยู่หลังแฟล็กที่ต้องเปิดอย่างชัดเจน เพราะยังต้องผ่านการใช้งานจริงอีกพอสมควรก่อนจะเหมาะเป็นค่าเริ่มต้นแบบเสถียรหรือเป็นสัญญาสาธารณะระยะยาว

ให้ปฏิบัติต่อฟีเจอร์เหล่านี้ต่างจาก config ปกติ:

- ให้ **ปิดไว้เป็นค่าเริ่มต้น** เว้นแต่เอกสารที่เกี่ยวข้องจะบอกให้คุณลองใช้
- คาดว่า **รูปแบบและพฤติกรรมอาจเปลี่ยนแปลง** ได้เร็วกว่าค่า config ที่เสถียร
- ให้เลือกใช้เส้นทางที่เสถียรก่อน หากมีอยู่แล้ว
- หากคุณกำลังนำ OpenClaw ไปใช้งานในวงกว้าง ให้ทดสอบแฟล็ก experimental ในสภาพแวดล้อมที่เล็กกว่าก่อนนำไปใช้เป็น baseline ร่วม

## แฟล็กที่มีการบันทึกไว้ในเอกสารในปัจจุบัน

| พื้นผิว                  | คีย์                                                      | ใช้เมื่อ                                                                                                       | เพิ่มเติม                                                                                     |
| ------------------------ | --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| รันไทม์โมเดลในเครื่อง    | `agents.defaults.experimental.localModelLean`             | แบ็กเอนด์ในเครื่องที่เล็กกว่าหรือเข้มงวดกว่ารับมือพื้นผิวเครื่องมือเริ่มต้นแบบเต็มของ OpenClaw ไม่ไหว        | [Local Models](/th/gateway/local-models)                                                         |
| การค้นหา memory          | `agents.defaults.memorySearch.experimental.sessionMemory` | คุณต้องการให้ `memory_search` สร้างดัชนีจากทรานสคริปต์ของ session ก่อนหน้า และยอมรับต้นทุนด้านการเก็บข้อมูล/ทำดัชนีที่เพิ่มขึ้น | [เอกสารอ้างอิงการตั้งค่า memory](/th/reference/memory-config#session-memory-search-experimental) |
| เครื่องมือวางแผนแบบมีโครงสร้าง | `tools.experimental.planTool`                             | คุณต้องการให้มีการเปิดเผยเครื่องมือ `update_plan` แบบมีโครงสร้างสำหรับการติดตามงานหลายขั้นตอนในรันไทม์และ UI ที่รองรับ | [เอกสารอ้างอิงการตั้งค่า Gateway](/th/gateway/configuration-reference#toolsexperimental)         |

## โหมด lean สำหรับโมเดลในเครื่อง

`agents.defaults.experimental.localModelLean: true` เป็นวาล์วระบายแรงกดดันสำหรับชุดการใช้งานโมเดลในเครื่องที่อ่อนกว่า มันจะตัดเครื่องมือเริ่มต้นที่มีน้ำหนักมากอย่าง `browser`, `cron` และ `message` ออก เพื่อให้รูปทรงของพรอมป์ต์เล็กลงและเปราะบางน้อยลงสำหรับแบ็กเอนด์ที่รองรับบริบทขนาดเล็กหรือแบ็กเอนด์ที่เข้ากันได้กับ OpenAI แบบเข้มงวดกว่า

นี่ **ไม่ใช่** เส้นทางปกติโดยเจตนา หากแบ็กเอนด์ของคุณรองรับรันไทม์เต็มรูปแบบได้อย่างราบรื่น ก็ให้ปิดค่านี้ไว้

## Experimental ไม่ได้แปลว่าซ่อนอยู่

หากฟีเจอร์ใดเป็น experimental OpenClaw ควรบอกให้ชัดเจนทั้งในเอกสารและในพาธของ config เอง สิ่งที่มัน **ไม่ควร** ทำ คือแอบยัดพฤติกรรมพรีวิวเข้าไปในปุ่มควบคุมที่ดูเหมือนเสถียรตามปกติแล้วทำเหมือนว่านั่นเป็นเรื่องธรรมดา เพราะนั่นคือวิธีที่ทำให้พื้นผิวของ config รกรุงรัง
