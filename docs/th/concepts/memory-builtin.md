เอนจินที่มีมาในตัวคือแบ็กเอนด์ Memory เริ่มต้น โดยจะจัดเก็บดัชนี Memory ของคุณไว้ใน
ฐานข้อมูล SQLite แยกตามเอเจนต์ และไม่ต้องมี dependency เพิ่มเติมเพื่อเริ่มต้นใช้งาน

## สิ่งที่มีให้

- **การค้นหาแบบคีย์เวิร์ด** ผ่านการทำดัชนีข้อความเต็มด้วย FTS5 (การให้คะแนน BM25)
- **การค้นหาแบบเวกเตอร์** ผ่าน embeddings จากผู้ให้บริการที่รองรับใดก็ได้
- **การค้นหาแบบไฮบริด** ที่รวมทั้งสองแบบเข้าด้วยกันเพื่อผลลัพธ์ที่ดีที่สุด
- **การรองรับ CJK** ผ่านการแบ่งโทเค็นแบบ trigram สำหรับภาษาจีน ญี่ปุ่น และเกาหลี
- **การเร่งความเร็วด้วย sqlite-vec** สำหรับคำสั่งค้นหาเวกเตอร์ภายในฐานข้อมูล (ไม่บังคับ)

## เริ่มต้นใช้งาน

หากคุณมีคีย์ API สำหรับ OpenAI, Gemini, Voyage หรือ Mistral เอนจินที่มีมาในตัว
จะตรวจจับได้อัตโนมัติและเปิดใช้การค้นหาแบบเวกเตอร์ โดยไม่ต้องตั้งค่าใด ๆ

หากต้องการตั้งค่าผู้ให้บริการแบบชัดเจน:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "openai",
      },
    },
  },
}
```

หากไม่มีผู้ให้บริการ embedding จะใช้ได้เฉพาะการค้นหาแบบคีย์เวิร์ด

หากต้องการบังคับใช้ผู้ให้บริการ embedding ภายในเครื่องแบบมีมาในตัว ให้ชี้ `local.modelPath` ไปยัง
ไฟล์ GGUF:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "local",
        fallback: "none",
        local: {
          modelPath: "~/.node-llama-cpp/models/embeddinggemma-300m-qat-Q8_0.gguf",
        },
      },
    },
  },
}
```

## ผู้ให้บริการ embedding ที่รองรับ

| Provider | ID        | Auto-detected | Notes                               |
| -------- | --------- | ------------- | ----------------------------------- |
| OpenAI   | `openai`  | ใช่           | ค่าเริ่มต้น: `text-embedding-3-small`   |
| Gemini   | `gemini`  | ใช่           | รองรับหลายสื่อ (ภาพ + เสียง) |
| Voyage   | `voyage`  | ใช่           |                                     |
| Mistral  | `mistral` | ใช่           |                                     |
| Ollama   | `ollama`  | ไม่ใช่        | ในเครื่อง, ต้องตั้งค่าเองอย่างชัดเจน               |
| Local    | `local`   | ใช่ (อันดับแรก)   | โมเดล GGUF, ดาวน์โหลดประมาณ 0.6 GB        |

การตรวจจับอัตโนมัติจะเลือกผู้ให้บริการตัวแรกที่สามารถ resolve คีย์ API ได้ ตามลำดับ
ที่แสดงไว้ ตั้งค่า `memorySearch.provider` เพื่อ override

## การทำดัชนีทำงานอย่างไร

OpenClaw จะทำดัชนี `MEMORY.md` และ `memory/*.md` เป็นชังก์ (~400 โทเค็น โดยมี
การซ้อนทับ 80 โทเค็น) แล้วจัดเก็บไว้ในฐานข้อมูล SQLite แยกตามเอเจนต์

- **ตำแหน่งดัชนี:** `~/.openclaw/memory/<agentId>.sqlite`
- **การเฝ้าดูไฟล์:** การเปลี่ยนแปลงไฟล์ Memory จะทริกเกอร์การทำดัชนีใหม่แบบ debounce (1.5 วินาที)
- **การทำดัชนีใหม่อัตโนมัติ:** เมื่อผู้ให้บริการ embedding โมเดล หรือการตั้งค่า chunking
  เปลี่ยนไป ดัชนีทั้งหมดจะถูกสร้างใหม่โดยอัตโนมัติ
- **ทำดัชนีใหม่ตามต้องการ:** `openclaw memory index --force`

## ควรใช้เมื่อใด

เอนจินที่มีมาในตัวเป็นตัวเลือกที่เหมาะสมสำหรับผู้ใช้ส่วนใหญ่:

- ใช้งานได้ทันทีโดยไม่ต้องมี dependency เพิ่มเติม
- จัดการการค้นหาแบบคีย์เวิร์ดและเวกเตอร์ได้ดี
- รองรับผู้ให้บริการ embedding ทั้งหมด
- การค้นหาแบบไฮบริดผสานข้อดีของทั้งสองแนวทางการดึงข้อมูลเข้าด้วยกัน

พิจารณาเปลี่ยนไปใช้ [QMD](/th/concepts/memory-qmd) หากคุณต้องการ reranking, query
expansion หรืออยากทำดัชนีไดเรกทอรีนอกเวิร์กสเปซ

พิจารณา [Honcho](/th/concepts/memory-honcho) หากคุณต้องการ Memory ข้ามเซสชันพร้อม
การสร้างโมเดลผู้ใช้อัตโนมัติ

## การแก้ไขปัญหา

**ปิดใช้งานการค้นหา Memory อยู่หรือไม่?** ตรวจสอบ `openclaw memory status` หากไม่พบผู้ให้บริการ
ให้ตั้งค่าเองอย่างชัดเจนหรือเพิ่มคีย์ API

**ไม่พบผู้ให้บริการในเครื่องหรือไม่?** ยืนยันว่าพาธในเครื่องมีอยู่จริง แล้วรัน:

```bash
openclaw memory status --deep --agent main
openclaw memory index --force --agent main
```

ทั้งคำสั่ง CLI แบบสแตนด์อโลนและ Gateway ใช้ provider id `local` เดียวกัน
หากตั้งค่าผู้ให้บริการเป็น `auto` local embeddings จะถูกพิจารณาก่อนเป็นอันดับแรกก็ต่อเมื่อ
`memorySearch.local.modelPath` ชี้ไปยังไฟล์ในเครื่องที่มีอยู่จริง

**ผลลัพธ์เก่าหรือไม่อัปเดต?** รัน `openclaw memory index --force` เพื่อสร้างใหม่ ตัวเฝ้าดู
อาจพลาดการเปลี่ยนแปลงได้ในบางกรณีขอบที่พบไม่บ่อย

**โหลด sqlite-vec ไม่ได้หรือไม่?** OpenClaw จะ fallback ไปใช้ cosine similarity ภายในโปรเซส
โดยอัตโนมัติ ตรวจสอบ log เพื่อดูข้อผิดพลาดเฉพาะในการโหลด

## การกำหนดค่า

สำหรับการตั้งค่าผู้ให้บริการ embedding การปรับแต่งการค้นหาแบบไฮบริด (ค่าน้ำหนัก, MMR, temporal
decay) การทำดัชนีแบบแบตช์ Memory หลายสื่อ sqlite-vec พาธเพิ่มเติม และปุ่มปรับแต่งอื่น ๆ ทั้งหมด ดู
[ข้อมูลอ้างอิงการกำหนดค่า Memory](/th/reference/memory-config)
