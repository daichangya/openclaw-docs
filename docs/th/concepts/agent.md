# รันไทม์ของเอเจนต์

OpenClaw รันรันไทม์เอเจนต์แบบฝังตัวเพียงตัวเดียว

## Workspace (จำเป็น)

OpenClaw ใช้ไดเรกทอรี workspace ของเอเจนต์เพียงรายการเดียว (`agents.defaults.workspace`) เป็นไดเรกทอรีทำงาน (`cwd`) **เพียงแห่งเดียว** ของเอเจนต์สำหรับเครื่องมือและบริบท

คำแนะนำ: ใช้ `openclaw setup` เพื่อสร้าง `~/.openclaw/openclaw.json` หากยังไม่มี และเริ่มต้นไฟล์ใน workspace

ผัง workspace แบบเต็ม + คู่มือสำรองข้อมูล: [workspace ของเอเจนต์](/th/concepts/agent-workspace)

หากเปิดใช้ `agents.defaults.sandbox` เซสชันที่ไม่ใช่เซสชันหลักสามารถ override ค่านี้ได้ด้วย
workspace ต่อเซสชันภายใต้ `agents.defaults.sandbox.workspaceRoot` (ดู
[การกำหนดค่า Gateway](/th/gateway/configuration))

## ไฟล์ Bootstrap (ถูก inject)

ภายใน `agents.defaults.workspace`, OpenClaw คาดว่าจะมีไฟล์ที่ผู้ใช้แก้ไขได้ดังต่อไปนี้:

- `AGENTS.md` — คำสั่งการทำงาน + “ความทรงจำ”
- `SOUL.md` — บุคลิก ขอบเขต น้ำเสียง
- `TOOLS.md` — บันทึกเกี่ยวกับเครื่องมือที่ผู้ใช้ดูแลเอง (เช่น `imsg`, `sag`, แบบแผนต่างๆ)
- `BOOTSTRAP.md` — พิธีเริ่มต้นแบบครั้งเดียวในการรันครั้งแรก (จะถูกลบหลังเสร็จสิ้น)
- `IDENTITY.md` — ชื่อ/บรรยากาศ/อีโมจิของเอเจนต์
- `USER.md` — โปรไฟล์ผู้ใช้ + วิธีการเรียกแทนตัวที่ต้องการ

ในเทิร์นแรกของเซสชันใหม่ OpenClaw จะ inject เนื้อหาของไฟล์เหล่านี้เข้าไปในบริบทของเอเจนต์โดยตรง

ไฟล์ว่างจะถูกข้าม ไฟล์ขนาดใหญ่จะถูกตัดและ truncate พร้อมเครื่องหมายกำกับเพื่อให้พรอมป์ต์ยังคงกระชับ (ให้อ่านไฟล์เพื่อดูเนื้อหาเต็ม)

หากไม่มีไฟล์ OpenClaw จะ inject บรรทัด marker เดียวว่า “missing file” (และ `openclaw setup` จะสร้างเทมเพลตค่าเริ่มต้นที่ปลอดภัยให้)

`BOOTSTRAP.md` จะถูกสร้างขึ้นเฉพาะสำหรับ **workspace ใหม่เอี่ยม** เท่านั้น (ยังไม่มีไฟล์ bootstrap อื่นอยู่) หากคุณลบมันหลังจากทำพิธีเสร็จแล้ว มันไม่ควรถูกสร้างขึ้นใหม่ในการรีสตาร์ตครั้งต่อๆ ไป

หากต้องการปิดการสร้างไฟล์ bootstrap ทั้งหมด (สำหรับ workspace ที่เตรียมไว้ล่วงหน้า) ให้ตั้งค่า:

```json5
{ agent: { skipBootstrap: true } }
```

## เครื่องมือในตัว

เครื่องมือแกนหลัก (read/exec/edit/write และเครื่องมือระบบที่เกี่ยวข้อง) พร้อมใช้งานเสมอ
โดยขึ้นอยู่กับนโยบายเครื่องมือ `apply_patch` เป็นตัวเลือกและถูกควบคุมด้วย
`tools.exec.applyPatch` `TOOLS.md` **ไม่ได้** ควบคุมว่าเครื่องมือใดมีอยู่บ้าง; มันเป็น
แนวทางสำหรับวิธีที่_คุณ_ต้องการให้เครื่องมือเหล่านั้นถูกใช้งาน

## Skills

OpenClaw โหลด Skills จากตำแหน่งเหล่านี้ (ลำดับความสำคัญสูงสุดก่อน):

- Workspace: `<workspace>/skills`
- Skills เอเจนต์ของโปรเจ็กต์: `<workspace>/.agents/skills`
- Skills เอเจนต์ส่วนตัว: `~/.agents/skills`
- แบบจัดการ/ในเครื่อง: `~/.openclaw/skills`
- แบบ bundled (มาพร้อมการติดตั้ง)
- โฟลเดอร์ skill เพิ่มเติม: `skills.load.extraDirs`

Skills สามารถถูกควบคุมด้วย config/env ได้ (ดู `skills` ใน [การกำหนดค่า Gateway](/th/gateway/configuration))

## ขอบเขตรันไทม์

รันไทม์เอเจนต์แบบฝังตัวถูกสร้างบนแกนเอเจนต์ Pi (โมเดล เครื่องมือ และ
prompt pipeline) การจัดการเซสชัน การค้นพบ การเชื่อมต่อเครื่องมือ และการส่งผ่านช่องทาง
เป็นเลเยอร์ที่ OpenClaw ดูแลเองซึ่งวางอยู่บนแกนนั้น

## เซสชัน

ทรานสคริปต์ของเซสชันจะถูกเก็บเป็น JSONL ที่:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

Session ID มีความคงที่และถูกเลือกโดย OpenClaw
ระบบจะไม่อ่านโฟลเดอร์เซสชันแบบเก่าจากเครื่องมืออื่น

## การ steer ระหว่างการสตรีม

เมื่อ queue mode เป็น `steer` ข้อความขาเข้าจะถูก inject เข้าไปในการรันปัจจุบัน
steering ที่อยู่ในคิวจะถูกส่ง **หลังจากเทิร์นปัจจุบันของผู้ช่วยทำการเรียกใช้เครื่องมือเสร็จสิ้น
แล้ว** ก่อนการเรียก LLM ครั้งถัดไป ขณะนี้ steering จะไม่ข้าม
การเรียกใช้เครื่องมือที่เหลือจากข้อความผู้ช่วยปัจจุบันอีกต่อไป; มันจะ inject ข้อความในคิว
ที่ขอบเขตโมเดลถัดไปแทน

เมื่อ queue mode เป็น `followup` หรือ `collect` ข้อความขาเข้าจะถูกพักไว้จนกว่า
เทิร์นปัจจุบันจะจบ จากนั้นจึงเริ่มเทิร์นเอเจนต์ใหม่พร้อม payloads ที่อยู่ในคิว ดู
[Queue](/th/concepts/queue) สำหรับพฤติกรรมของโหมด + debounce/cap

block streaming จะส่ง assistant blocks ที่เสร็จสมบูรณ์ทันทีที่จบ; โดยค่าเริ่มต้นจะ
**ปิดอยู่** (`agents.defaults.blockStreamingDefault: "off"`).
ปรับขอบเขตได้ผ่าน `agents.defaults.blockStreamingBreak` (`text_end` เทียบกับ `message_end`; ค่าเริ่มต้นคือ text_end)
ควบคุมการแบ่ง chunk แบบนุ่มนวลของบล็อกด้วย `agents.defaults.blockStreamingChunk` (ค่าเริ่มต้นอยู่ที่
800–1200 อักขระ; จะพยายามตัดที่ย่อหน้าก่อน จากนั้นเป็นขึ้นบรรทัดใหม่ และประโยคเป็นลำดับสุดท้าย)
รวม streamed chunks ด้วย `agents.defaults.blockStreamingCoalesce` เพื่อลด
สแปมบรรทัดเดียว (รวมแบบอิงช่วงว่างก่อนส่ง) ช่องทางที่ไม่ใช่ Telegram ต้อง
เปิด `*.blockStreaming: true` อย่างชัดเจนเพื่อเปิดใช้การตอบแบบเป็นบล็อก
ระบบจะส่งสรุปเครื่องมือแบบละเอียดเมื่อเริ่มใช้เครื่องมือ (ไม่มี debounce); Control UI
จะสตรีมผลลัพธ์ของเครื่องมือผ่านเหตุการณ์ของเอเจนต์เมื่อมีให้ใช้
รายละเอียดเพิ่มเติม: [การสตรีม + การแบ่ง chunk](/th/concepts/streaming)

## การอ้างอิงโมเดล

การอ้างอิงโมเดลใน config (เช่น `agents.defaults.model` และ `agents.defaults.models`) จะถูกแยกโดยแบ่งที่ `/` **ตัวแรก**

- ใช้ `provider/model` เมื่อตั้งค่าโมเดล
- หาก model ID เองมี `/` อยู่ด้วย (แบบ OpenRouter) ให้ใส่คำนำหน้าผู้ให้บริการด้วย (ตัวอย่าง: `openrouter/moonshotai/kimi-k2`)
- หากคุณละผู้ให้บริการไว้ OpenClaw จะลองใช้นามแฝงก่อน จากนั้นลอง
  จับคู่ผู้ให้บริการที่กำหนดค่าไว้ซึ่งมีความเป็นเอกลักษณ์สำหรับ model id นั้นแบบตรงตัว และหลังจากนั้นจึง fallback
  ไปยังผู้ให้บริการค่าเริ่มต้นที่กำหนดไว้ หากผู้ให้บริการนั้นไม่เปิดให้ใช้
  โมเดลค่าเริ่มต้นที่กำหนดไว้อีกต่อไป OpenClaw จะ fallback ไปยัง
  ผู้ให้บริการ/โมเดลรายการแรกที่กำหนดค่าไว้ แทนที่จะแสดงค่าเริ่มต้นของผู้ให้บริการที่ถูกลบไปแล้วและล้าสมัย

## การกำหนดค่า (ขั้นต่ำ)

- `channels.whatsapp.allowFrom` (แนะนำอย่างยิ่ง)
