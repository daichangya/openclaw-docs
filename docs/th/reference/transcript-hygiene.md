# สุขอนามัยของทรานสคริปต์ (การปรับแก้เฉพาะผู้ให้บริการ)

เอกสารนี้อธิบาย **การแก้ไขเฉพาะผู้ให้บริการ** ที่ใช้กับทรานสคริปต์ก่อนการรัน
(การสร้างบริบทของโมเดล) การปรับแก้เหล่านี้เป็นการปรับ **ในหน่วยความจำ**
ที่ใช้เพื่อให้เป็นไปตามข้อกำหนดที่เข้มงวดของผู้ให้บริการ ขั้นตอนด้านสุขอนามัยเหล่านี้จะ **ไม่**
เขียนทับทรานสคริปต์ JSONL ที่จัดเก็บไว้บนดิสก์ใหม่ อย่างไรก็ตาม กระบวนการซ่อมแซมไฟล์เซสชันแยกต่างหากอาจเขียนทับไฟล์ JSONL
ที่มีรูปแบบไม่ถูกต้องใหม่ โดยลบบรรทัดที่ไม่ถูกต้องก่อนโหลดเซสชัน เมื่อมีการซ่อมแซมเกิดขึ้น
ไฟล์ต้นฉบับจะถูกสำรองไว้ข้างไฟล์เซสชัน

ขอบเขตรวมถึง:

- การทำให้ ID การเรียกใช้เครื่องมือปลอดภัย
- การตรวจสอบความถูกต้องของอินพุตการเรียกใช้เครื่องมือ
- การซ่อมแซมการจับคู่ผลลัพธ์ของเครื่องมือ
- การตรวจสอบ / การจัดลำดับของเทิร์น
- การล้างข้อมูลลายเซ็นความคิด
- การทำให้เพย์โหลดรูปภาพปลอดภัย
- การติดแท็กแหล่งที่มาของอินพุตผู้ใช้ (สำหรับพรอมป์ที่ส่งต่อข้ามเซสชัน)

หากคุณต้องการรายละเอียดการจัดเก็บทรานสคริปต์ ดูที่:

- [/reference/session-management-compaction](/th/reference/session-management-compaction)

## ทำงานที่ใด

สุขอนามัยของทรานสคริปต์ทั้งหมดถูกรวมศูนย์ไว้ในตัวรันเนอร์แบบฝัง:

- การเลือกนโยบาย: `src/agents/transcript-policy.ts`
- การใช้การทำให้ปลอดภัย/การซ่อมแซม: `sanitizeSessionHistory` ใน `src/agents/pi-embedded-runner/replay-history.ts`

นโยบายใช้ `provider`, `modelApi` และ `modelId` เพื่อตัดสินใจว่าจะใช้อะไร

แยกจากสุขอนามัยของทรานสคริปต์ ไฟล์เซสชันจะถูกซ่อมแซม (หากจำเป็น) ก่อนโหลด:

- `repairSessionFileIfNeeded` ใน `src/agents/session-file-repair.ts`
- ถูกเรียกจาก `run/attempt.ts` และ `compact.ts` (ตัวรันเนอร์แบบฝัง)

---

## กฎส่วนกลาง: การทำให้รูปภาพปลอดภัย

เพย์โหลดรูปภาพจะถูกทำให้ปลอดภัยเสมอเพื่อป้องกันการปฏิเสธจากฝั่งผู้ให้บริการเนื่องจากข้อจำกัด
ด้านขนาด (ลดขนาด/บีบอัดใหม่รูปภาพ base64 ที่มีขนาดใหญ่เกินไป)

สิ่งนี้ยังช่วยควบคุมแรงกดดันด้านโทเค็นจากรูปภาพสำหรับโมเดลที่รองรับการมองเห็น
โดยทั่วไป ขนาดสูงสุดที่เล็กลงจะลดการใช้โทเค็น ขณะที่ขนาดที่ใหญ่ขึ้นจะคงรายละเอียดไว้

การติดตั้งใช้งาน:

- `sanitizeSessionMessagesImages` ใน `src/agents/pi-embedded-helpers/images.ts`
- `sanitizeContentBlocksImages` ใน `src/agents/tool-images.ts`
- ด้านสูงสุดของรูปภาพสามารถกำหนดค่าได้ผ่าน `agents.defaults.imageMaxDimensionPx` (ค่าเริ่มต้น: `1200`)

---

## กฎส่วนกลาง: การเรียกใช้เครื่องมือที่มีรูปแบบไม่ถูกต้อง

บล็อกการเรียกใช้เครื่องมือของผู้ช่วยที่ไม่มีทั้ง `input` และ `arguments` จะถูกทิ้ง
ก่อนสร้างบริบทของโมเดล ซึ่งป้องกันการปฏิเสธจากผู้ให้บริการที่เกิดจากการเรียกใช้เครื่องมือ
ที่ถูกบันทึกไว้ไม่สมบูรณ์ (เช่น หลังจากเกิดความล้มเหลวจากการจำกัดอัตรา)

การติดตั้งใช้งาน:

- `sanitizeToolCallInputs` ใน `src/agents/session-transcript-repair.ts`
- ใช้ใน `sanitizeSessionHistory` ใน `src/agents/pi-embedded-runner/replay-history.ts`

## กฎส่วนกลาง: แหล่งที่มาของอินพุตข้ามเซสชัน

เมื่อเอเจนต์ส่งพรอมป์ไปยังอีกเซสชันผ่าน `sessions_send` (รวมถึง
ขั้นตอนตอบกลับ/ประกาศแบบเอเจนต์ถึงเอเจนต์) OpenClaw จะบันทึกเทิร์นผู้ใช้ที่สร้างขึ้นพร้อมกับ:

- `message.provenance.kind = "inter_session"`

เมทาดาทานี้จะถูกเขียนในเวลาที่เพิ่มลงทรานสคริปต์ และไม่เปลี่ยนบทบาท
(`role: "user"` ยังคงเดิมเพื่อความเข้ากันได้กับผู้ให้บริการ) ตัวอ่านทรานสคริปต์สามารถใช้ข้อมูลนี้
เพื่อหลีกเลี่ยงการตีความพรอมป์ภายในที่ส่งต่อแล้วว่าเป็นคำสั่งที่เขียนโดยผู้ใช้ปลายทาง

ระหว่างการสร้างบริบทใหม่ OpenClaw จะเติมตัวทำเครื่องหมาย `[Inter-session message]`
สั้น ๆ ไว้หน้าข้อความผู้ใช้เหล่านั้นในหน่วยความจำด้วย เพื่อให้โมเดลสามารถแยกความแตกต่างจาก
คำสั่งของผู้ใช้ปลายทางภายนอกได้

## เมทริกซ์ผู้ให้บริการ (พฤติกรรมปัจจุบัน)

**OpenAI / OpenAI Codex**

- การทำให้รูปภาพปลอดภัยเท่านั้น
- ทิ้งลายเซ็นการให้เหตุผลที่กำพร้า (รายการการให้เหตุผลแบบเดี่ยวที่ไม่มีบล็อกเนื้อหาตามมา) สำหรับทรานสคริปต์ OpenAI Responses/Codex
- ไม่มีการทำให้ ID การเรียกใช้เครื่องมือปลอดภัย
- ไม่มีการซ่อมแซมการจับคู่ผลลัพธ์ของเครื่องมือ
- ไม่มีการตรวจสอบหรือจัดลำดับเทิร์นใหม่
- ไม่มีผลลัพธ์ของเครื่องมือแบบสังเคราะห์
- ไม่มีการลบลายเซ็นความคิด

**Google (Generative AI / Gemini CLI / Antigravity)**

- การทำให้ ID การเรียกใช้เครื่องมือปลอดภัย: อักขระตัวอักษรและตัวเลขอย่างเข้มงวด
- การซ่อมแซมการจับคู่ผลลัพธ์ของเครื่องมือและผลลัพธ์ของเครื่องมือแบบสังเคราะห์
- การตรวจสอบเทิร์น (การสลับเทิร์นแบบ Gemini)
- การแก้ไขลำดับเทิร์นของ Google (เติม user bootstrap ขนาดเล็กไว้ข้างหน้าหากประวัติเริ่มต้นด้วยผู้ช่วย)
- Antigravity Claude: ปรับลายเซ็นการคิดให้เป็นมาตรฐาน; ทิ้งบล็อกการคิดที่ไม่มีลายเซ็น

**Anthropic / Minimax (เข้ากันได้กับ Anthropic)**

- การซ่อมแซมการจับคู่ผลลัพธ์ของเครื่องมือและผลลัพธ์ของเครื่องมือแบบสังเคราะห์
- การตรวจสอบเทิร์น (รวมเทิร์นผู้ใช้ที่ต่อเนื่องกันเพื่อให้เป็นไปตามการสลับอย่างเข้มงวด)

**Mistral (รวมถึงการตรวจจับตาม model-id)**

- การทำให้ ID การเรียกใช้เครื่องมือปลอดภัย: strict9 (อักขระตัวอักษรและตัวเลข ความยาว 9)

**OpenRouter Gemini**

- การล้างข้อมูลลายเซ็นความคิด: ลบค่า `thought_signature` ที่ไม่ใช่ base64 (เก็บ base64 ไว้)

**อย่างอื่นทั้งหมด**

- การทำให้รูปภาพปลอดภัยเท่านั้น

## พฤติกรรมในอดีต (ก่อน 2026.1.22)

ก่อนรีลีส 2026.1.22 OpenClaw ใช้สุขอนามัยของทรานสคริปต์หลายชั้น:

- ส่วนขยาย **transcript-sanitize** ทำงานในทุกการสร้างบริบท และสามารถ:
  - ซ่อมแซมการจับคู่การใช้เครื่องมือ/ผลลัพธ์
  - ทำให้ ID การเรียกใช้เครื่องมือปลอดภัย (รวมถึงโหมดไม่เข้มงวดที่คง `_`/`-` ไว้)
- ตัวรันเนอร์ยังทำการทำให้ปลอดภัยเฉพาะผู้ให้บริการด้วย ซึ่งทำให้เกิดงานซ้ำซ้อน
- มีการกลายพันธุ์เพิ่มเติมเกิดขึ้นนอกนโยบายของผู้ให้บริการ รวมถึง:
  - ลบแท็ก `<final>` ออกจากข้อความของผู้ช่วยก่อนการบันทึกถาวร
  - ทิ้งเทิร์นข้อผิดพลาดของผู้ช่วยที่ว่างเปล่า
  - ตัดแต่งเนื้อหาของผู้ช่วยหลังการเรียกใช้เครื่องมือ

ความซับซ้อนนี้ทำให้เกิด regression ข้ามผู้ให้บริการ (โดยเฉพาะการจับคู่
`openai-responses` `call_id|fc_id`) การล้างระบบใน 2026.1.22 ได้ลบส่วนขยายออก รวมศูนย์
ตรรกะไว้ในตัวรันเนอร์ และทำให้ OpenAI เป็นแบบ **ไม่แตะต้อง** นอกเหนือจากการทำให้รูปภาพปลอดภัย
