การบันทึก Trajectory คือเครื่องบันทึกการบินต่อเซสชันของ OpenClaw โดยจะบันทึก
ไทม์ไลน์แบบมีโครงสร้างสำหรับการรันของเอเจนต์แต่ละครั้ง จากนั้น `/export-trajectory` จะจัดแพ็กเกจ
เซสชันปัจจุบันเป็นบันเดิลสำหรับการซัพพอร์ตที่ปกปิดข้อมูลแล้ว

ใช้สิ่งนี้เมื่อคุณต้องการตอบคำถาม เช่น:

- มีการส่งพรอมป์, system prompt และเครื่องมือใดไปยังโมเดล?
- ข้อความทรานสคริปต์และการเรียกใช้เครื่องมือใดที่นำไปสู่คำตอบนี้?
- การรันหมดเวลา ถูกยกเลิก ทำ Compaction หรือเจอข้อผิดพลาดจากผู้ให้บริการหรือไม่?
- โมเดล, Plugin, Skills และการตั้งค่ารันไทม์ใดที่เปิดใช้งานอยู่?
- ผู้ให้บริการส่งคืนเมทาดาทาการใช้งานและ prompt-cache ใดมาบ้าง?

## เริ่มต้นอย่างรวดเร็ว

ส่งสิ่งนี้ในเซสชันที่กำลังใช้งานอยู่:

```text
/export-trajectory
```

alias:

```text
/trajectory
```

OpenClaw จะเขียนบันเดิลไว้ภายใต้ workspace:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

คุณสามารถเลือกชื่อไดเรกทอรีเอาต์พุตแบบ relative ได้:

```text
/export-trajectory bug-1234
```

พาธแบบกำหนดเองจะถูกแก้หาอยู่ภายใน `.openclaw/trajectory-exports/` โดยพาธแบบ absolute
และพาธ `~` จะถูกปฏิเสธ

## การเข้าถึง

การส่งออก Trajectory เป็นคำสั่งสำหรับเจ้าของ ผู้ส่งต้องผ่านการตรวจสอบการอนุญาตของคำสั่งตามปกติ
และการตรวจสอบสิทธิ์เจ้าของสำหรับช่องทางนั้น

## สิ่งที่ถูกบันทึก

การบันทึก Trajectory เปิดใช้งานตามค่าเริ่มต้นสำหรับการรันเอเจนต์ OpenClaw

เหตุการณ์ของรันไทม์ประกอบด้วย:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.completed`
- `trace.artifacts`
- `session.ended`

เหตุการณ์ของทรานสคริปต์ก็จะถูกสร้างขึ้นใหม่จาก branch ที่ใช้งานอยู่ของเซสชันด้วย:

- ข้อความผู้ใช้
- ข้อความผู้ช่วย
- การเรียกใช้เครื่องมือ
- ผลลัพธ์ของเครื่องมือ
- Compaction
- การเปลี่ยนโมเดล
- label และรายการเซสชันแบบกำหนดเอง

เหตุการณ์จะถูกเขียนเป็น JSON Lines พร้อมเครื่องหมายสคีมานี้:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## ไฟล์ในบันเดิล

บันเดิลที่ส่งออกอาจมีสิ่งต่อไปนี้:

| ไฟล์                  | เนื้อหา                                                                                      |
| --------------------- | -------------------------------------------------------------------------------------------- |
| `manifest.json`       | สคีมาของบันเดิล ไฟล์ต้นทาง จำนวนเหตุการณ์ และรายการไฟล์ที่สร้างขึ้น                         |
| `events.jsonl`        | ไทม์ไลน์ของรันไทม์และทรานสคริปต์ตามลำดับ                                                   |
| `session-branch.json` | branch ทรานสคริปต์ที่ใช้งานอยู่และส่วนหัวของเซสชันที่ปกปิดข้อมูลแล้ว                        |
| `metadata.json`       | เวอร์ชัน OpenClaw, OS/รันไทม์, โมเดล, snapshot ของ config, Plugin, Skills และเมทาดาทาของพรอมป์ |
| `artifacts.json`      | สถานะสุดท้าย ข้อผิดพลาด การใช้งาน prompt cache จำนวน Compaction ข้อความผู้ช่วย และเมทาดาทาเครื่องมือ |
| `prompts.json`        | พรอมป์ที่ส่งไปและรายละเอียดบางส่วนของการสร้างพรอมป์                                         |
| `system-prompt.txt`   | system prompt ที่คอมไพล์ล่าสุด เมื่อมีการบันทึก                                              |
| `tools.json`          | นิยามเครื่องมือที่ส่งไปยังโมเดล เมื่อมีการบันทึก                                             |

`manifest.json` จะแสดงรายการไฟล์ที่มีอยู่ในบันเดิลนั้น ไฟล์บางส่วนจะถูกละไว้
เมื่อเซสชันไม่ได้บันทึกข้อมูลรันไทม์ที่สอดคล้องกัน

## ตำแหน่งการบันทึก

ตามค่าเริ่มต้น เหตุการณ์ Trajectory ของรันไทม์จะถูกเขียนไว้ข้างไฟล์เซสชัน:

```text
<session>.trajectory.jsonl
```

OpenClaw ยังเขียนไฟล์ตัวชี้แบบ best-effort ไว้ข้างเซสชันด้วย:

```text
<session>.trajectory-path.json
```

ตั้งค่า `OPENCLAW_TRAJECTORY_DIR` เพื่อจัดเก็บ sidecar ของ Trajectory ของรันไทม์ไว้ใน
ไดเรกทอรีเฉพาะ:

```bash
export OPENCLAW_TRAJECTORY_DIR=/var/lib/openclaw/trajectories
```

เมื่อตั้งค่าตัวแปรนี้ OpenClaw จะเขียนไฟล์ JSONL หนึ่งไฟล์ต่อ session id ลงใน
ไดเรกทอรีนั้น

## ปิดการบันทึก

ตั้งค่า `OPENCLAW_TRAJECTORY=0` ก่อนเริ่ม OpenClaw:

```bash
export OPENCLAW_TRAJECTORY=0
```

การทำเช่นนี้จะปิดการบันทึก Trajectory ของรันไทม์ `/export-trajectory` ยังคงส่งออก
branch ของทรานสคริปต์ได้ แต่ไฟล์ที่มีเฉพาะข้อมูลรันไทม์ เช่น compiled context,
artifacts ของผู้ให้บริการ และเมทาดาทาของพรอมป์ อาจหายไป

## ความเป็นส่วนตัวและขีดจำกัด

บันเดิล Trajectory ถูกออกแบบมาสำหรับการซัพพอร์ตและการดีบัก ไม่ใช่สำหรับโพสต์สาธารณะ
OpenClaw จะปกปิดค่าที่ละเอียดอ่อนก่อนเขียนไฟล์ส่งออก:

- credentials และฟิลด์เพย์โหลดที่ดูคล้ายข้อมูลลับที่รู้จัก
- ข้อมูลรูปภาพ
- พาธของ state ภายในเครื่อง
- พาธของ workspace ซึ่งจะถูกแทนที่ด้วย `$WORKSPACE_DIR`
- พาธโฮมไดเรกทอรี เมื่อสามารถตรวจพบได้

ตัวส่งออกยังจำกัดขนาดของอินพุตด้วย:

- ไฟล์ sidecar ของรันไทม์: 50 MiB
- ไฟล์เซสชัน: 50 MiB
- เหตุการณ์ของรันไทม์: 200,000
- เหตุการณ์ที่ส่งออกรวมทั้งหมด: 250,000
- บรรทัดของเหตุการณ์รันไทม์แต่ละบรรทัดจะถูกตัดทอนหากเกิน 256 KiB

ตรวจสอบบันเดิลก่อนแชร์ออกนอกทีมของคุณ การปกปิดข้อมูลเป็นแบบ best-effort
และไม่สามารถรู้ข้อมูลลับเฉพาะแอปพลิเคชันทุกชนิดได้

## การแก้ไขปัญหา

หากการส่งออกไม่มีเหตุการณ์ของรันไทม์:

- ยืนยันว่า OpenClaw ถูกเริ่มโดยไม่มี `OPENCLAW_TRAJECTORY=0`
- ตรวจสอบว่า `OPENCLAW_TRAJECTORY_DIR` ชี้ไปยังไดเรกทอรีที่เขียนได้
- รันอีกหนึ่งข้อความในเซสชัน แล้วส่งออกอีกครั้ง
- ตรวจสอบ `manifest.json` สำหรับ `runtimeEventCount`

หากคำสั่งปฏิเสธพาธเอาต์พุต:

- ใช้ชื่อแบบ relative เช่น `bug-1234`
- อย่าส่ง `/tmp/...` หรือ `~/...`
- เก็บการส่งออกไว้ภายใน `.openclaw/trajectory-exports/`

หากการส่งออกล้มเหลวด้วยข้อผิดพลาดด้านขนาด แสดงว่าเซสชันหรือ sidecar เกิน
ขีดจำกัดความปลอดภัยของการส่งออก เริ่มเซสชันใหม่หรือส่งออก reproduction ที่เล็กกว่า
