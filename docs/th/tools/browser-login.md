# การเข้าสู่ระบบเบราว์เซอร์ + การโพสต์บน X/Twitter

## การเข้าสู่ระบบด้วยตนเอง (แนะนำ)

เมื่อเว็บไซต์ต้องการการเข้าสู่ระบบ ให้ **ลงชื่อเข้าใช้ด้วยตนเอง** ในโปรไฟล์เบราว์เซอร์บน **host** (เบราว์เซอร์ openclaw)

**อย่า** ให้ข้อมูลรับรองของคุณกับโมเดล การเข้าสู่ระบบอัตโนมัติมักกระตุ้นการป้องกัน anti-bot และอาจทำให้บัญชีถูกล็อกได้

กลับไปยังเอกสารเบราว์เซอร์หลัก: [Browser](/th/tools/browser)

## ใช้โปรไฟล์ Chrome ใด?

OpenClaw ควบคุม **โปรไฟล์ Chrome เฉพาะ** (ชื่อ `openclaw`, UI โทนสีส้ม) ซึ่งแยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำวัน

สำหรับการเรียกใช้เครื่องมือเบราว์เซอร์ของเอเจนต์:

- ตัวเลือกเริ่มต้น: เอเจนต์ควรใช้เบราว์เซอร์ `openclaw` ที่แยกไว้ของตน
- ใช้ `profile="user"` เฉพาะเมื่อเซสชันที่ล็อกอินอยู่เดิมมีความสำคัญ และผู้ใช้อยู่ที่คอมพิวเตอร์เพื่อคลิก/อนุมัติพรอมป์ต์การเชื่อมต่อใดๆ
- หากคุณมีหลายโปรไฟล์เบราว์เซอร์ของผู้ใช้ ให้ระบุโปรไฟล์อย่างชัดเจนแทนการเดา

มี 2 วิธีง่ายๆ ในการเข้าถึง:

1. **ขอให้เอเจนต์เปิดเบราว์เซอร์** แล้วคุณค่อยลงชื่อเข้าใช้เอง
2. **เปิดผ่าน CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

หากคุณมีหลายโปรไฟล์ ให้ส่ง `--browser-profile <name>` (ค่าเริ่มต้นคือ `openclaw`)

## X/Twitter: โฟลว์ที่แนะนำ

- **อ่าน/ค้นหา/เธรด:** ใช้เบราว์เซอร์บน **host** (เข้าสู่ระบบด้วยตนเอง)
- **โพสต์อัปเดต:** ใช้เบราว์เซอร์บน **host** (เข้าสู่ระบบด้วยตนเอง)

## Sandboxing + การเข้าถึงเบราว์เซอร์บน host

เซสชันเบราว์เซอร์แบบ sandboxed **มีแนวโน้มมากกว่า** ที่จะกระตุ้นการตรวจจับบอต สำหรับ X/Twitter (และเว็บไซต์อื่นที่เข้มงวด) ควรใช้เบราว์เซอร์บน **host**

หากเอเจนต์อยู่ใน sandbox เครื่องมือเบราว์เซอร์จะใช้ sandbox เป็นค่าเริ่มต้น หากต้องการอนุญาตการควบคุม host:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

จากนั้นให้กำหนดเป้าหมายไปที่เบราว์เซอร์บน host:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

หรือปิดการใช้ sandboxing สำหรับเอเจนต์ที่โพสต์อัปเดต
