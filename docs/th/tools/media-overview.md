# การสร้างและทำความเข้าใจสื่อ

OpenClaw สามารถสร้างรูปภาพ วิดีโอ และเพลง ทำความเข้าใจสื่อขาเข้า (รูปภาพ เสียง วิดีโอ) และพูดคำตอบออกเสียงด้วยการแปลงข้อความเป็นเสียงพูด ความสามารถด้านสื่อทั้งหมดขับเคลื่อนผ่านเครื่องมือ: เอเจนต์จะตัดสินใจว่าจะใช้เมื่อใดตามบทสนทนา และแต่ละเครื่องมือจะแสดงขึ้นก็ต่อเมื่อมีการกำหนดค่าผู้ให้บริการที่รองรับอย่างน้อยหนึ่งราย

## ความสามารถโดยสรุป

| ความสามารถ           | เครื่องมือ             | ผู้ให้บริการ                                                                                 | สิ่งที่ทำได้                                               |
| -------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| การสร้างรูปภาพ     | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                            | สร้างหรือแก้ไขรูปภาพจากพรอมป์ข้อความหรือรูปภาพอ้างอิง |
| การสร้างวิดีโอ     | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | สร้างวิดีโอจากข้อความ รูปภาพ หรือวิดีโอที่มีอยู่แล้ว    |
| การสร้างเพลง     | `music_generate` | ComfyUI, Google, MiniMax                                                                     | สร้างเพลงหรือแทร็กเสียงจากพรอมป์ข้อความ         |
| การแปลงข้อความเป็นเสียงพูด (TTS) | `tts`            | ElevenLabs, Google, Microsoft, MiniMax, OpenAI, xAI                                          | แปลงคำตอบขาออกให้เป็นเสียงพูด               |
| การทำความเข้าใจสื่อ  | (อัตโนมัติ)      | ผู้ให้บริการโมเดลใดก็ตามที่รองรับการมองเห็น/เสียง รวมถึงตัวสำรองผ่าน CLI                                  | สรุปรูปภาพ เสียง และวิดีโอขาเข้า             |

## วิธีการทำงานของการสร้างแบบอะซิงก์

การสร้างวิดีโอและเพลงจะทำงานเป็นงานเบื้องหลัง เนื่องจากการประมวลผลของผู้ให้บริการมักใช้เวลาตั้งแต่ 30 วินาทีไปจนถึงหลายนาที เมื่อเอเจนต์เรียก `video_generate` หรือ `music_generate` OpenClaw จะส่งคำขอไปยังผู้ให้บริการ ส่งคืนรหัสงานทันที และติดตามงานนั้นในบัญชีงาน เอเจนต์จะยังคงตอบข้อความอื่นต่อไปในขณะที่งานกำลังทำงานอยู่ เมื่อผู้ให้บริการดำเนินการเสร็จ OpenClaw จะปลุกเอเจนต์เพื่อให้โพสต์สื่อที่เสร็จแล้วกลับไปยังช่องทางเดิม การสร้างรูปภาพและ TTS เป็นแบบซิงโครนัสและเสร็จสิ้นภายในคำตอบเดียวกัน

Deepgram, ElevenLabs, Mistral, OpenAI และ xAI ทั้งหมดสามารถถอดเสียง
เสียงขาเข้าผ่านเส้นทาง batch `tools.media.audio` ได้เมื่อมีการกำหนดค่าไว้ Deepgram,
ElevenLabs, Mistral, OpenAI และ xAI ยังลงทะเบียนผู้ให้บริการ STT แบบสตรีมมิงสำหรับ Voice Call
ด้วย ดังนั้นเสียงโทรศัพท์สดจึงสามารถส่งต่อไปยังผู้ให้บริการที่เลือกได้
โดยไม่ต้องรอให้การบันทึกเสร็จสมบูรณ์ก่อน

Google รองรับพื้นผิวการใช้งานด้านรูปภาพ วิดีโอ เพลง batch TTS เสียงแบบเรียลไทม์ฝั่งแบ็กเอนด์
และการทำความเข้าใจสื่อของ OpenClaw ส่วน OpenAI รองรับพื้นผิวการใช้งานด้านรูปภาพ
วิดีโอ batch TTS batch STT, Voice Call streaming STT, เสียงแบบเรียลไทม์ฝั่งแบ็กเอนด์
และการฝังหน่วยความจำ xAI ในปัจจุบันรองรับพื้นผิวการใช้งานด้านรูปภาพ วิดีโอ
การค้นหา การรันโค้ด batch TTS, batch STT และ Voice Call streaming STT
ของ OpenClaw เสียง Realtime ของ xAI เป็นความสามารถจากต้นทาง แต่ยังไม่ได้
ลงทะเบียนใน OpenClaw จนกว่าสัญญาเสียงแบบเรียลไทม์ที่ใช้ร่วมกันจะสามารถรองรับ
ความสามารถนั้นได้
