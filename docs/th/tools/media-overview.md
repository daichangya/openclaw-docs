# การสร้างและการทำความเข้าใจสื่อ

OpenClaw สร้างภาพ วิดีโอ และเพลง ทำความเข้าใจสื่อขาเข้า (ภาพ เสียง วิดีโอ) และพูดคำตอบออกเสียงด้วยการแปลงข้อความเป็นเสียง ความสามารถด้านสื่อทั้งหมดขับเคลื่อนด้วยเครื่องมือ: เอเจนต์จะตัดสินใจว่าจะใช้เมื่อใดตามบทสนทนา และแต่ละเครื่องมือจะปรากฏขึ้นก็ต่อเมื่อมีผู้ให้บริการเบื้องหลังอย่างน้อยหนึ่งรายที่ถูกกำหนดค่าไว้

## ภาพรวมความสามารถ

| ความสามารถ | เครื่องมือ | ผู้ให้บริการ | สิ่งที่ทำ |
| -------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| การสร้างภาพ | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI | สร้างหรือแก้ไขภาพจากพรอมป์ข้อความหรือภาพอ้างอิง |
| การสร้างวิดีโอ | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | สร้างวิดีโอจากข้อความ ภาพ หรือวิดีโอที่มีอยู่ |
| การสร้างเพลง | `music_generate` | ComfyUI, Google, MiniMax | สร้างเพลงหรือแทร็กเสียงจากพรอมป์ข้อความ |
| การแปลงข้อความเป็นเสียง (TTS) | `tts` | ElevenLabs, Microsoft, MiniMax, OpenAI, xAI | แปลงคำตอบขาออกเป็นเสียงพูด |
| การทำความเข้าใจสื่อ | (อัตโนมัติ) | ผู้ให้บริการโมเดลใดก็ตามที่รองรับ vision/audio รวมถึง CLI fallbacks | สรุปภาพ เสียง และวิดีโอขาเข้า |

## การสร้างแบบ async ทำงานอย่างไร

การสร้างวิดีโอและเพลงทำงานเป็นงานเบื้องหลัง เนื่องจากการประมวลผลของผู้ให้บริการมักใช้เวลาตั้งแต่ 30 วินาทีไปจนถึงหลายนาที เมื่อเอเจนต์เรียก `video_generate` หรือ `music_generate` OpenClaw จะส่งคำขอไปยังผู้ให้บริการ คืน task ID กลับมาทันที และติดตามงานนั้นใน task ledger เอเจนต์ยังคงตอบข้อความอื่นต่อไปได้ในขณะที่งานกำลังรัน เมื่อผู้ให้บริการทำเสร็จ OpenClaw จะปลุกเอเจนต์เพื่อให้สามารถโพสต์สื่อที่เสร็จแล้วกลับไปยังช่องทางเดิมได้ ส่วนการสร้างภาพและ TTS เป็นแบบ synchronous และเสร็จสิ้นในบรรทัดเดียวกับคำตอบ

Deepgram, ElevenLabs, Mistral, OpenAI และ xAI สามารถถอดเสียง
ขาเข้าผ่านเส้นทาง batch `tools.media.audio` ได้ทั้งหมดเมื่อมีการกำหนดค่าไว้ Deepgram,
ElevenLabs, Mistral, OpenAI และ xAI ยังลงทะเบียนผู้ให้บริการ STT แบบสตรีมสำหรับ
Voice Call ด้วย ดังนั้นเสียงโทรศัพท์แบบสดจึงสามารถส่งต่อไปยังผู้ให้บริการที่เลือกได้
โดยไม่ต้องรอให้การบันทึกเสร็จสิ้น

OpenAI แมปไปยังพื้นผิวด้านภาพ วิดีโอ TTS แบบ batch STT แบบ batch
STT แบบสตรีมสำหรับ Voice Call เสียงแบบเรียลไทม์ และ memory embedding
ของ OpenClaw ปัจจุบัน xAI แมปไปยังพื้นผิวด้านภาพ วิดีโอ การค้นหา การรันโค้ด TTS แบบ batch STT แบบ batch
และ STT แบบสตรีมสำหรับ Voice Call ของ OpenClaw เสียงแบบเรียลไทม์ของ xAI เป็นความสามารถ
ของต้นทาง แต่ยังไม่ถูกลงทะเบียนใน OpenClaw จนกว่าสัญญาเสียงแบบเรียลไทม์ร่วม
จะสามารถแสดงความสามารถนั้นได้
