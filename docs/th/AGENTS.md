# คู่มือเอกสาร

ไดเรกทอรีนี้ดูแลการเขียนเอกสาร กฎลิงก์ของ Mintlify และนโยบาย i18n ของเอกสาร

## กฎของ Mintlify

- เอกสารถูกโฮสต์บน Mintlify (`https://docs.openclaw.ai`)
- ลิงก์เอกสารภายในใน `docs/**/*.md` ต้องคงเป็นพาธสัมพัทธ์จากราก และไม่มีนามสกุล `.md` หรือ `.mdx` (ตัวอย่าง: `[Config](/configuration)`)
- การอ้างอิงข้ามถึงส่วนต่าง ๆ ควรใช้แองเคอร์บนพาธสัมพัทธ์จากราก (ตัวอย่าง: `[Hooks](/configuration#hooks)`)
- หัวข้อเอกสารควรหลีกเลี่ยง em dash และเครื่องหมายอะพอสทรอฟี เพราะการสร้างแองเคอร์ของ Mintlify มีความเปราะบางกับสิ่งเหล่านี้
- README และเอกสารอื่น ๆ ที่แสดงผลบน GitHub ควรใช้ URL เอกสารแบบสมบูรณ์ เพื่อให้ลิงก์ทำงานได้นอก Mintlify
- เนื้อหาเอกสารต้องคงความเป็นกลาง: ห้ามใช้ชื่ออุปกรณ์ ชื่อโฮสต์ หรือพาธภายในเครื่องของบุคคล ให้ใช้ตัวยึดตำแหน่งอย่าง `user@gateway-host`

## กฎเนื้อหาเอกสาร

- สำหรับเอกสาร ข้อความ UI และรายการตัวเลือก ให้เรียงบริการ/ผู้ให้บริการตามตัวอักษร เว้นแต่ว่าส่วนนั้นจะอธิบายลำดับรันไทม์หรือลำดับการตรวจจับอัตโนมัติโดยเฉพาะ
- รักษาการตั้งชื่อ plugin ที่มากับระบบให้สอดคล้องกับกฎคำศัพท์ plugin ทั่วทั้งรีโปใน `AGENTS.md` ระดับราก

## i18n ของเอกสาร

- เอกสารภาษาต่างประเทศไม่ได้รับการดูแลในรีโปนี้ เอาต์พุตการเผยแพร่ที่สร้างขึ้นจะอยู่ในรีโป `openclaw/docs` แยกต่างหาก (มักโคลนไว้ในเครื่องเป็น `../openclaw-docs`)
- ห้ามเพิ่มหรือแก้ไขเอกสารที่แปลแล้วภายใต้ `docs/<locale>/**` ที่นี่
- ให้ถือว่าเอกสารภาษาอังกฤษในรีโปนี้พร้อมกับไฟล์อภิธานศัพท์เป็นแหล่งข้อมูลจริง
- ไปป์ไลน์: อัปเดตเอกสารภาษาอังกฤษที่นี่ อัปเดต `docs/.i18n/glossary.<locale>.json` ตามความจำเป็น จากนั้นปล่อยให้การซิงก์รีโปสำหรับเผยแพร่และ `scripts/docs-i18n` ทำงานใน `openclaw/docs`
- ก่อนรัน `scripts/docs-i18n` ใหม่ ให้เพิ่มรายการอภิธานศัพท์สำหรับคำศัพท์ทางเทคนิค ชื่อหน้า หรือป้ายกำกับนำทางสั้น ๆ ใหม่ ๆ ที่ต้องคงเป็นภาษาอังกฤษหรือใช้คำแปลแบบตายตัว
- `pnpm docs:check-i18n-glossary` คือการตรวจสอบสำหรับชื่อเอกสารภาษาอังกฤษที่มีการเปลี่ยนแปลง และป้ายกำกับลิงก์เอกสารภายในแบบสั้น
- หน่วยความจำการแปลจะอยู่ในไฟล์ `docs/.i18n/*.tm.jsonl` ที่สร้างขึ้นในรีโปสำหรับเผยแพร่
- ดู `docs/.i18n/README.md`
