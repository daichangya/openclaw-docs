# วันที่และเวลา

OpenClaw ใช้ค่าเริ่มต้นเป็น **เวลาท้องถิ่นของโฮสต์สำหรับเวลาประทับของการรับส่งข้อมูล** และใช้ **เขตเวลาของผู้ใช้เฉพาะใน system prompt**
เวลาประทับจากผู้ให้บริการจะถูกเก็บรักษาไว้เพื่อให้เครื่องมือยังคงมีความหมายแบบเนทีฟของตนเอง (เวลาปัจจุบันดูได้ผ่าน `session_status`)

## envelope ของข้อความ (ใช้เวลาท้องถิ่นเป็นค่าเริ่มต้น)

ข้อความขาเข้าจะถูกห่อด้วยเวลาประทับ (ละเอียดระดับนาที):

```
[Provider ... 2026-01-05 16:26 PST] message text
```

เวลาประทับใน envelope นี้จะใช้ **เวลาท้องถิ่นของโฮสต์เป็นค่าเริ่มต้น** โดยไม่ขึ้นกับเขตเวลาของผู้ให้บริการ

คุณสามารถ override พฤติกรรมนี้ได้:

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | IANA timezone
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

- `envelopeTimezone: "utc"` ใช้ UTC
- `envelopeTimezone: "local"` ใช้เขตเวลาของโฮสต์
- `envelopeTimezone: "user"` ใช้ `agents.defaults.userTimezone` (หากไม่มีจะ fallback ไปยังเขตเวลาของโฮสต์)
- ใช้เขตเวลา IANA แบบระบุชัดเจน (เช่น `"America/Chicago"`) สำหรับโซนคงที่
- `envelopeTimestamp: "off"` จะลบเวลาประทับแบบสัมบูรณ์ออกจากส่วนหัวของ envelope
- `envelopeElapsed: "off"` จะลบส่วนต่อท้ายเวลาแบบผ่านไปแล้ว (รูปแบบ `+2m`)

### ตัวอย่าง

**เวลาท้องถิ่น (ค่าเริ่มต้น):**

```
[WhatsApp +1555 2026-01-18 00:19 PST] hello
```

**เขตเวลาของผู้ใช้:**

```
[WhatsApp +1555 2026-01-18 00:19 CST] hello
```

**เปิดใช้เวลาที่ผ่านไปแล้ว:**

```
[WhatsApp +1555 +30s 2026-01-18T05:19Z] follow-up
```

## system prompt: วันที่และเวลาปัจจุบัน

หากทราบเขตเวลาของผู้ใช้ system prompt จะรวมส่วน
**วันที่และเวลาปัจจุบัน** โดยมี **เฉพาะเขตเวลา** (ไม่มีรูปแบบนาฬิกา/เวลา)
เพื่อให้ prompt cache คงที่:

```
Time zone: America/Chicago
```

เมื่อเอเจนต์ต้องการเวลาปัจจุบัน ให้ใช้เครื่องมือ `session_status`; การ์ดสถานะจะมีบรรทัดเวลาประทับรวมอยู่

## บรรทัด system event (ใช้เวลาท้องถิ่นเป็นค่าเริ่มต้น)

system event ที่ถูกนำเข้าคิวและแทรกในบริบทของเอเจนต์จะมีเวลาประทับนำหน้า โดยใช้การเลือกเขตเวลาเดียวกับ envelope ของข้อความ (ค่าเริ่มต้น: เวลาท้องถิ่นของโฮสต์)

```
System: [2026-01-12 12:19:17 PST] Model switched.
```

### กำหนดค่าเขตเวลาของผู้ใช้ + รูปแบบเวลา

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
      timeFormat: "auto", // auto | 12 | 24
    },
  },
}
```

- `userTimezone` ตั้งค่า **เขตเวลาท้องถิ่นของผู้ใช้** สำหรับบริบทของพรอมป์
- `timeFormat` ควบคุมการแสดงผลแบบ **12 ชั่วโมง/24 ชั่วโมง** ในพรอมป์ โดย `auto` จะอิงตามค่ากำหนดของระบบปฏิบัติการ

## การตรวจจับรูปแบบเวลา (auto)

เมื่อ `timeFormat: "auto"` OpenClaw จะตรวจสอบค่ากำหนดของระบบปฏิบัติการ (macOS/Windows)
และ fallback ไปใช้การจัดรูปแบบตาม locale ค่าที่ตรวจพบจะถูก **แคชต่อโปรเซส**
เพื่อหลีกเลี่ยงการเรียกระบบซ้ำ ๆ

## payload ของเครื่องมือ + ตัวเชื่อมต่อ (เวลาผู้ให้บริการแบบดิบ + ฟิลด์ที่ทำ normalization แล้ว)

เครื่องมือของช่องทางจะส่งกลับ **เวลาประทับแบบเนทีฟของผู้ให้บริการ** และเพิ่มฟิลด์ที่ทำ normalization แล้วเพื่อความสม่ำเสมอ:

- `timestampMs`: epoch milliseconds (UTC)
- `timestampUtc`: สตริง UTC แบบ ISO 8601

ฟิลด์ดิบจากผู้ให้บริการจะถูกเก็บไว้ครบเพื่อไม่ให้ข้อมูลสูญหาย

- Slack: สตริงลักษณะ epoch จาก API
- Discord: เวลาประทับ UTC แบบ ISO
- Telegram/WhatsApp: เวลาประทับแบบตัวเลข/ISO ตามผู้ให้บริการ

หากคุณต้องการเวลาท้องถิ่น ให้แปลงต่อในขั้นตอนปลายทางโดยใช้เขตเวลาที่ทราบ
