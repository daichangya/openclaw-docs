# การมีส่วนร่วมกับ Threat Model ของ OpenClaw

ขอบคุณที่ช่วยทำให้ OpenClaw ปลอดภัยยิ่งขึ้น Threat model นี้เป็นเอกสารที่มีการพัฒนาอย่างต่อเนื่อง และเรายินดีรับการมีส่วนร่วมจากทุกคน - คุณไม่จำเป็นต้องเป็นผู้เชี่ยวชาญด้านความปลอดภัยก็ได้

## วิธีในการมีส่วนร่วม

### เพิ่มภัยคุกคาม

พบเวกเตอร์การโจมตีหรือความเสี่ยงที่เรายังไม่ได้ครอบคลุมหรือไม่? เปิด issue บน [openclaw/trust](https://github.com/openclaw/trust/issues) แล้วอธิบายด้วยคำพูดของคุณเอง คุณไม่จำเป็นต้องรู้จัก framework ใดๆ หรือกรอกครบทุกช่อง - แค่อธิบายสถานการณ์ก็พอ

**สิ่งที่ควรใส่ (แต่ไม่บังคับ):**

- สถานการณ์การโจมตีและวิธีที่อาจถูกใช้โจมตีได้
- ส่วนใดของ OpenClaw ที่ได้รับผลกระทบ (CLI, gateway, channels, ClawHub, MCP servers ฯลฯ)
- คุณคิดว่ารุนแรงแค่ไหน (low / medium / high / critical)
- ลิงก์ไปยังงานวิจัยที่เกี่ยวข้อง, CVEs หรือกรณีตัวอย่างในโลกจริง

เราจะจัดการเรื่องการแมปไปยัง ATLAS, threat IDs และการประเมินความเสี่ยงระหว่างการทบทวนเอง หากคุณต้องการใส่รายละเอียดเหล่านั้นมาด้วยก็เยี่ยม - แต่ไม่ได้คาดหวังไว้

> **นี่มีไว้สำหรับการเพิ่มข้อมูลใน threat model ไม่ใช่สำหรับรายงานช่องโหว่ที่ยังใช้งานโจมตีได้จริง** หากคุณพบช่องโหว่ที่สามารถใช้โจมตีได้ โปรดดู [Trust page](https://trust.openclaw.ai) ของเราสำหรับคำแนะนำเรื่อง responsible disclosure

### เสนอแนวทางลดความเสี่ยง

มีไอเดียว่าจะจัดการกับภัยคุกคามที่มีอยู่แล้วอย่างไรหรือไม่? เปิด issue หรือ PR โดยอ้างอิงถึงภัยคุกคามนั้น แนวทางลดความเสี่ยงที่มีประโยชน์ควรเฉพาะเจาะจงและนำไปปฏิบัติได้จริง - เช่น "จำกัดอัตราต่อผู้ส่งที่ 10 ข้อความ/นาทีที่ gateway" ดีกว่า "ทำ rate limiting"

### เสนอ Attack Chain

attack chains แสดงให้เห็นว่าภัยคุกคามหลายรายการรวมกันเป็นสถานการณ์โจมตีที่สมจริงได้อย่างไร หากคุณเห็นการผสมกันที่อันตราย ให้บรรยายขั้นตอนและวิธีที่ผู้โจมตีจะเชื่อมแต่ละส่วนเข้าด้วยกัน คำบรรยายสั้นๆ ว่าการโจมตีเกิดขึ้นจริงอย่างไรในทางปฏิบัติมีค่ามากกว่าเทมเพลตแบบเป็นทางการ

### แก้ไขหรือปรับปรุงเนื้อหาที่มีอยู่

พิมพ์ผิด, ต้องการชี้แจง, ข้อมูลล้าสมัย, ตัวอย่างที่ดีกว่า - ยินดีรับ PR โดยไม่ต้องเปิด issue ก่อน

## สิ่งที่เราใช้

### MITRE ATLAS

threat model นี้สร้างบน [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) ซึ่งเป็น framework ที่ออกแบบมาเฉพาะสำหรับภัยคุกคามด้าน AI/ML เช่น prompt injection, tool misuse และการโจมตีเอเจนต์ คุณไม่จำเป็นต้องรู้จัก ATLAS เพื่อจะมีส่วนร่วม - เราจะทำการแมปสิ่งที่ส่งเข้ามาเข้ากับ framework นี้ระหว่างการทบทวน

### Threat IDs

ภัยคุกคามแต่ละรายการจะมี ID เช่น `T-EXEC-003` หมวดหมู่มีดังนี้:

| โค้ด    | หมวดหมู่                                   |
| ------- | ------------------------------------------ |
| RECON   | การลาดตระเวน - การรวบรวมข้อมูล             |
| ACCESS  | การเข้าถึงเริ่มต้น - การเข้าสู่ระบบ         |
| EXEC    | การรันคำสั่ง - การกระทำที่เป็นอันตราย      |
| PERSIST | การคงอยู่ - การรักษาการเข้าถึงไว้           |
| EVADE   | การหลบหลีกการป้องกัน - การหลีกเลี่ยงการตรวจจับ |
| DISC    | การค้นพบ - การเรียนรู้เกี่ยวกับสภาพแวดล้อม |
| EXFIL   | การขโมยข้อมูล - การนำข้อมูลออกไป            |
| IMPACT  | ผลกระทบ - ความเสียหายหรือการหยุดชะงัก      |

maintainers จะเป็นผู้กำหนด IDs ระหว่างการทบทวน คุณไม่จำเป็นต้องเลือกเอง

### ระดับความเสี่ยง

| ระดับ         | ความหมาย                                                         |
| ------------- | ---------------------------------------------------------------- |
| **Critical**  | ระบบถูกยึดทั้งหมด หรือมีโอกาสเกิดสูง + ผลกระทบระดับวิกฤต      |
| **High**      | มีแนวโน้มเกิดความเสียหายอย่างมีนัยสำคัญ หรือมีโอกาสเกิดปานกลาง + ผลกระทบระดับวิกฤต |
| **Medium**    | ความเสี่ยงปานกลาง หรือมีโอกาสเกิดต่ำ + ผลกระทบสูง             |
| **Low**       | มีโอกาสเกิดต่ำและผลกระทบจำกัด                                  |

หากคุณไม่แน่ใจเรื่องระดับความเสี่ยง เพียงอธิบายผลกระทบมา แล้วเราจะประเมินให้

## กระบวนการทบทวน

1. **Triage** - เราทบทวนสิ่งที่ส่งเข้ามาใหม่ภายใน 48 ชั่วโมง
2. **Assessment** - เราตรวจสอบความเป็นไปได้ กำหนดการแมปไปยัง ATLAS และ threat ID รวมถึงยืนยันระดับความเสี่ยง
3. **Documentation** - เราตรวจสอบให้แน่ใจว่าทุกอย่างถูกจัดรูปแบบและครบถ้วน
4. **Merge** - เพิ่มเข้าไปใน threat model และ visualization
