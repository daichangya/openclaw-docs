# การมีส่วนร่วมกับโมเดลภัยคุกคามของ OpenClaw

ขอบคุณที่ช่วยทำให้ OpenClaw ปลอดภัยยิ่งขึ้น โมเดลภัยคุกคามนี้เป็นเอกสารที่มีการพัฒนาอย่างต่อเนื่อง และเรายินดีรับการมีส่วนร่วมจากทุกคน — คุณไม่จำเป็นต้องเป็นผู้เชี่ยวชาญด้านความปลอดภัยก็ได้

## วิธีการมีส่วนร่วม

### เพิ่มภัยคุกคาม

พบเวกเตอร์การโจมตีหรือความเสี่ยงที่เรายังไม่ได้ครอบคลุมหรือไม่? เปิด issue บน [openclaw/trust](https://github.com/openclaw/trust/issues) แล้วอธิบายด้วยคำพูดของคุณเอง คุณไม่จำเป็นต้องรู้ framework ใด ๆ หรือกรอกทุกช่องให้ครบ — เพียงอธิบายสถานการณ์มาก็พอ

**สิ่งที่ควรใส่มาให้ (แต่ไม่บังคับ):**

- สถานการณ์การโจมตีและวิธีที่อาจถูกนำไปใช้โจมตี
- ส่วนใดของ OpenClaw ที่ได้รับผลกระทบ (CLI, Gateway, channels, ClawHub, เซิร์ฟเวอร์ MCP ฯลฯ)
- คุณคิดว่ามันรุนแรงแค่ไหน (ต่ำ / ปานกลาง / สูง / วิกฤต)
- ลิงก์ไปยังงานวิจัยที่เกี่ยวข้อง, CVE หรือกรณีตัวอย่างในโลกจริง

เราจะจัดการการแมปกับ ATLAS, threat ID และการประเมินความเสี่ยงระหว่างการทบทวนเอง หากคุณต้องการใส่รายละเอียดเหล่านั้นมาด้วยก็เยี่ยมมาก — แต่ไม่ได้คาดหวังไว้

> **นี่มีไว้สำหรับเพิ่มข้อมูลลงในโมเดลภัยคุกคาม ไม่ใช่การรายงานช่องโหว่ที่ยังใช้งานโจมตีได้จริง** หากคุณพบช่องโหว่ที่สามารถถูกนำไปใช้โจมตีได้ โปรดดู [หน้า Trust](https://trust.openclaw.ai) ของเราสำหรับคำแนะนำในการเปิดเผยอย่างรับผิดชอบ

### เสนอแนวทางบรรเทา

มีแนวคิดว่าจะจัดการกับภัยคุกคามที่มีอยู่แล้วอย่างไรหรือไม่? เปิด issue หรือ PR โดยอ้างอิงภัยคุกคามนั้น แนวทางบรรเทาที่มีประโยชน์ควรเฉพาะเจาะจงและนำไปปฏิบัติได้จริง — ตัวอย่างเช่น “จำกัดอัตราต่อผู้ส่งที่ 10 ข้อความ/นาทีที่ Gateway” ดีกว่า “ทำ rate limiting”

### เสนอห่วงโซ่การโจมตี

ห่วงโซ่การโจมตีแสดงให้เห็นว่าภัยคุกคามหลายอย่างรวมกันเป็นสถานการณ์โจมตีที่สมจริงได้อย่างไร หากคุณเห็นชุดการรวมกันที่อันตราย ให้อธิบายขั้นตอนและวิธีที่ผู้โจมตีจะเชื่อมโยงมันเข้าด้วยกัน เรื่องเล่าสั้น ๆ เกี่ยวกับการดำเนินไปของการโจมตีในทางปฏิบัติมีคุณค่ามากกว่าเทมเพลตแบบเป็นทางการ

### แก้ไขหรือปรับปรุงเนื้อหาที่มีอยู่

คำผิด, การทำให้ชัดเจนขึ้น, ข้อมูลที่ล้าสมัย, ตัวอย่างที่ดีกว่า — ยินดีรับ PR โดยไม่ต้องเปิด issue ก่อน

## สิ่งที่เราใช้

### MITRE ATLAS

โมเดลภัยคุกคามนี้สร้างขึ้นบน [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) ซึ่งเป็น framework ที่ออกแบบมาเฉพาะสำหรับภัยคุกคามด้าน AI/ML เช่น prompt injection, การใช้เครื่องมือในทางที่ผิด และการโจมตีเอเจนต์ คุณไม่จำเป็นต้องรู้จัก ATLAS เพื่อมีส่วนร่วม — เราจะแมปข้อมูลที่ส่งเข้ามาเข้ากับ framework นี้ระหว่างการทบทวน

### Threat ID

ภัยคุกคามแต่ละรายการจะได้รับ ID เช่น `T-EXEC-003` โดยมีหมวดหมู่ดังนี้:

| Code    | หมวดหมู่                                  |
| ------- | ----------------------------------------- |
| RECON   | Reconnaissance - การรวบรวมข้อมูล          |
| ACCESS  | Initial access - การได้สิทธิ์เข้าถึงเริ่มต้น |
| EXEC    | Execution - การรันการกระทำที่เป็นอันตราย   |
| PERSIST | Persistence - การคงการเข้าถึงไว้          |
| EVADE   | Defense evasion - การหลีกเลี่ยงการตรวจจับ |
| DISC    | Discovery - การเรียนรู้เกี่ยวกับสภาพแวดล้อม |
| EXFIL   | Exfiltration - การขโมยข้อมูล              |
| IMPACT  | Impact - ความเสียหายหรือการรบกวน          |

ผู้ดูแลจะเป็นผู้กำหนด ID ระหว่างการทบทวน คุณไม่จำเป็นต้องเลือกเอง

### ระดับความเสี่ยง

| ระดับ        | ความหมาย                                                         |
| ------------ | ---------------------------------------------------------------- |
| **วิกฤต**    | ระบบถูกยึดทั้งหมด หรือมีความเป็นไปได้สูงพร้อมผลกระทบระดับวิกฤต |
| **สูง**      | มีแนวโน้มเกิดความเสียหายอย่างมีนัยสำคัญ หรือมีความเป็นไปได้ปานกลางพร้อมผลกระทบระดับวิกฤต |
| **ปานกลาง** | ความเสี่ยงระดับปานกลาง หรือมีความเป็นไปได้ต่ำพร้อมผลกระทบสูง |
| **ต่ำ**      | ไม่น่าจะเกิดขึ้นและมีผลกระทบจำกัด                             |

หากคุณไม่แน่ใจเรื่องระดับความเสี่ยง เพียงอธิบายผลกระทบมา แล้วเราจะเป็นผู้ประเมิน

## กระบวนการทบทวน

1. **คัดกรองเบื้องต้น** - เราทบทวนข้อมูลที่ส่งเข้ามาใหม่ภายใน 48 ชั่วโมง
2. **การประเมิน** - เราตรวจสอบความเป็นไปได้ กำหนดการแมปกับ ATLAS และ threat ID และยืนยันระดับความเสี่ยง
3. **การจัดทำเอกสาร** - เราตรวจสอบให้แน่ใจว่าทุกอย่างมีรูปแบบถูกต้องและครบถ้วน
4. **Merge** - เพิ่มเข้าสู่โมเดลภัยคุกคามและการแสดงผลภาพ
