ติดตั้งใช้งาน OpenClaw บน Railway ด้วยเทมเพลตแบบคลิกครั้งเดียว และเข้าถึงผ่านเว็บ Control UI
นี่คือเส้นทางที่ง่ายที่สุดแบบ "ไม่ต้องใช้เทอร์มินัลบนเซิร์ฟเวอร์": Railway จะรัน Gateway ให้คุณ

## รายการตรวจสอบอย่างรวดเร็ว (ผู้ใช้ใหม่)

1. คลิก **Deploy on Railway** (ด้านล่าง)
2. เพิ่ม **Volume** ที่ mount ที่ `/data`
3. ตั้งค่า **Variables** ที่จำเป็น (อย่างน้อย `OPENCLAW_GATEWAY_PORT` และ `OPENCLAW_GATEWAY_TOKEN`)
4. เปิดใช้ **HTTP Proxy** บนพอร์ต `8080`
5. เปิด `https://<your-railway-domain>/openclaw` และเชื่อมต่อโดยใช้ shared secret ที่กำหนดไว้ เทมเพลตนี้ใช้ `OPENCLAW_GATEWAY_TOKEN` เป็นค่าเริ่มต้น; หากคุณแทนที่ด้วย password auth ให้ใช้รหัสผ่านนั้นแทน

## ติดตั้งใช้งานแบบคลิกครั้งเดียว

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

หลังติดตั้งใช้งาน ให้ค้นหา URL สาธารณะของคุณใน **Railway → service ของคุณ → Settings → Domains**

Railway จะ:

- ให้โดเมนที่สร้างขึ้นอัตโนมัติ (มักเป็น `https://<something>.up.railway.app`) หรือ
- ใช้โดเมนแบบกำหนดเองของคุณ หากคุณเชื่อมไว้แล้ว

จากนั้นเปิด:

- `https://<your-railway-domain>/openclaw` — Control UI

## สิ่งที่คุณจะได้รับ

- OpenClaw Gateway + Control UI ที่โฮสต์ไว้
- พื้นที่เก็บข้อมูลถาวรผ่าน Railway Volume (`/data`) เพื่อให้ `openclaw.json`,
  `auth-profiles.json` ต่อเอเจนต์, state ของช่องทาง/provider, เซสชัน และ
  workspace อยู่รอดข้ามการ redeploy

## การตั้งค่า Railway ที่จำเป็น

### Public Networking

เปิดใช้ **HTTP Proxy** สำหรับ service

- พอร์ต: `8080`

### Volume (จำเป็น)

แนบ volume ที่ mount ที่:

- `/data`

### Variables

ตั้งค่าตัวแปรเหล่านี้บน service:

- `OPENCLAW_GATEWAY_PORT=8080` (จำเป็น — ต้องตรงกับพอร์ตใน Public Networking)
- `OPENCLAW_GATEWAY_TOKEN` (จำเป็น; ให้ถือว่าเป็น admin secret)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (แนะนำ)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (แนะนำ)

## เชื่อมต่อช่องทาง

ใช้ Control UI ที่ `/openclaw` หรือรัน `openclaw onboard` ผ่าน shell ของ Railway เพื่อดูคำแนะนำการตั้งค่าช่องทาง:

- [Telegram](/th/channels/telegram) (เร็วที่สุด — ใช้เพียง bot token)
- [Discord](/th/channels/discord)
- [All channels](/th/channels)

## การสำรองข้อมูลและการย้ายระบบ

ส่งออก state, config, auth profile และ workspace ของคุณ:

```bash
openclaw backup create
```

คำสั่งนี้จะสร้าง backup archive แบบพกพาที่มี state ของ OpenClaw พร้อม workspace
ที่กำหนดค่าไว้ ดู [Backup](/cli/backup) สำหรับรายละเอียด
