ปรับใช้ OpenClaw บน Northflank ด้วยเทมเพลตแบบคลิกเดียว และเข้าถึงผ่าน web Control UI
นี่คือเส้นทางที่ง่ายที่สุดแบบ “ไม่ต้องใช้เทอร์มินัลบนเซิร์ฟเวอร์”: Northflank จะรัน Gateway ให้คุณ

## วิธีเริ่มต้น

1. คลิก [Deploy OpenClaw](https://northflank.com/stacks/deploy-openclaw) เพื่อเปิดเทมเพลต
2. สร้าง[บัญชีบน Northflank](https://app.northflank.com/signup) หากคุณยังไม่มี
3. คลิก **Deploy OpenClaw now**
4. ตั้งค่าตัวแปรสภาพแวดล้อมที่จำเป็น: `OPENCLAW_GATEWAY_TOKEN` (ใช้ค่าที่สุ่มอย่างแข็งแรง)
5. คลิก **Deploy stack** เพื่อ build และรันเทมเพลต OpenClaw
6. รอให้การปรับใช้เสร็จ จากนั้นคลิก **View resources**
7. เปิดบริการ OpenClaw
8. เปิด URL สาธารณะของ OpenClaw ที่ `/openclaw` แล้วเชื่อมต่อโดยใช้ shared secret ที่ตั้งค่าไว้ เทมเพลตนี้ใช้ `OPENCLAW_GATEWAY_TOKEN` โดยค่าเริ่มต้น; หากคุณแทนที่ด้วย password auth ให้ใช้ password นั้นแทน

## สิ่งที่คุณจะได้รับ

- OpenClaw Gateway + Control UI แบบโฮสต์แล้ว
- พื้นที่จัดเก็บแบบคงทนผ่าน Northflank Volume (`/data`) เพื่อให้ `openclaw.json`,
  `auth-profiles.json` แยกตามเอเจนต์, สถานะของช่องทาง/ผู้ให้บริการ, เซสชัน และ
  workspace ยังคงอยู่แม้มีการ deploy ใหม่

## เชื่อมต่อช่องทาง

ใช้ Control UI ที่ `/openclaw` หรือรัน `openclaw onboard` ผ่าน SSH เพื่อดูคำแนะนำการตั้งค่าช่องทาง:

- [Telegram](/th/channels/telegram) (เร็วที่สุด — ใช้แค่ bot token)
- [Discord](/th/channels/discord)
- [ทุกช่องทาง](/th/channels)
